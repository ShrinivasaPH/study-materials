What are Arrays? Easy
Arrays are the most fundamental data structure. They store multiple elements of the same type in contiguous memory locations, allowing O(1) random access. Think of them as a row of boxes where each box holds a value and can be accessed instantly using its index.
Access: O(1)
Search: O(n)
Insertion: O(n)
Deletion: O(n)
Creating and Accessing Arrays
# Method 1: Using list literal
arr = [1, 2, 3, 4, 5]

# Method 2: Using list constructor
arr = list("hello")  # ['h', 'e', 'l', 'l', 'o']

# Access elements (0-indexed)
first = arr[0]  # 1
last = arr[-1]  # 5 (negative indexing from end)

# Slicing (returns new array)
subset = arr[1:4]  # [2, 3, 4]
🎓 Fun Fact: The concept of zero-based indexing was chosen by Ada Lovelace's colleague, and it stuck! Early programming languages like FORTRAN used 1-based indexing, but C popularized 0-based indexing due to how pointer arithmetic works.
Common Array Operations
arr = [10, 20, 30]

# Add element (O(1) amortized)
arr.append(40)  # [10, 20, 30, 40]

# Insert at specific position (O(n))
arr.insert(1, 15)  # [10, 15, 20, 30, 40]

# Remove element (O(n))
arr.remove(20)  # [10, 15, 30, 40]

# Pop from end (O(1))
last = arr.pop()  # last = 40

# Extend array with another array
arr.extend([50, 60])  # [10, 15, 30, 50, 60]

Finding Maximum Element - Two Approaches
# Approach 1: Built-in function (Pythonic)
def find_max_builtin(arr):
    return max(arr)

# Time: O(n), Space: O(1)

# Approach 2: Manual iteration (Better for interviews)
def find_max_manual(arr):
    if not arr:
        return None
    
    max_val = arr[0]
    for num in arr[1:]:
        if num > max_val:
            max_val = num
    
    return max_val

🔗 Linked Lists

Understanding Linked Lists Medium
Linked Lists are linear data structures where elements (nodes) are stored in a chain. Unlike arrays, linked lists don't require contiguous memory. Each node contains data and a reference to the next node. This makes insertion/deletion at known positions O(1), but accessing elements requires traversal.
Access: O(n)
Search: O(n)
Insertion: O(1)*
Deletion: O(1)*

*At known position with reference

Node and LinkedList Implementation
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None
    
    def append(self, data):
        """Add element at end"""
        new_node = Node(data)
        
        if not self.head:
            self.head = new_node
            return
        
        current = self.head
        while current.next:
            current = current.next
        current.next = new_node
    
    def display(self):
        """Print all elements"""
        elements = []
        current = self.head
        while current:
            elements.append(current.data)
            current = current.next
        return " → ".join(map(str, elements))
🎓 Fun Fact: Linked lists were invented to handle the problem of dynamic memory allocation. Before dynamic memory, programmers had to guess array sizes in advance. A wrong guess meant either wasted memory or a crash!
Reversing a Linked List (Classic Interview Question!)
# Iterative approach (O(n) time, O(1) space)
def reverse_linked_list(head):
    prev = None
    current = head
    
    while current:
        # Store next node
        next_temp = current.next
        
        # Reverse the link
        current.next = prev
        
        # Move pointers forward
        prev = current
        current = next_temp
    
    return prev  # New head

# Example: 1 → 2 → 3 becomes 3 → 2 → 1
Types of Linked Lists
Type Structure Reverse? Use Case
Singly Linked A → B → C → None Yes (reverse traversal) Stack, Basic chains
Doubly Linked None ← A ↔ B ↔ C → None Both directions Deques, Browser history
Circular Linked A → B → C → A (loops) Careful iteration Round-robin scheduling

⚖️ Stacks & Queues

Stacks: Last-In-First-Out (LIFO) Easy
A Stack follows the LIFO principle: the last element added is the first one removed. Imagine a stack of plates - you add and remove from the top. Perfect for problems involving matching pairs, undo functionality, and depth-first traversal.
Push: O(1)
Pop: O(1)
Peek: O(1)
Space: O(n)
Stack Implementation and Usage
# Method 1: Using list (Python)
stack = []
stack.append(1)      # push
stack.append(2)      
stack.append(3)      
top = stack.pop()     # pop → 3

# Method 2: Custom Stack class
class Stack:
    def __init__(self):
        self.items = []
    
    def push(self, item):
        self.items.append(item)
    
    def pop(self):
        return self.items.pop() if self.items else None
    
    def peek(self):
        return self.items[-1] if self.items else None
    
    def is_empty(self):
        return len(self.items) == 0
🎓 Fun Fact: The "Undo" functionality in every text editor uses a stack! Each action is pushed onto the stack, and Ctrl+Z pops the most recent action to undo it.
Classic Problem: Balanced Parentheses
def is_balanced(s):
    """Check if parentheses/brackets are balanced"""
    stack = []
    matching = {
        ')': '(', 
        ']': '[', 
        '}': '{'
    }
    
    for char in s:
        if char in '([{':
            stack.append(char)
        elif char in ')]}':
            if not stack or stack.pop() != matching[char]:
                return False
    
    return len(stack) == 0

# Tests
is_balanced("({[]})")  # True
is_balanced("([)]")    # False
Queues: First-In-First-Out (FIFO) Easy
A Queue follows the FIFO principle: the first element added is the first one removed. Like a real queue at a store - you serve the person who arrived first. Essential for BFS traversal and handling tasks in order.
Enqueue: O(1)
Dequeue: O(1)
Peek: O(1)
Space: O(n)
Queue Implementation
from collections import deque

# Method 1: Using collections.deque (Recommended)
queue = deque()
queue.append(1)       # enqueue
queue.append(2)
queue.append(3)
first = queue.popleft()  # dequeue → 1

# Method 2: Custom Queue class
class Queue:
    def __init__(self):
        self.items = deque()
    
    def enqueue(self, item):
        self.items.append(item)
    
    def dequeue(self):
        return self.items.popleft() if self.items else None
    
    def is_empty(self):
        return len(self.items) == 0

🌳 Trees & Binary Search Trees

Tree Fundamentals Medium
Trees are hierarchical data structures with a root node and zero or more child nodes. Each node (except root) has exactly one parent. Trees are crucial for organizing hierarchical data, searching, and recursive algorithms. Binary trees restrict each node to at most 2 children.
Tree Node and Binary Tree Implementation
class TreeNode:
    def __init__(self, val):
        self.val = val
        self.left = None
        self.right = None

# Create a simple tree:
#       1
#      / \
#     2   3

root = TreeNode(1)
root.left = TreeNode(2)
root.right = TreeNode(3)
Inorder Traversal (Left → Root → Right)
def inorder(root):
    """Left subtree → Node → Right subtree"""
    if not root:
        return
    
    inorder(root.left)
    print(root.val, end=' ')  # Process node
    inorder(root.right)

# For BST, inorder gives sorted sequence!
# Useful for: Getting sorted data from BST
Preorder Traversal (Root → Left → Right)
def preorder(root):
    """Node → Left subtree → Right subtree"""
    if not root:
        return
    
    print(root.val, end=' ')  # Process node
    preorder(root.left)
    preorder(root.right)

# Useful for: Copying tree, getting root first
# Example output: 1 2 4 5 3
Postorder Traversal (Left → Right → Root)
def postorder(root):
    """Left subtree → Right subtree → Node"""
    if not root:
        return
    
    postorder(root.left)
    postorder(root.right)
    print(root.val, end=' ')  # Process node

# Useful for: Deleting tree, post-processing
# Example output: 4 5 2 3 1
🎓 Fun Fact: The three traversal methods (Inorder, Preorder, Postorder) can reconstruct a unique binary tree if combined! But be careful - two traversals alone might not be enough.
Binary Search Trees (BST) Medium
A Binary Search Tree maintains the BST property: for every node, all values in the left subtree are smaller, and all values in the right subtree are larger. This enables fast searching, insertion, and deletion compared to unsorted arrays.
Search: O(log n) avg
Insert: O(log n) avg
Delete: O(log n) avg
Worst Case: O(n) unbalanced
BST Search and Insert
class BST:
    def search(self, root, val):
        """Search for value in BST"""
        if not root:
            return False
        
        if root.val == val:
            return True
        elif val < root.val:
            return self.search(root.left, val)
        else:
            return self.search(root.right, val)
    
    def insert(self, root, val):
        """Insert value maintaining BST property"""
        if not root:
            return TreeNode(val)
        
        if val < root.val:
            root.left = self.insert(root.left, val)
        elif val > root.val:
            root.right = self.insert(root.right, val)
        
        return root

📊 Sorting Algorithms

Sorting Algorithm Comparison Hard
Sorting is arranging elements in ascending or descending order. Different algorithms have different tradeoffs between time complexity, space complexity, and stability. Understanding when to use each is crucial for interviews.
Algorithm Time (Best) Time (Avg) Time (Worst) Space Stable?
Bubble Sort O(n) O(n²) O(n²) O(1) ✓ Yes
Selection Sort O(n²) O(n²) O(n²) O(1) ✗ No
Insertion Sort O(n) O(n²) O(n²) O(1) ✓ Yes
Merge Sort O(n log n) O(n log n) O(n log n) O(n) ✓ Yes
Quick Sort O(n log n) O(n log n) O(n²) O(log n) ✗ No
Heap Sort O(n log n) O(n log n) O(n log n) O(1) ✗ No
Merge Sort - Divide & Conquer O(n log n)
def merge_sort(arr):
    """Divide array, recursively sort, then merge"""
    if len(arr) <= 1:
        return arr
    
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    
    return merge(left, right)

def merge(left, right):
    """Merge two sorted arrays"""
    result = []
    i = j = 0
    
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    
    result.extend(left[i:])
    result.extend(right[j:])
    return result
🎓 Fun Fact: Python's built-in sort uses Timsort - a hybrid of merge sort and insertion sort! It was created by Tim Peters in 2002 and is incredibly efficient for real-world data.
Quick Sort - Fast Average Case O(n log n)
def quick_sort(arr):
    """Choose pivot, partition, recursively sort"""
    if len(arr) <= 1:
        return arr
    
    # Choose middle element as pivot
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    
    return (quick_sort(left) + middle + 
            quick_sort(right))

# In-place version (interview preferred):
def quick_sort_inplace(arr, low, high):
    if low < high:
        pi = partition(arr, low, high)
        quick_sort_inplace(arr, low, pi - 1)
        quick_sort_inplace(arr, pi + 1, high)

🔍 Searching Algorithms

Linear Search vs Binary Search Easy
Linear search checks every element sequentially - simple but slow. Binary search works on sorted arrays by eliminating half the remaining elements with each comparison - much faster but requires sorted data. Understanding when to use each is fundamental.
Linear Search: O(n)
Binary Search: O(log n)
Binary Search Implementation
# Recursive approach
def binary_search_recursive(arr, target, left, right):
    if left > right:
        return -1  # Not found
    
    mid = (left + right) // 2
    
    if arr[mid] == target:
        return mid
    elif arr[mid] > target:
        return binary_search_recursive(arr, target, 
                                        left, mid - 1)
    else:
        return binary_search_recursive(arr, target, 
                                        mid + 1, right)

# Iterative approach (more efficient, no stack overhead)
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    
    while left <= right:
        mid = (left + right) // 2
        
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    
    return -1  # Not found
🎮 Interactive: Binary Search Visualizer
🎓 Fun Fact: Donald Knuth calculated that binary search was misimplemented in most programs for over 20 years! The bug was in computing the middle index. Using mid = left + (right - left) // 2 prevents integer overflow.

🕸️ Graphs & Traversal

Graph Representations Hard
Graphs consist of vertices (nodes) and edges (connections). They model relationships like social networks, transportation routes, and web links. Choosing the right representation affects algorithm efficiency.
Representation Space Edge Lookup Edge Insertion Best For
Adjacency Matrix O(V²) O(1) O(1) Dense graphs
Adjacency List O(V+E) O(degree) O(1) Sparse graphs
Edge List O(E) O(E) O(1) Iterating edges
Graph Traversal: BFS vs DFS
from collections import deque

# Breadth-First Search (BFS) - Level order
def bfs(graph, start):
    visited = set()
    queue = deque([start])
    visited.add(start)
    
    while queue:
        node = queue.popleft()
        print(node, end=' ')
        
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)

# Depth-First Search (DFS) - Go deep
def dfs(graph, node, visited):
    visited.add(node)
    print(node, end=' ')
    
    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs(graph, neighbor, visited)
🎓 Fun Fact: BFS and DFS are like two ways of exploring a maze: BFS explores level by level (like expanding in circles), while DFS goes as deep as possible before backtracking (like following a wall).
