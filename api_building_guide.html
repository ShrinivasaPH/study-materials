Chapter 1

What exactly is an API?

Before writing a single line of code, let's build rock-solid mental models.

An API (Application Programming Interface) is a contract between two pieces of software. It defines how they talk to each other — what you can ask for, how to ask, and what you'll get back.

Think of a restaurant 🍽️. You (the client) don't go into the kitchen and cook your own food. Instead, you talk to the waiter (the API) who takes your order, passes it to the kitchen (the server), and brings back exactly what you asked for.

A REST API is a specific style of API that runs over HTTP — the same protocol your browser uses. It's stateless, meaning every request is self-contained and carries all the info needed.

🤯
Fun Fact

Roy Fielding coined the term "REST" in his 2000 PhD dissertation. He was also one of the principal authors of the HTTP spec. Talk about one productive thesis!

🌐

REST API

Uses HTTP verbs (GET, POST…) and URLs. Stateless. Returns JSON/XML. The most popular style today.

⚡

GraphQL API

Query exactly what you need, nothing more. Single endpoint. Developed by Facebook in 2012.

🔌

WebSocket API

Persistent two-way connection. Great for real-time apps: chat, live scores, stock tickers.

💡 REST vs HTTP: HTTP is the transport protocol. REST is a set of architectural constraints that use HTTP. All REST APIs use HTTP, but not all HTTP APIs are RESTful!

Chapter 2

The 5 HTTP Methods you must know

These verbs tell the server what you want to do. Click each one to explore.

GET

Read / Retrieve

Fetch data. Safe & idempotent. Never changes server state.

POST

Create

Submit new data. Not idempotent. Response: 201 Created.

PUT

Full Replace

Fully replace a resource. Idempotent. Must send all fields.

PATCH

Partial Update

Update only specific fields. More efficient than PUT.

DELETE

Remove

Delete a resource. Idempotent. Returns 204 or 200.

💡
Key Concept: Idempotency

An operation is idempotent if doing it multiple times gives the same result as doing it once. DELETE /users/5 ten times still results in user 5 being gone — the first call does the work, the rest are no-ops.

PUT vs PATCH — What's the Difference?

🔄 PUT — Full Replace
PUT /users/5
// You MUST send all fields
{
  "name": "Shrinivasa",
  "email": "new@email.com",
  "role": "admin",
  "age": 30
}
// Missing fields get nulled/defaulted!
🎯 PATCH — Partial Update
PATCH /users/5
// Send ONLY what changed
{
  "email": "new@email.com"
}
// Other fields unchanged ✅

HEAD & OPTIONS — the underrated two

🔭

HEAD

Like GET, but the server returns headers only — no body. Use it to check if a resource exists, get its Content-Length before downloading, or validate caching headers without wasting bandwidth.

🤝

OPTIONS

Ask the server "what methods do you support here?" Browsers send this automatically as a CORS preflight before cross-origin requests. Returns an Allow header listing valid methods.

HEAD & OPTIONS examples
# HEAD — check a file exists and get its size before downloading
HEAD /api/v1/exports/report-2025.pdf HTTP/1.1

# Response (no body)
HTTP/1.1 200 OK
Content-Length: 52428800
Content-Type: application/pdf
Last-Modified: Thu, 29 May 2025 09:00:00 GMT

# OPTIONS — discover allowed methods (browsers use for CORS)
OPTIONS /api/v1/products HTTP/1.1
Origin: https://myapp.com

# Response
HTTP/1.1 200 OK
Allow: GET, POST, HEAD, OPTIONS
Access-Control-Allow-Methods: GET, POST, PUT, PATCH, DELETE
Access-Control-Allow-Origin: https://myapp.com

Full CRUD with FastAPI

CRUD = Create, Read, Update, Delete. The four fundamental operations of any data-driven API.

python — Full CRUD
from fastapi import FastAPI, HTTPException, status
from pydantic import BaseModel, Field
from typing import Optional, List
from datetime import datetime

app = FastAPI()

class ProductCreate(BaseModel):
    name: str = Field(..., min_length=2, max_length=100)
    price: float = Field(..., gt=0)
    category: str
    in_stock: bool = True

class Product(ProductCreate):
    id: int
    created_at: str

products_db: dict[int, Product] = {}
_id = 0

# ─── CREATE ───────────────────────────────
@app.post("/products", response_model=Product,
         status_code=status.HTTP_201_CREATED)
async def create_product(product: ProductCreate):
    global _id
    _id += 1
    new_product = Product(
        id=_id,
        created_at=datetime.now().isoformat(),
        **product.dict()
    )
    products_db[_id] = new_product
    return new_product

# ─── READ ALL ─────────────────────────────
@app.get("/products", response_model=List[Product])
async def list_products(
    skip: int = 0,
    limit: int = Query(default=10, le=100),
    category: Optional[str] = None
):
    items = list(products_db.values())
    if category:
        items = [p for p in items if p.category == category]
    return items[skip : skip + limit]

# ─── READ ONE ─────────────────────────────
@app.get("/products/{product_id}", response_model=Product)
async def get_product(product_id: int):
    if product_id not in products_db:
        raise HTTPException(
            status_code=404,
            detail=f"Product {product_id} not found"
        )
    return products_db[product_id]

# ─── UPDATE (PATCH) ───────────────────────
@app.patch("/products/{product_id}", response_model=Product)
async def update_product(product_id: int, updates: dict):
    if product_id not in products_db:
        raise HTTPException(status_code=404, detail="Not found")
    product = products_db[product_id]
    updated = product.dict().update(updates)
    products_db[product_id] = Product(**updated)
    return products_db[product_id]

# ─── DELETE ───────────────────────────────
@app.delete("/products/{product_id}",
           status_code=status.HTTP_204_NO_CONTENT)
async def delete_product(product_id: int):
    if product_id not in products_db:
        raise HTTPException(status_code=404, detail="Not found")
    del products_db[product_id]

Data Validation with Pydantic

Pydantic is FastAPI's superpower. Define your data models once and get validation, serialization, and documentation for free.

python — Pydantic Models
from pydantic import BaseModel, Field, validator, EmailStr
from typing import Optional, List
from enum import Enum

class Role(str, Enum):
    admin  = "admin"
    user   = "user"
    viewer = "viewer"

class UserCreate(BaseModel):
    username: str = Field(
        ...,
        min_length=3,
        max_length=50,
        pattern=r"^[a-zA-Z0-9_]+$",
        example="shrinivasa_ph"
    )
    email: EmailStr
    age: Optional[int] = Field(None, ge=13, le=120)
    role: Role = Role.user
    tags: List[str] = []

    @validator("username")
    def username_not_reserved(cls, v):
        reserved = ["admin", "root", "system"]
        if v.lower() in reserved:
            raise ValueError("Username is reserved")
        return v

    class Config:
        schema_extra = {
            "example": {
                "username": "john_doe",
                "email": "john@example.com",
                "age": 28,
                "role": "user"
            }
        }

# FastAPI automatically returns 422 Unprocessable Entity
# with detailed error messages when validation fails!
🧙
Pydantic Magic

Field validators like ge (greater than or equal), le (less than or equal), min_length, max_length, and pattern are checked automatically before your route function even runs. No manual if-else validation needed!

JWT Authentication

JWT (JSON Web Tokens) are the industry standard for stateless API authentication. The server issues a token, and the client sends it with every request.

python — JWT Auth
from fastapi import FastAPI, Depends, HTTPException, status
from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm
from jose import JWTError, jwt
from passlib.context import CryptContext
from datetime import datetime, timedelta
from pydantic import BaseModel

# pip install python-jose passlib[bcrypt]

SECRET_KEY = "your-secret-key-keep-it-safe!"
ALGORITHM  = "HS256"
TOKEN_EXPIRE_MINUTES = 30

pwd_context = CryptContext(schemes=["bcrypt"])
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")
app = FastAPI()

def create_token(data: dict) -> str:
    payload = data.copy()
    payload["exp"] = datetime.utcnow() + timedelta(minutes=TOKEN_EXPIRE_MINUTES)
    return jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)

async def get_current_user(token: str = Depends(oauth2_scheme)):
    credentials_exception = HTTPException(
        status_code=status.HTTP_401_UNAUTHORIZED,
        detail="Invalid credentials",
        headers={"WWW-Authenticate": "Bearer"}
    )
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        username: str = payload.get("sub")
        if not username:
            raise credentials_exception
    except JWTError:
        raise credentials_exception
    return username

# 🔐 Protected route — requires valid JWT
@app.get("/me")
async def read_me(user: str = Depends(get_current_user)):
    return {"username": user, "msg": "Authenticated!"}

🔑 How JWT Works: Header.Payload.Signature — all base64 encoded. The signature is created using your SECRET_KEY, so only your server can verify it. Never store sensitive data in the payload — it's not encrypted, just encoded!

Async FastAPI — Handle 10,000 requests/sec

FastAPI is built on asyncio. Use async/await for I/O-bound operations (DB queries, external APIs) to handle massive concurrency.

python — Async Routes
import asyncio, httpx
from fastapi import FastAPI
from databases import Database

app = FastAPI()
db = Database("postgresql://user:pass@localhost/mydb")

@app.on_event("startup")
async def startup():
    await db.connect()

# ✅ Async DB query — non-blocking
@app.get("/users")
async def get_users():
    query = "SELECT * FROM users LIMIT 50"
    return await db.fetch_all(query)

# ✅ Fan-out: call 3 external APIs concurrently
@app.get("/dashboard/{user_id}")
async def get_dashboard(user_id: int):
    async with httpx.AsyncClient() as client:
        profile, orders, notifications = await asyncio.gather(
            client.get(f"/profile/{user_id}"),
            client.get(f"/orders/{user_id}"),
            client.get(f"/notifications/{user_id}")
        )
    return {
        "profile": profile.json(),
        "orders": orders.json(),
        "notifications": notifications.json()
    }
# Three API calls in parallel, not serial!
# 3 × 100ms = 100ms, not 300ms 🔥

@app.route("/users/<int:user_id>", methods=["GET", "DELETE"])
def user(user_id):
    if request.method == "GET":
        return jsonify({"id": user_id})
    elif request.method == "DELETE":
        return "", 204  # No content

# POST with JSON body
@app.route("/users", methods=["POST"])
def create_user():
    data = request.get_json()
    if not data or "name" not in data:
        return jsonify({"error": "name is required"}), 400
    return jsonify({"id": 1, **data}), 201

# Error handlers
@app.errorhandler(404)
def not_found(e):
    return jsonify({"error": "not found"}), 404

Flask CRUD with SQLAlchemy

python — Flask + SQLAlchemy CRUD
from flask import Flask, request, jsonify
from flask_sqlalchemy import SQLAlchemy

app = Flask(__name__)
app.config["SQLALCHEMY_DATABASE_URI"] = "sqlite:///products.db"
db = SQLAlchemy(app)

class Product(db.Model):
    id       = db.Column(db.Integer, primary_key=True)
    name     = db.Column(db.String(100), nullable=False)
    price    = db.Column(db.Float, nullable=False)
    in_stock = db.Column(db.Boolean, default=True)

    def to_dict(self):
        return {
            "id": self.id, "name": self.name,
            "price": self.price, "in_stock": self.in_stock
        }

@app.route("/products", methods=["GET"])
def list_products():
    page = request.args.get("page", 1, type=int)
    per_page = request.args.get("per_page", 10, type=int)
    paginated = Product.query.paginate(page=page, per_page=per_page)
    return jsonify({
        "items": [p.to_dict() for p in paginated.items],
        "total": paginated.total,
        "pages": paginated.pages
    })

@app.route("/products", methods=["POST"])
def create_product():
    data = request.get_json()
    product = Product(**data)
    db.session.add(product)
    db.session.commit()
    return jsonify(product.to_dict()), 201

Middleware & Decorators

python — Middleware
from flask import Flask, request, g
import time, logging

app = Flask(__name__)

# Before every request
@app.before_request
def start_timer():
    g.start = time.time()

# After every request
@app.after_request
def log_request(response):
    elapsed = time.time() - g.start
    logging.info(f"{request.method} {request.path} → {response.status_code} ({elapsed:.3f}s)")
    response.headers["X-Response-Time"] = f"{elapsed:.3f}s"
    return response

# Custom auth decorator
from functools import wraps

def require_api_key(f):
    @wraps(f)
    def decorated(*args, **kwargs):
        key = request.headers.get("X-API-Key")
        if key != "secret-key-123":
            return {"error": "Invalid API key"}, 401
        return f(*args, **kwargs)
    return decorated

@app.route("/secret")
@require_api_key
def secret_endpoint():
    return {"data": "🔐 Super secret!"}

DRF Serializers — Your Data's Bodyguard

python — serializers.py
from rest_framework import serializers
from .models import Product, Category

class CategorySerializer(serializers.ModelSerializer):
    class Meta:
        model = Category
        fields = ["id", "name"]

class ProductSerializer(serializers.ModelSerializer):
    # Nested serializer — includes full category object
    category = CategorySerializer(read_only=True)
    category_id = serializers.IntegerField(write_only=True)
    
    # Computed field
    full_name = serializers.SerializerMethodField()

    def get_full_name(self, obj):
        return f"{obj.category.name} — {obj.name}"

    def validate_price(self, value):
        if value <= 0:
            raise serializers.ValidationError("Price must be positive")
        return value

    class Meta:
        model = Product
        fields = ["id", "name", "price", "category", "category_id", "full_name"]
        read_only_fields = ["id", "full_name"]

ViewSets — Maximum Power, Minimum Code

from rest_framework import viewsets, filters
from rest_framework.decorators import action
from rest_framework.response import Response
from django_filters.rest_framework import DjangoFilterBackend

class ProductViewSet(viewsets.ModelViewSet):
    """
    Automatically provides:
    GET    /products/         → list()
    POST   /products/         → create()
    GET    /products/{id}/    → retrieve()
    PUT    /products/{id}/    → update()
    PATCH  /products/{id}/    → partial_update()
    DELETE /products/{id}/    → destroy()
    """
    queryset = Product.objects.all()
    serializer_class = ProductSerializer
    filter_backends = [DjangoFilterBackend, filters.SearchFilter]
    filterset_fields = ["category", "in_stock"]
    search_fields = ["name", "description"]

    # Custom action: POST /products/{id}/toggle_stock/
    @action(detail=True, methods=["post"])
    def toggle_stock(self, request, pk=None):
        product = self.get_object()
        product.in_stock = not product.in_stock
        product.save()
        return Response({"in_stock": product.in_stock})

from rest_framework.permissions import BasePermission

class IsOwnerOrReadOnly(BasePermission):
    """Allow owners to edit; everyone else read-only."""
    def has_object_permission(self, request, view, obj):
        if request.method in ("GET", "HEAD", "OPTIONS"):
            return True
        return obj.owner == request.user

class IsAdminOrReadOnly(BasePermission):
    def has_permission(self, request, view):
        if request.method in ("GET", "HEAD", "OPTIONS"):
            return True
        return request.user.is_staff

API Design Best Practices

Good APIs are intuitive, consistent, and pleasant to use. Follow these principles.

📐

Use Nouns for Endpoints

Resources should be nouns: /users, /products. Not /getUsers or /createProduct — that's what HTTP methods are for.

🔢

Version Your API

Always version: /api/v1/users. When you break things (and you will), v2 lets old clients keep working while new ones upgrade.

📦

Use Plural Nouns

/users not /user. /products/42 not /product/42. Consistency beats cleverness every time.

🔗

Nested Resources

/users/42/orders expresses "orders belonging to user 42". Keep nesting shallow — max 2 levels deep.

📋

Consistent Responses

Always return the same shape. Wrap everything: {"data":…, "meta":…, "errors":…}. Never surprise your clients.

⚡

Paginate Everything

Never return unbounded lists. Use ?page=1&per_page=20 or cursor-based pagination. Your database (and clients) will thank you.

Good vs Bad API Design 👇

❌ Anti-patterns to Avoid
Bad API Design
# Action verbs in URLs
GET /getUser?id=5
POST /createUser
GET /deleteUser?id=5

# Inconsistent naming
GET /Users       # PascalCase
GET /user_orders # snake_case
GET /productList # camelCase

# Returning 200 for errors
GET /users/999
→ 200 OK
  {"success": false, "msg": "not found"}
✅ Best Practices
Good API Design
# Resource-based URLs
GET    /api/v1/users/5
POST   /api/v1/users
DELETE /api/v1/users/5

# Consistent kebab-case
GET /api/v1/users
GET /api/v1/user-orders
GET /api/v1/product-categories

# Proper status codes
GET /api/v1/users/999
→ 404 Not Found
  {"error": "User not found", "code": "USER_NOT_FOUND"}

House Price Regressor — Train & Persist

Train once, serve forever. Always save the full pipeline (scaler + model together) so you can't accidentally skip preprocessing at inference time.

python — train_regression.py
import pandas as pd
import numpy as np
from sklearn.datasets import fetch_california_housing
from sklearn.ensemble import GradientBoostingRegressor
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
from sklearn.model_selection import train_test_split
from sklearn.metrics import mean_absolute_error, r2_score
import joblib, json

# ─── Load & split ─────────────────────────────────────
data = fetch_california_housing(as_frame=True)
X_train, X_test, y_train, y_test = train_test_split(
    data.data, data.target, test_size=0.2, random_state=42
)

# ─── Build pipeline (scaler + model) ──────────────────
# 🔑 Key insight: always bundle preprocessing into the pipeline.
# If you scale separately, you WILL forget to scale at inference.
pipeline = Pipeline([
    ("scaler", StandardScaler()),
    ("model",  GradientBoostingRegressor(
        n_estimators=200, max_depth=4,
        learning_rate=0.1, random_state=42
    ))
])

pipeline.fit(X_train, y_train)

# ─── Evaluate ─────────────────────────────────────────
preds = pipeline.predict(X_test)
print(f"MAE  : {mean_absolute_error(y_test, preds):.4f}")
print(f"R²   : {r2_score(y_test, preds):.4f}")

# ─── Save artefacts ───────────────────────────────────
joblib.dump(pipeline, "models/house_price_pipeline.joblib")

# Save feature names alongside the model — the API needs them
metadata = {
    "features": list(data.data.columns),
    "target_unit": "100k USD",
    "model_version": "1.0.0",
    "sklearn_version": __import__("sklearn").__version__,
}
with open("models/house_price_metadata.json", "w") as f:
    json.dump(metadata, f, indent=2)

print("✅ Pipeline saved to models/house_price_pipeline.joblib")

from fastapi import FastAPI, HTTPException
from pydantic import BaseModel, Field
from contextlib import asynccontextmanager
import joblib, numpy as np, json
from pathlib import Path

# ─── Lifespan: load model ONCE at startup ─────────────
# Never load inside the route — that's 200ms of disk I/O per request!
MODEL = {}

@asynccontextmanager
async def lifespan(app: FastAPI):
    MODEL["pipeline"] = joblib.load("models/house_price_pipeline.joblib")
    with open("models/house_price_metadata.json") as f:
        MODEL["meta"] = json.load(f)
    print("✅ Model loaded")
    yield          # app runs here
    MODEL.clear()  # cleanup on shutdown

app = FastAPI(
    title="House Price Prediction API",
    version="1.0.0",
    lifespan=lifespan
)

# ─── Schemas ──────────────────────────────────────────
class HouseFeatures(BaseModel):
    MedInc:     float = Field(..., gt=0,   example=5.5,   description="Median income in $10k")
    HouseAge:   float = Field(..., ge=0,   example=20.0,  description="Median house age (years)")
    AveRooms:   float = Field(..., gt=0,   example=5.2,   description="Average rooms per household")
    AveBedrms:  float = Field(..., gt=0,   example=1.1)
    Population: float = Field(..., gt=0,   example=800.0)
    AveOccup:   float = Field(..., gt=0,   example=3.0)
    Latitude:   float = Field(..., ge=32, le=42, example=37.5)
    Longitude:  float = Field(..., ge=-124, le=-114, example=-122.4)

class PredictionResponse(BaseModel):
    predicted_price_usd: float
    predicted_price_100k: float
    confidence_note: str
    model_version: str

# ─── Routes ───────────────────────────────────────────
@app.post("/predict",
    response_model=PredictionResponse,
    summary="Predict house price",
    tags=["Predictions"])
async def predict_price(features: HouseFeatures):
    pipeline = MODEL["pipeline"]
    meta     = MODEL["meta"]

    # Build input array in the EXACT column order the model was trained on
    X = np.array([[
        features.MedInc, features.HouseAge, features.AveRooms,
        features.AveBedrms, features.Population, features.AveOccup,
        features.Latitude, features.Longitude
    ]])

    raw = float(pipeline.predict(X)[0])
    return PredictionResponse(
        predicted_price_100k=round(raw, 4),
        predicted_price_usd=round(raw * 100_000, 2),
        confidence_note="GBR point estimate — no prediction interval",
        model_version=meta["model_version"],
    )

@app.post("/predict/batch", summary="Batch predictions", tags=["Predictions"])
async def predict_batch(rows: list[HouseFeatures]):
    if len(rows) > 500:
        raise HTTPException(400, "Batch limit is 500 rows")
    pipeline = MODEL["pipeline"]
    X = np.array([[
        r.MedInc, r.HouseAge, r.AveRooms, r.AveBedrms,
        r.Population, r.AveOccup, r.Latitude, r.Longitude
    ] for r in rows])
    preds = pipeline.predict(X)
    return {"predictions_100k": [round(float(p), 4) for p in preds]}

@app.get("/model/info", tags=["Model"])
async def model_info():
    return MODEL["meta"]

Request & Response Schema Design

Good schemas do three things: validate input, document the API automatically, and prevent model garbage-in/garbage-out errors.

python — rich schema patterns
from pydantic import BaseModel, Field, field_validator
from typing import Optional, List
from enum import Enum

# ─── Clean enums beat raw strings ─────────────────────
class Neighbourhood(str, Enum):
    urban    = "urban"
    suburban = "suburban"
    rural    = "rural"

class HouseInput(BaseModel):
    area_sqft:     float = Field(..., gt=0, le=20000, example=1400)
    bedrooms:      int   = Field(..., ge=1, le=20,    example=3)
    bathrooms:     float = Field(..., ge=0.5,         example=2.0)
    age_years:     int   = Field(..., ge=0, le=200)
    neighbourhood: Neighbourhood = Neighbourhood.suburban
    has_garage:    bool = False
    has_pool:      bool = False

    @field_validator("bathrooms")
    @classmethod
    def bathrooms_reasonable(cls, v, values):
        bedrooms = values.data.get("bedrooms", 1)
        if v > bedrooms * 2:
            raise ValueError("bathrooms can't greatly exceed bedrooms")
        return v

class PriceResponse(BaseModel):
    prediction:    float
    lower_bound:   Optional[float] = None  # if model supports intervals
    upper_bound:   Optional[float] = None
    currency:      str = "USD"
    model_version: str
    features_used: List[str]  # transparency: which features mattered

# ─── Batch schema ─────────────────────────────────────
class BatchRequest(BaseModel):
    houses:   List[HouseInput] = Field(..., max_length=500)
    include_intervals: bool = False

class BatchResponse(BaseModel):
    predictions: List[float]
    count: int
    model_version: str

💡 Pro tip: Use Enums for categorical features — they prevent typos, show up in the auto-docs dropdown, and let you detect training/serving skew at schema validation time rather than at model inference.

🎯 Sanity tests matter more than you think. Always add at least one domain-logic test (higher income → higher predicted price). It catches silent bugs where the model predicts correctly but returns the wrong field, or scaling was applied twice.

import pandas as pd

probs = pipeline.predict_proba(X_test)[:, 1]
print("AUC-ROC:", round(roc_auc_score(y_test, probs), 4))
print(classification_report(y_test, pipeline.predict(X_test)))

joblib.dump(pipeline.classes_, "models/churn_classes.joblib")

Classification API — Probabilities + Decision

python — churn_api.py
from fastapi import FastAPI
from pydantic import BaseModel, Field
from enum import Enum
from contextlib import asynccontextmanager
import joblib, numpy as np

class ContractType(str, Enum):
    month_to_month = "Month-to-month"
    one_year       = "One year"
    two_year       = "Two year"

class ChurnInput(BaseModel):
    tenure:          int   = Field(..., ge=0,  example=24, description="Months with company")
    monthly_charges: float = Field(..., gt=0,  example=65.50)
    total_charges:   float = Field(..., ge=0,  example=1571.0)
    num_products:    int   = Field(..., ge=1, le=8, example=2)
    contract_type:   ContractType = ContractType.month_to_month
    payment_method:  str  = Field(example="Electronic check")

class ChurnResponse(BaseModel):
    churn_probability: float    # P(churn) — always expose this!
    retain_probability: float
    prediction: str              # "churn" | "retain"
    risk_level: str              # "high" | "medium" | "low"
    threshold_used: float
    recommended_action: str

MODEL = {}
THRESHOLD = 0.45  # tuned on validation set — lower = more sensitive to churn

@asynccontextmanager
async def lifespan(app):
    MODEL["pipeline"] = joblib.load("models/churn_pipeline.joblib")
    yield

app = FastAPI(title="Churn Prediction API", lifespan=lifespan)

@app.post("/predict/churn", response_model=ChurnResponse)
async def predict_churn(customer: ChurnInput):
    X = [[
        customer.tenure, customer.monthly_charges,
        customer.total_charges, customer.num_products,
        customer.contract_type.value, customer.payment_method
    ]]
    probs = MODEL["pipeline"].predict_proba(X)[0]
    churn_prob = float(probs[1])  # index 1 = churn class
    will_churn = churn_prob >= THRESHOLD

    # Business logic layered on top of the model
    risk = "high" if churn_prob >= 0.7 else "medium" if churn_prob >= 0.45 else "low"
    actions = {
        "high":   "Trigger retention call — offer discount or upgrade",
        "medium": "Send personalised email with loyalty offer",
        "low":    "No action needed — customer is stable",
    }
    return ChurnResponse(
        churn_probability=round(churn_prob, 4),
        retain_probability=round(1 - churn_prob, 4),
        prediction="churn" if will_churn else "retain",
        risk_level=risk,
        threshold_used=THRESHOLD,
        recommended_action=actions[risk],
    )
📊
Always return probabilities

Never return just "churn: true/false". Return the raw probability (churn_probability: 0.73). Downstream systems can apply their own thresholds, build alerts based on risk bands, or display a confidence bar in the UI. A binary yes/no throws away valuable calibration information.

from fastapi import FastAPI
from pydantic import BaseModel
from typing import List, Dict
import joblib, numpy as np

app = FastAPI()

class TextInput(BaseModel):
    text: str
    top_k: int = 3  # return top 3 classes with probabilities

class ClassProbability(BaseModel):
    label: str
    probability: float

class MultiClassResponse(BaseModel):
    predicted_class: str
    confidence: float
    top_classes: List[ClassProbability]
    is_confident: bool  # True if confidence > 0.7

@app.post("/classify", response_model=MultiClassResponse)
async def classify(inp: TextInput):
    pipeline = MODEL["pipeline"]
    classes  = MODEL["classes"]   # e.g. ["electronics","books","clothing",...]

    probs = pipeline.predict_proba([inp.text])[0]
    top_idx = np.argsort(probs)[::-1][:inp.top_k]
    best_idx = int(np.argmax(probs))

    return MultiClassResponse(
        predicted_class=classes[best_idx],
        confidence=round(float(probs[best_idx]), 4),
        top_classes=[
            ClassProbability(label=classes[i], probability=round(float(probs[i]), 4))
            for i in top_idx
        ],
        is_confident=float(probs[best_idx]) >= 0.70,
    )

Threshold Tuning via API Parameter

python — dynamic threshold endpoint
from fastapi import FastAPI, Query
from pydantic import BaseModel, Field

app = FastAPI()

# Allow callers to pass their own threshold
# Great for A/B testing different decision boundaries
@app.post("/predict/churn")
async def predict_with_threshold(
    customer: ChurnInput,
    threshold: float = Query(0.45, ge=0.0, le=1.0, description="Decision threshold (default 0.45)"),
):
    probs = MODEL["pipeline"].predict_proba([customer.dict().values()])[0]
    churn_prob = float(probs[1])
    return {
        "churn_probability": round(churn_prob, 4),
        "prediction": "churn" if churn_prob >= threshold else "retain",
        "threshold_used": threshold,
    }

# Usage:
# POST /predict/churn?threshold=0.6  ← conservative (fewer false alarms)
# POST /predict/churn?threshold=0.3  ← aggressive (catch more churners)

# Why tune thresholds?
# Precision vs Recall tradeoff:
# - High threshold → fewer false positives (only call very-likely churners)
# - Low threshold  → fewer false negatives (catch more churners, more false alarms)
# Right answer depends on cost of a missed churn vs cost of an unwanted call.

Customer Segmentation with K-Means

python — train_clustering.py
import pandas as pd, numpy as np, joblib, json
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline

df = pd.read_csv("customers.csv")
FEATURES = ["recency_days", "frequency", "monetary_value"]
X = df[FEATURES]

pipeline = Pipeline([
    ("scaler", StandardScaler()),
    ("kmeans", KMeans(n_clusters=4, random_state=42, n_init="auto"))
])
pipeline.fit(X)

labels = pipeline.predict(X)
df["segment"] = labels

# ─── Human-readable segment names ─────────────────────
# Compute cluster centroids (in original scale) to name them
centroids_scaled = pipeline.named_steps["kmeans"].cluster_centers_
centroids = pipeline.named_steps["scaler"].inverse_transform(centroids_scaled)

# Rank by monetary value → name segments
ranking = centroids[:, 2].argsort()[::-1]
segment_names = {}
names = ["Champions", "Loyal Customers", "At Risk", "Lost"]
for rank, cluster_id in enumerate(ranking):
    segment_names[int(cluster_id)] = names[rank]

joblib.dump(pipeline, "models/segmentation_pipeline.joblib")
with open("models/segment_names.json", "w") as f:
    json.dump({"names": segment_names, "features": FEATURES}, f)
print("✅ Saved. Segments:", segment_names)

Cluster Assignment API

python — segmentation_api.py
from fastapi import FastAPI
from pydantic import BaseModel, Field
from contextlib import asynccontextmanager
from typing import List
import joblib, numpy as np, json

MODEL = {}

@asynccontextmanager
async def lifespan(app):
    MODEL["pipeline"] = joblib.load("models/segmentation_pipeline.joblib")
    with open("models/segment_names.json") as f:
        meta = json.load(f)
    MODEL["names"] = {int(k): v for k, v in meta["names"].items()}
    yield

app = FastAPI(title="Customer Segmentation API", lifespan=lifespan)

class CustomerRFM(BaseModel):
    recency_days:    int   = Field(..., ge=0,  example=14,     description="Days since last purchase")
    frequency:       int   = Field(..., ge=1,  example=8,      description="Total purchases in 12 months")
    monetary_value:  float = Field(..., ge=0,  example=340.5,  description="Total spend in USD")

class SegmentResponse(BaseModel):
    cluster_id:   int
    segment_name: str
    description:  str
    distances_to_all_clusters: List[float]  # transparency

DESCRIPTIONS = {
    "Champions":       "Bought recently, buys often, high spender. Reward them!",
    "Loyal Customers": "Regular buyers, moderate spend. Upsell opportunity.",
    "At Risk":         "Was a good customer, hasn't purchased recently. Win-back campaign!",
    "Lost":            "Low recency, frequency, and spend. Low-cost reactivation attempt only.",
}

@app.post("/segment", response_model=SegmentResponse)
async def segment_customer(customer: CustomerRFM):
    pipeline  = MODEL["pipeline"]
    X = np.array([[customer.recency_days, customer.frequency, customer.monetary_value]])

    cluster_id = int(pipeline.predict(X)[0])

    # Distance to each cluster centre (lower = closer match)
    X_scaled = pipeline.named_steps["scaler"].transform(X)
    kmeans   = pipeline.named_steps["kmeans"]
    distances = [round(float(d), 4) for d in kmeans.transform(X_scaled)[0]]

    name = MODEL["names"][cluster_id]
    return SegmentResponse(
        cluster_id=cluster_id,
        segment_name=name,
        description=DESCRIPTIONS[name],
        distances_to_all_clusters=distances,
    )

@app.get("/segments")
async def list_segments():
    # Useful for UI dropdowns and documentation
    return {k: {"name": v, "description": DESCRIPTIONS[v]}
            for k, v in MODEL["names"].items()}

app = FastAPI(title="Transaction Anomaly Detection")

class Transaction(BaseModel):
    amount:         float = Field(..., gt=0, example=8500.00)
    hour_of_day:    int   = Field(..., ge=0, le=23)
    day_of_week:    int   = Field(..., ge=0, le=6)
    merchant_cat:   int   = Field(..., ge=0)
    distance_km:    float = Field(..., ge=0)

class AnomalyResult(BaseModel):
    is_anomaly:   bool
    anomaly_score: float  # more negative = more anomalous
    action:       str

@app.post("/detect", response_model=AnomalyResult)
async def detect_anomaly(txn: Transaction):
    X = np.array([[txn.amount, txn.hour_of_day, txn.day_of_week,
                    txn.merchant_cat, txn.distance_km]])
    pipeline  = MODEL["pipeline"]
    score = float(pipeline.named_steps["iso"].score_samples(
        pipeline.named_steps["scaler"].transform(X)
    )[0])
    is_anomaly = bool(pipeline.predict(X)[0] == -1)
    return AnomalyResult(
        is_anomaly=is_anomaly,
        anomaly_score=round(score, 4),
        action="Block and alert fraud team" if is_anomaly else "Approve transaction",
    )

from typing import List
import torch

MODEL = {}

@asynccontextmanager
async def lifespan(app):

class TextRequest(BaseModel):
    text: str = Field(..., min_length=1, max_length=512, example="I absolutely love this product!")

class BatchTextRequest(BaseModel):
    texts: List[str] = Field(..., max_length=100)

class SentimentResult(BaseModel):
    label: str       # "POSITIVE" | "NEGATIVE"
    score: float    # confidence 0.0 → 1.0
    text_preview: str

@app.post("/sentiment", response_model=SentimentResult)
async def analyse_sentiment(req: TextRequest):
    result = MODEL["pipe"](req.text)[0]
    return SentimentResult(
        label=result["label"],
        score=round(result["score"], 4),
        text_preview=req.text[:80] + ("..." if len(req.text) > 80 else ""),
    )

@app.post("/sentiment/batch", response_model=List[SentimentResult])
async def analyse_batch(req: BatchTextRequest):
    if any(len(t) > 512 for t in req.texts):
        raise HTTPException(400, "All texts must be ≤ 512 characters")
    results = MODEL["pipe"](req.texts)
    return [
        SentimentResult(
            label=r["label"], score=round(r["score"], 4),
            text_preview=t[:80]
        )
        for r, t in zip(results, req.texts)
    ]

from fastapi import FastAPI
from pydantic import BaseModel, Field
import numpy as np

app = FastAPI()

class ClassifyRequest(BaseModel):
    text: str = Field(..., min_length=10, example="Subject: Claim your prize now!")
    explain: bool = False  # if True, return top contributing words

@app.post("/classify/text")
async def classify_text(req: ClassifyRequest):
    pipeline = MODEL["pipeline"]
    classes  = pipeline.classes_
    probs    = pipeline.predict_proba([req.text])[0]
    best     = int(np.argmax(probs))
    resp     = {
        "label":       classes[best],
        "confidence":  round(float(probs[best]), 4),
        "all_classes": {c: round(float(p), 4) for c, p in zip(classes, probs)},
    }
    if req.explain:
        # Top token contributions (works for LR + TF-IDF)
        tfidf   = pipeline.named_steps["tfidf"]
        clf     = pipeline.named_steps["clf"]
        vec     = tfidf.transform([req.text])
        coef    = clf.coef_[best] if len(classes) > 2 else clf.coef_[0]
        scores  = (vec * coef).toarray()[0]
        top_idx = np.argsort(scores)[::-1][:10]
        vocab   = tfidf.get_feature_names_out()
        resp["top_tokens"] = [
            {"token": vocab[i], "weight": round(float(scores[i]), 4)}
            for i in top_idx if scores[i] > 0
        ]
    return resp

from fastapi import FastAPI
from fastapi.responses import StreamingResponse
from pydantic import BaseModel
from transformers import AutoModelForCausalLM, AutoTokenizer, TextIteratorStreamer
from threading import Thread
import json

app = FastAPI()

class GenerateRequest(BaseModel):
    prompt:      str
    max_tokens:  int = 256
    temperature: float = 0.7

# Token-by-token streaming — same UX as ChatGPT
@app.post("/generate")
async def generate(req: GenerateRequest):
    tokenizer = MODEL["tokenizer"]
    model     = MODEL["model"]
    streamer  = TextIteratorStreamer(tokenizer, skip_special_tokens=True)
    inputs    = tokenizer(req.prompt, return_tensors="pt")

    def generate_worker():
        model.generate(
            **inputs, max_new_tokens=req.max_tokens,
            temperature=req.temperature, streamer=streamer
        )

    Thread(target=generate_worker, daemon=True).start()

    def token_generator():
        for token in streamer:
            # Server-Sent Events format (compatible with EventSource)
            yield f"data: {json.dumps({'token': token})}\n\n"
        yield "data: [DONE]\n\n"

    return StreamingResponse(
        token_generator(),
        media_type="text/event-stream"
    )

    label: str
    probability: float

async def classify_image(
    file: UploadFile = File(...),
    top_k: int = 5
):
    if file.content_type not in {"image/jpeg", "image/png", "image/webp"}:
        raise HTTPException(400, "Supported formats: JPEG, PNG, WebP")

    content = await file.read()
    image   = Image.open(io.BytesIO(content)).convert("RGB")
    tensor  = MODEL["transform"](image).unsqueeze(0)  # add batch dim

    with torch.no_grad():

        probs  = torch.softmax(logits, dim=1)[0]

    classes = MODEL["classes"]
    return [
        Prediction(label=classes[i], probability=round(float(p), 4))
        for i, p in zip(top_idx.tolist(), top_probs.tolist())
    ]

Time Series Forecasting API

python — forecasting_api.py
from fastapi import FastAPI
from pydantic import BaseModel, Field
from typing import List
import numpy as np

app = FastAPI()

class ForecastRequest(BaseModel):
    # Last N observations (the model's lookback window)
    history:  List[float] = Field(..., min_length=30, max_length=365)
    horizon:  int = Field(7, ge=1, le=90, description="Days to forecast")

class ForecastResponse(BaseModel):
    forecast:        List[float]
    forecast_dates:  List[str]
    confidence_80_lower: List[float]
    confidence_80_upper: List[float]

@app.post("/forecast", response_model=ForecastResponse)
async def forecast(req: ForecastRequest):
    model = MODEL["model"]
    scaler = MODEL["scaler"]

    # Normalize, reshape for LSTM: (batch=1, seq_len, features=1)
    arr    = np.array(req.history).reshape(-1, 1)
    scaled = scaler.transform(arr).reshape(1, -1, 1)

    preds_scaled = model.predict(scaled, horizon=req.horizon)
    preds = scaler.inverse_transform(preds_scaled.reshape(-1, 1)).flatten()

    # Simple ±10% prediction interval (replace with proper conformal intervals)
    std = np.std(req.history[-30:]) * 1.28
    from datetime import date, timedelta
    today = date.today()
    dates = [(today + timedelta(days=i+1)).isoformat() for i in range(req.horizon)]

    return ForecastResponse(
        forecast=[round(float(p), 2) for p in preds],
        forecast_dates=dates,
        confidence_80_lower=[round(float(p - std), 2) for p in preds],
        confidence_80_upper=[round(float(p + std), 2) for p in preds],
    )

⚠️ Never load models inside route handlers. Each request would deserialise a 200MB file from disk — 300-500ms latency hit every time. Load once at startup using lifespan, store in a module-level dict.

⚠️ Always save the full pipeline (scaler + encoder + model together). If you save only the model, you'll forget to apply the same preprocessing at inference. The scaler's mean_ and scale_ won't survive a server restart.

✅ Return probabilities, not just labels. "churn_probability": 0.73 lets callers tune their own threshold. A hard label throws away calibration information every downstream system could use.

🎯 Add a /model/info endpoint that returns model version, training date, feature list, and performance metrics. Ops teams will thank you during incidents.

# ─── 2. Structured logging middleware ─────────────────
@app.middleware("http")
async def log_requests(request, call_next):
    request_id = str(uuid.uuid4())
    start = time.perf_counter()
    response = await call_next(request)
    elapsed = (time.perf_counter() - start) * 1000
    logger.info({
        "request_id": request_id,
        "path": request.url.path,
        "latency_ms": round(elapsed, 1),
        "status": response.status_code,
    })
    return response

API Security — Don't skip this

Security isn't optional. Here are the must-knows before you go to production.

🔐 Authentication Methods

1

API Keys

Simple. Pass in header: X-API-Key: abc123. Good for server-to-server. No expiry by default.

2

JWT (Bearer Tokens)

Authorization: Bearer eyJ.... Self-contained, stateless, expirable. Industry standard for user auth.

3

OAuth 2.0

Delegated access ("Login with Google"). Complex but necessary for third-party integrations.

4

mTLS

Mutual TLS — both client AND server verify certificates. Gold standard for microservices security.
