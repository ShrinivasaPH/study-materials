Chapter 1

What exactly is an API?

Before writing a single line of code, let's build rock-solid mental models.

An API (Application Programming Interface) is a contract between two pieces of software. It defines how they talk to each other — what you can ask for, how to ask, and what you'll get back.

Think of a restaurant 🍽️. You (the client) don't go into the kitchen and cook your own food. Instead, you talk to the waiter (the API) who takes your order, passes it to the kitchen (the server), and brings back exactly what you asked for.

A REST API is a specific style of API that runs over HTTP — the same protocol your browser uses. It's stateless, meaning every request is self-contained and carries all the info needed.

🤯
Fun Fact

Roy Fielding coined the term "REST" in his 2000 PhD dissertation. He was also one of the principal authors of the HTTP spec. Talk about one productive thesis!

🌐

REST API

Uses HTTP verbs (GET, POST…) and URLs. Stateless. Returns JSON/XML. The most popular style today.

⚡

GraphQL API

Query exactly what you need, nothing more. Single endpoint. Developed by Facebook in 2012.

🔌

WebSocket API

Persistent two-way connection. Great for real-time apps: chat, live scores, stock tickers.

💡 REST vs HTTP: HTTP is the transport protocol. REST is a set of architectural constraints that use HTTP. All REST APIs use HTTP, but not all HTTP APIs are RESTful!

Chapter 2

The 5 HTTP Methods you must know

These verbs tell the server what you want to do. Click each one to explore.

GET

Read / Retrieve

Fetch data. Safe & idempotent. Never changes server state.

POST

Create

Submit new data. Not idempotent. Response: 201 Created.

PUT

Full Replace

Fully replace a resource. Idempotent. Must send all fields.

PATCH

Partial Update

Update only specific fields. More efficient than PUT.

DELETE

Remove

Delete a resource. Idempotent. Returns 204 or 200.

💡
Key Concept: Idempotency

An operation is idempotent if doing it multiple times gives the same result as doing it once. DELETE /users/5 ten times still results in user 5 being gone — the first call does the work, the rest are no-ops.

PUT vs PATCH — What's the Difference?

🔄 PUT — Full Replace
PUT /users/5
// You MUST send all fields
{
  "name": "Shrinivasa",
  "email": "new@email.com",
  "role": "admin",
  "age": 30
}
// Missing fields get nulled/defaulted!
🎯 PATCH — Partial Update
PATCH /users/5
// Send ONLY what changed
{
  "email": "new@email.com"
}
// Other fields unchanged ✅

HEAD & OPTIONS — the underrated two

🔭

HEAD

Like GET, but the server returns headers only — no body. Use it to check if a resource exists, get its Content-Length before downloading, or validate caching headers without wasting bandwidth.

🤝

OPTIONS

Ask the server "what methods do you support here?" Browsers send this automatically as a CORS preflight before cross-origin requests. Returns an Allow header listing valid methods.

HEAD & OPTIONS examples
# HEAD — check a file exists and get its size before downloading
HEAD /api/v1/exports/report-2025.pdf HTTP/1.1

# Response (no body)
HTTP/1.1 200 OK
Content-Length: 52428800
Content-Type: application/pdf
Last-Modified: Thu, 29 May 2025 09:00:00 GMT

# OPTIONS — discover allowed methods (browsers use for CORS)
OPTIONS /api/v1/products HTTP/1.1
Origin: https://myapp.com

# Response
HTTP/1.1 200 OK
Allow: GET, POST, HEAD, OPTIONS
Access-Control-Allow-Methods: GET, POST, PUT, PATCH, DELETE
Access-Control-Allow-Origin: https://myapp.com

Full CRUD with FastAPI

CRUD = Create, Read, Update, Delete. The four fundamental operations of any data-driven API.

python — Full CRUD
from fastapi import FastAPI, HTTPException, status
from pydantic import BaseModel, Field
from typing import Optional, List
from datetime import datetime

app = FastAPI()

class ProductCreate(BaseModel):
    name: str = Field(..., min_length=2, max_length=100)
    price: float = Field(..., gt=0)
    category: str
    in_stock: bool = True

class Product(ProductCreate):
    id: int
    created_at: str

products_db: dict[int, Product] = {}
_id = 0

# ─── CREATE ───────────────────────────────
@app.post("/products", response_model=Product,
         status_code=status.HTTP_201_CREATED)
async def create_product(product: ProductCreate):
    global _id
    _id += 1
    new_product = Product(
        id=_id,
        created_at=datetime.now().isoformat(),
        **product.dict()
    )
    products_db[_id] = new_product
    return new_product

# ─── READ ALL ─────────────────────────────
@app.get("/products", response_model=List[Product])
async def list_products(
    skip: int = 0,
    limit: int = Query(default=10, le=100),
    category: Optional[str] = None
):
    items = list(products_db.values())
    if category:
        items = [p for p in items if p.category == category]
    return items[skip : skip + limit]

# ─── READ ONE ─────────────────────────────
@app.get("/products/{product_id}", response_model=Product)
async def get_product(product_id: int):
    if product_id not in products_db:
        raise HTTPException(
            status_code=404,
            detail=f"Product {product_id} not found"
        )
    return products_db[product_id]

# ─── UPDATE (PATCH) ───────────────────────
@app.patch("/products/{product_id}", response_model=Product)
async def update_product(product_id: int, updates: dict):
    if product_id not in products_db:
        raise HTTPException(status_code=404, detail="Not found")
    product = products_db[product_id]
    updated = product.dict().update(updates)
    products_db[product_id] = Product(**updated)
    return products_db[product_id]

# ─── DELETE ───────────────────────────────
@app.delete("/products/{product_id}",
           status_code=status.HTTP_204_NO_CONTENT)
async def delete_product(product_id: int):
    if product_id not in products_db:
        raise HTTPException(status_code=404, detail="Not found")
    del products_db[product_id]

Data Validation with Pydantic

Pydantic is FastAPI's superpower. Define your data models once and get validation, serialization, and documentation for free.

python — Pydantic Models
from pydantic import BaseModel, Field, validator, EmailStr
from typing import Optional, List
from enum import Enum

class Role(str, Enum):
    admin  = "admin"
    user   = "user"
    viewer = "viewer"

class UserCreate(BaseModel):
    username: str = Field(
        ...,
        min_length=3,
        max_length=50,
        pattern=r"^[a-zA-Z0-9_]+$",
        example="shrinivasa_ph"
    )
    email: EmailStr
    age: Optional[int] = Field(None, ge=13, le=120)
    role: Role = Role.user
    tags: List[str] = []

    @validator("username")
    def username_not_reserved(cls, v):
        reserved = ["admin", "root", "system"]
        if v.lower() in reserved:
            raise ValueError("Username is reserved")
        return v

    class Config:
        schema_extra = {
            "example": {
                "username": "john_doe",
                "email": "john@example.com",
                "age": 28,
                "role": "user"
            }
        }

# FastAPI automatically returns 422 Unprocessable Entity
# with detailed error messages when validation fails!
🧙
Pydantic Magic

Field validators like ge (greater than or equal), le (less than or equal), min_length, max_length, and pattern are checked automatically before your route function even runs. No manual if-else validation needed!

JWT Authentication

JWT (JSON Web Tokens) are the industry standard for stateless API authentication. The server issues a token, and the client sends it with every request.

python — JWT Auth
from fastapi import FastAPI, Depends, HTTPException, status
from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm
from jose import JWTError, jwt
from passlib.context import CryptContext
from datetime import datetime, timedelta
from pydantic import BaseModel

# pip install python-jose passlib[bcrypt]

SECRET_KEY = "your-secret-key-keep-it-safe!"
ALGORITHM  = "HS256"
TOKEN_EXPIRE_MINUTES = 30

pwd_context = CryptContext(schemes=["bcrypt"])
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")
app = FastAPI()

def create_token(data: dict) -> str:
    payload = data.copy()
    payload["exp"] = datetime.utcnow() + timedelta(minutes=TOKEN_EXPIRE_MINUTES)
    return jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)

async def get_current_user(token: str = Depends(oauth2_scheme)):
    credentials_exception = HTTPException(
        status_code=status.HTTP_401_UNAUTHORIZED,
        detail="Invalid credentials",
        headers={"WWW-Authenticate": "Bearer"}
    )
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        username: str = payload.get("sub")
        if not username:
            raise credentials_exception
    except JWTError:
        raise credentials_exception
    return username

# 🔐 Protected route — requires valid JWT
@app.get("/me")
async def read_me(user: str = Depends(get_current_user)):
    return {"username": user, "msg": "Authenticated!"}

🔑 How JWT Works: Header.Payload.Signature — all base64 encoded. The signature is created using your SECRET_KEY, so only your server can verify it. Never store sensitive data in the payload — it's not encrypted, just encoded!

Async FastAPI — Handle 10,000 requests/sec

FastAPI is built on asyncio. Use async/await for I/O-bound operations (DB queries, external APIs) to handle massive concurrency.

python — Async Routes
import asyncio, httpx
from fastapi import FastAPI
from databases import Database

app = FastAPI()
db = Database("postgresql://user:pass@localhost/mydb")

@app.on_event("startup")
async def startup():
    await db.connect()

# ✅ Async DB query — non-blocking
@app.get("/users")
async def get_users():
    query = "SELECT * FROM users LIMIT 50"
    return await db.fetch_all(query)

# ✅ Fan-out: call 3 external APIs concurrently
@app.get("/dashboard/{user_id}")
async def get_dashboard(user_id: int):
    async with httpx.AsyncClient() as client:
        profile, orders, notifications = await asyncio.gather(
            client.get(f"/profile/{user_id}"),
            client.get(f"/orders/{user_id}"),
            client.get(f"/notifications/{user_id}")
        )
    return {
        "profile": profile.json(),
        "orders": orders.json(),
        "notifications": notifications.json()
    }
# Three API calls in parallel, not serial!
# 3 × 100ms = 100ms, not 300ms 🔥

@app.route("/users/<int:user_id>", methods=["GET", "DELETE"])
def user(user_id):
    if request.method == "GET":
        return jsonify({"id": user_id})
    elif request.method == "DELETE":
        return "", 204  # No content

# POST with JSON body
@app.route("/users", methods=["POST"])
def create_user():
    data = request.get_json()
    if not data or "name" not in data:
        return jsonify({"error": "name is required"}), 400
    return jsonify({"id": 1, **data}), 201

# Error handlers
@app.errorhandler(404)
def not_found(e):
    return jsonify({"error": "not found"}), 404

Flask CRUD with SQLAlchemy

python — Flask + SQLAlchemy CRUD
from flask import Flask, request, jsonify
from flask_sqlalchemy import SQLAlchemy

app = Flask(__name__)
app.config["SQLALCHEMY_DATABASE_URI"] = "sqlite:///products.db"
db = SQLAlchemy(app)

class Product(db.Model):
    id       = db.Column(db.Integer, primary_key=True)
    name     = db.Column(db.String(100), nullable=False)
    price    = db.Column(db.Float, nullable=False)
    in_stock = db.Column(db.Boolean, default=True)

    def to_dict(self):
        return {
            "id": self.id, "name": self.name,
            "price": self.price, "in_stock": self.in_stock
        }

@app.route("/products", methods=["GET"])
def list_products():
    page = request.args.get("page", 1, type=int)
    per_page = request.args.get("per_page", 10, type=int)
    paginated = Product.query.paginate(page=page, per_page=per_page)
    return jsonify({
        "items": [p.to_dict() for p in paginated.items],
        "total": paginated.total,
        "pages": paginated.pages
    })

@app.route("/products", methods=["POST"])
def create_product():
    data = request.get_json()
    product = Product(**data)
    db.session.add(product)
    db.session.commit()
    return jsonify(product.to_dict()), 201

Middleware & Decorators

python — Middleware
from flask import Flask, request, g
import time, logging

app = Flask(__name__)

# Before every request
@app.before_request
def start_timer():
    g.start = time.time()

# After every request
@app.after_request
def log_request(response):
    elapsed = time.time() - g.start
    logging.info(f"{request.method} {request.path} → {response.status_code} ({elapsed:.3f}s)")
    response.headers["X-Response-Time"] = f"{elapsed:.3f}s"
    return response

# Custom auth decorator
from functools import wraps

def require_api_key(f):
    @wraps(f)
    def decorated(*args, **kwargs):
        key = request.headers.get("X-API-Key")
        if key != "secret-key-123":
            return {"error": "Invalid API key"}, 401
        return f(*args, **kwargs)
    return decorated

@app.route("/secret")
@require_api_key
def secret_endpoint():
    return {"data": "🔐 Super secret!"}

DRF Serializers — Your Data's Bodyguard

python — serializers.py
from rest_framework import serializers
from .models import Product, Category

class CategorySerializer(serializers.ModelSerializer):
    class Meta:
        model = Category
        fields = ["id", "name"]

class ProductSerializer(serializers.ModelSerializer):
    # Nested serializer — includes full category object
    category = CategorySerializer(read_only=True)
    category_id = serializers.IntegerField(write_only=True)
    
    # Computed field
    full_name = serializers.SerializerMethodField()

    def get_full_name(self, obj):
        return f"{obj.category.name} — {obj.name}"

    def validate_price(self, value):
        if value <= 0:
            raise serializers.ValidationError("Price must be positive")
        return value

    class Meta:
        model = Product
        fields = ["id", "name", "price", "category", "category_id", "full_name"]
        read_only_fields = ["id", "full_name"]

ViewSets — Maximum Power, Minimum Code

python — views.py
from rest_framework import viewsets, filters
from rest_framework.decorators import action
from rest_framework.response import Response
from django_filters.rest_framework import DjangoFilterBackend

class ProductViewSet(viewsets.ModelViewSet):
    """
    Automatically provides:
    GET    /products/         → list()
    POST   /products/         → create()
    GET    /products/{id}/    → retrieve()
    PUT    /products/{id}/    → update()
    PATCH  /products/{id}/    → partial_update()
    DELETE /products/{id}/    → destroy()
    """
    queryset = Product.objects.all()
    serializer_class = ProductSerializer
    filter_backends = [DjangoFilterBackend, filters.SearchFilter]
    filterset_fields = ["category", "in_stock"]
    search_fields = ["name", "description"]

    # Custom action: POST /products/{id}/toggle_stock/
    @action(detail=True, methods=["post"])
    def toggle_stock(self, request, pk=None):
        product = self.get_object()
        product.in_stock = not product.in_stock
        product.save()
        return Response({"in_stock": product.in_stock})

from rest_framework.permissions import BasePermission

class IsOwnerOrReadOnly(BasePermission):
    """Allow owners to edit; everyone else read-only."""
    def has_object_permission(self, request, view, obj):
        if request.method in ("GET", "HEAD", "OPTIONS"):
            return True
        return obj.owner == request.user

class IsAdminOrReadOnly(BasePermission):
    def has_permission(self, request, view):
        if request.method in ("GET", "HEAD", "OPTIONS"):
            return True
        return request.user.is_staff

API Design Best Practices

Good APIs are intuitive, consistent, and pleasant to use. Follow these principles.

📐

Use Nouns for Endpoints

Resources should be nouns: /users, /products. Not /getUsers or /createProduct — that's what HTTP methods are for.

🔢

Version Your API

Always version: /api/v1/users. When you break things (and you will), v2 lets old clients keep working while new ones upgrade.

📦

Use Plural Nouns

/users not /user. /products/42 not /product/42. Consistency beats cleverness every time.

🔗

Nested Resources

/users/42/orders expresses "orders belonging to user 42". Keep nesting shallow — max 2 levels deep.

📋

Consistent Responses

Always return the same shape. Wrap everything: {"data":…, "meta":…, "errors":…}. Never surprise your clients.

⚡

Paginate Everything

Never return unbounded lists. Use ?page=1&per_page=20 or cursor-based pagination. Your database (and clients) will thank you.

Good vs Bad API Design 👇

❌ Anti-patterns to Avoid
Bad API Design
# Action verbs in URLs
GET /getUser?id=5
POST /createUser
GET /deleteUser?id=5

# Inconsistent naming
GET /Users       # PascalCase
GET /user_orders # snake_case
GET /productList # camelCase

# Returning 200 for errors
GET /users/999
→ 200 OK
  {"success": false, "msg": "not found"}
✅ Best Practices
Good API Design
# Resource-based URLs
GET    /api/v1/users/5
POST   /api/v1/users
DELETE /api/v1/users/5

# Consistent kebab-case
GET /api/v1/users
GET /api/v1/user-orders
GET /api/v1/product-categories

# Proper status codes
GET /api/v1/users/999
→ 404 Not Found
  {"error": "User not found", "code": "USER_NOT_FOUND"}

API Security — Don't skip this

Security isn't optional. Here are the must-knows before you go to production.

🔐 Authentication Methods

1

API Keys

Simple. Pass in header: X-API-Key: abc123. Good for server-to-server. No expiry by default.

2

JWT (Bearer Tokens)

Authorization: Bearer eyJ.... Self-contained, stateless, expirable. Industry standard for user auth.

3

OAuth 2.0

Delegated access ("Login with Google"). Complex but necessary for third-party integrations.

python — FastAPI Security Headers
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from slowapi import Limiter
from slowapi.util import get_remote_address

app = FastAPI()
limiter = Limiter(key_func=get_remote_address)

# ✅ CORS — control who can call your API
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://myapp.com"],  # NOT "*"!
    allow_methods=["GET", "POST"],
    allow_headers=["Authorization"],
)

# ✅ Rate limiting — prevent abuse
@app.get("/login")
@limiter.limit("5/minute")
async def login():
    ...

# ✅ Always validate input (Pydantic)
# ✅ Never store plain-text passwords (use bcrypt)
# ✅ Use HTTPS everywhere
# ✅ Rotate secrets regularly
# ✅ Log suspicious activity

⚠️ Never log authorization headers, passwords, or tokens. One log leak can compromise your entire system.
