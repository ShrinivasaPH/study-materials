02
The Sequence-to-Sequence Problem

Before Transformers, sequence-to-sequence models built on RNNs and LSTMs were the gold standard for translation, summarization, and question answering. They worked — but they carried a structural flaw that set a hard ceiling on their capabilities, particularly for long or complex inputs.

In the classic Seq2Seq (Sequence-to-Sequence) architecture introduced by Sutskever et al. (2014), two recurrent networks are chained together. The encoder reads the entire source sentence — say, a French sentence to be translated into English — processing one token at a time and updating an internal hidden state \(h_t\) at each step. When it reaches the final token, that last hidden state becomes the context vector \(c\). This single vector — typically 512 or 1024 floating-point numbers — is the only information the decoder ever receives about the input. The decoder then uses \(c\) to initialize its own hidden state and begins generating the target-language tokens, one at a time, from start to end.

The problem is stark: a fixed-size vector must encode the full meaning, syntax, and word order of an arbitrarily long sentence. For short sentences of 5–10 words this works tolerably well. But for a 40-word sentence describing a complex legal condition, or a 100-token paragraph in a scientific paper, the model is forced into an impossible compression. Information from early tokens is systematically overwritten as the RNN processes later tokens, because the hidden state is updated in-place at every step. By the time the encoder finishes reading, the beginning of the sentence has been diluted or effectively forgotten.

The Fixed-Size Bottleneck A vector of 512 floats is asked to losslessly represent a sentence of 50 tokens, each of which carries its own semantic content, grammatical role, and contextual meaning. This is not a training problem that more data or a better optimizer can solve — it is a fundamental architectural limitation. Bahdanau et al. (2015) empirically showed that BLEU (translation quality) scores drop sharply for sentences longer than 30 tokens with this architecture, precisely because the context vector saturates.
⏳

Sequential Processing — No Parallelism

RNNs compute the hidden state at step \(t\) as a function of the previous hidden state \(h_{t-1}\) and the current input \(x_t\): the recurrence \(h_t = f(h_{t-1}, x_t)\) is inherently sequential. You cannot begin computing \(h_5\) until \(h_4\) is ready, and \(h_4\) until \(h_3\), and so on. Modern GPUs are optimized for massively parallel matrix operations — thousands of tensor cores that sit idle while the RNN serializes its computation. In practice, Transformers train 10–30× faster wall-clock on the same hardware because every token can be processed simultaneously in a single large batched matrix multiply.

🔍

Long-Range Dependencies — Gradient Pathways

Consider the sentence: "The keys that were found in the old cabinet by the door of the house were rusty." The subject "keys" and its predicate "were rusty" are separated by 12 tokens. For a gradient to connect them during backpropagation, it must flow backward through 12 recurrent steps, multiplying by the recurrent weight matrix at every step. Each multiplication either shrinks the gradient toward zero (vanishing) or inflates it exponentially (exploding). LSTMs add gating mechanisms to partially address this, but the path length is still O(n) — which is why even LSTMs struggle on sequences exceeding ~30–50 tokens for subtle dependencies.

📦

Fixed Context Vector — Lossy Compression

The context vector is a fixed-dimensional representation regardless of input length. When the input is a short sentence, this is fine — there is enough capacity. But when the input is a long paragraph, the model must decide what to "keep" and what to "discard" as it updates its hidden state through each token. Early tokens consistently suffer because the RNN must balance retaining early information against encoding later tokens. In effect, the decoder is asked to reconstruct a 50-word sentence from a 512-dimensional snapshot — like reconstructing a photograph from a thumbnail.

The Attention Breakthrough — Bahdanau et al., 2015 Dzmitry Bahdanau, KyungHyun Cho, and Yoshua Bengio proposed a remarkably clean solution: don't discard the intermediate encoder hidden states — keep all of them. Then give the decoder a learnable mechanism to selectively retrieve information from this full set of states at each decoding step. Specifically, at each decoder timestep, a small alignment network scores each encoder hidden state against the current decoder state, producing a probability distribution over encoder positions via softmax. The decoder then uses a weighted sum of all encoder hidden states — weighted by these alignment scores — as its context for that step. This meant the decoder could directly attend to the relevant part of the input sentence when translating each word, regardless of distance. The Transformer paper then radicalized this idea further: what if we abandon recurrence entirely and build a model whose architecture is entirely composed of attention operations, trained in fully parallel fashion?
03
The Attention Mechanism

Attention is the mechanism by which a neural network learns to selectively focus on different parts of its input, dynamically and in a context-dependent way, rather than treating all positions equally. It is the central innovation of the Transformer and the reason modern language models work as well as they do.

The fundamental operation is a weighted aggregation. Instead of picking a single relevant position (hard attention) or averaging all positions equally, the model computes a weighted sum of all available representations — where the weights are learned dynamically based on content similarity. A token that is highly relevant to the current query contributes heavily to the output; an irrelevant token contributes barely at all. Crucially, these weights are not fixed — they are recomputed at every layer and for every input, meaning the model's "routing" of information is completely input-dependent.

This is a profound shift from earlier paradigms. In a fully connected layer, every hidden unit is connected to every input with a fixed weight learned during training. In attention, the effective connections are determined at inference time by the content of the current input. A word like "bank" next to "river" will route attention very differently than "bank" next to "loan" — and the model learns to implement this disambiguation automatically, without explicit programming.

General Attention
\[ \text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V \]
Q
Query Matrix — Shape \(n \times d_k\). This represents what each of the n query tokens is "asking for." Every row \(q_i\) is a vector encoding the information needs of token \(i\). It is produced by a learned linear projection: \(Q = X W^Q\), where \(X \in \mathbb{R}^{n \times d_{model}}\) is the input and \(W^Q \in \mathbb{R}^{d_{model} \times d_k}\) is a learnable weight matrix. Because this is a projection, the model learns to extract only the aspects of each token's representation that are useful for querying.
K
Key Matrix — Shape \(m \times d_k\). This represents what each of the m candidate positions "advertises" about itself for matching purposes. Each row \(k_j\) is the key of token \(j\), also produced by a distinct learned projection: \(K = X W^K\). The dot product \(q_i \cdot k_j\) measures the compatibility or relevance between token \(i\)'s query and token \(j\)'s key. Because \(W^Q\) and \(W^K\) are separate matrices learned independently, the query space and key space are allowed to differ — meaning what a token "looks for" and what it "presents" for matching can be entirely different projections of its underlying representation.
V
Value Matrix — Shape \(m \times d_v\). This represents the actual payload that each position contributes to the output. The keys determine whether a position is attended to; the values determine what information is retrieved when it is. Produced by \(V = X W^V\). The output of attention is a weighted sum of rows of \(V\), so the value space determines the content of the resulting representation. Separating Keys from Values is what makes attention so flexible: a token can have a high-salience key (making it easy to find) while carrying a nuanced, multi-dimensional value representation.
d_k
Key / Query Dimension — The number of dimensions in the query and key vectors. In the original Transformer: \(d_k = d_{model}/h = 512/8 = 64\). This dimension controls the capacity of the matching operation — larger \(d_k\) means richer compatibility signals, but also more parameters and greater risk of large-magnitude dot products, which is why the scaling factor \(1/\sqrt{d_k}\) is critical. Keys and Queries always share the same dimension because they must be dot-producted together.
d_v
Value Dimension — The number of dimensions in each value vector. Also \(d_{model}/h\) in the original paper. This controls the richness of the retrieved content. After attention, each head produces an output of shape \(n \times d_v\). All \(h\) heads are then concatenated to get back to \(n \times d_{model}\). \(d_v\) can in principle differ from \(d_k\), though in practice they are usually equal.
QK⊤
Score Matrix — Shape \(n \times m\). This is the heart of the attention mechanism: a matrix of raw compatibility scores between every query position and every key position. Entry \((i,j)\) measures how much token \(i\) should attend to token \(j\), before normalization. Computing this is an O(n·m·d_k) operation — the dominant cost of attention. For self-attention where n = m, this is O(n²·d_k), which is the well-known quadratic cost of attention with respect to sequence length.
√d_k
Scaling Factor — The denominator that keeps dot products from exploding. If we have two random vectors \(q\) and \(k\) with independent components each drawn from \(N(0,1)\), their dot product has mean 0 and variance \(d_k\). So its standard deviation is \(\sqrt{d_k}\). Without scaling, at d_k=64 we would see dot products with typical magnitude ≈ 8; at d_k=512, magnitude ≈ 22. These large values drive the softmax into near-zero gradient regions, making the attention weights collapse toward one-hot distributions and causing gradients to nearly vanish. Dividing by \(\sqrt{d_k}\) restores a well-conditioned unit-variance regime regardless of dimension.
softmax
Attention Weight Normalization — The softmax is applied row-wise across the key dimension, converting the raw score vector for each query into a valid probability distribution \(\alpha_i \in [0,1]^m\) that sums to 1. This means each token distributes exactly "1 unit of attention" across all positions. A concentrated distribution (one large weight, others near zero) means the token is attending sharply to a specific position. A diffuse distribution means it is aggregating information broadly. The softmax is differentiable everywhere, allowing gradient-based learning to shape these distributions end-to-end.

Project inputs into Query, Key, and Value spaces

Given input \(X \in \mathbb{R}^{n \times d_{model}}\), compute three separate linear projections: \(Q = XW^Q\), \(K = XW^K\), \(V = XW^V\). Each weight matrix is independently learned during training. This is not a trivial step — by using separate learned projections, the model can represent the same token differently depending on whether it is acting as a "searcher" (Q), an "index entry" (K), or a "content carrier" (V). The projections reduce dimensionality from \(d_{model}\) down to \(d_k\) or \(d_v\), which controls the per-head capacity.

Compute pairwise compatibility scores

Compute the matrix product \(S = QK^\top \in \mathbb{R}^{n \times m}\). Each entry \(S_{ij} = q_i \cdot k_j\) is the dot product between query \(i\) and key \(j\). The dot product is used because it is fast, differentiable, and measures vector alignment — two vectors pointing in the same direction in the key/query space get a high score, while orthogonal vectors get near zero. This produces a full n × m score matrix — meaning every query token scores every key token simultaneously, enabling the full parallel computation that makes Transformers so efficient.

Scale by 1/√d_k to stabilize gradients

Divide the entire score matrix by \(\sqrt{d_k}\): \(S_{scaled} = S / \sqrt{d_k}\). This is essential for numerical stability. Without scaling, dot products at large \(d_k\) can reach magnitudes of 20–30, pushing the softmax's exponentials into overflow territory and causing the attention distribution to collapse toward a hard one-hot vector. A one-hot attention distribution has essentially zero gradient everywhere except at the max position, which completely halts learning. Scaling re-centers the scores so softmax operates in a well-conditioned regime throughout training.

Apply mask (if needed)

In decoder self-attention, we must prevent token \(i\) from attending to any token at position \(j > i\) — future positions that haven't been generated yet. This is achieved by adding a large negative constant (−∞ in the limit, practically −1e9) to all upper-triangular entries of the score matrix. After softmax, these become exactly zero because \(e^{-\infty} = 0\). In padding masks, tokens corresponding to padding positions (used to equalize batch lengths) are also masked so they don't contaminate the attention of real tokens.

Apply row-wise softmax to get attention weights

Apply softmax independently to each row of the scaled score matrix: \(\alpha = softmax(S_{scaled})\). Softmax exponentiates each element and normalizes by the row sum, producing a valid probability distribution for each query token. The result \(\alpha \in [0,1]^{n \times m}\) has every row summing to 1. Critically, softmax is a smooth operation everywhere: even very large score differences still produce non-zero gradients, unlike a hard argmax. This differentiability is what allows the entire attention mechanism to be trained end-to-end by gradient descent.

Compute weighted sum of Values to produce output

The final output is \(O = \alpha V \in \mathbb{R}^{n \times d_v}\). Each row of the output \(o_i = \sum_j \alpha_{ij} \cdot v_j\) is a convex combination of all value vectors — a blend weighted by how strongly query \(i\) attends to each position. If token "it" attends 80% to "cat" and 20% to "sat", its output representation will be 80% drawn from the value of "cat" and 20% from "sat". This contextual blending is what gives attention — and the Transformer — its power to represent meaning as a function of context rather than as a fixed lookup table.

04
The Q, K, V Framework — Intuition

The Query-Key-Value framework is not an arbitrary design choice — it is a carefully constructed abstraction borrowed from information retrieval systems, generalized to work with continuous learned representations instead of discrete symbolic lookups.

In a classic key-value database (like a Python dictionary), you have a set of entries, each with an exact key. When you issue a query, the system finds the entry whose key exactly matches your query and returns its value. This is a hard lookup: the result is binary — either a key matches or it doesn't. The attention mechanism generalizes this to a soft lookup: instead of finding an exact match, you compute a continuous similarity score between your query and every key in the database, convert these scores into weights via softmax, and return a weighted mixture of all the values. The genius of this approach is that it is fully differentiable — the "lookup" is smooth, and gradients can flow all the way back through the query, keys, and values to update the underlying representations during training.

The Soft Database Analogy in Detail Imagine a knowledge base where each entry has a Key (its index or description) and a Value (the actual content you want to retrieve). A hard database query says: "give me the entry with key 'capital of France'". A soft attention query says: "here is my current context (Q); please compute how relevant each entry's key (K) is to my context, weight all their values (V) by that relevance, and give me the resulting mixture." In attention, this database is itself made of the input tokens — so every token simultaneously acts as a potential "entry" in the database, and every token simultaneously asks queries against all other tokens. The database is dynamic, input-dependent, and the keys, queries, and values are all learned continuous projections rather than symbolic strings.
QUERY — "What am I seeking?"

🔍 Q (Query)

The Query for each token encodes what that token currently needs from the rest of the sequence. It is produced by a learned projection \(W^Q\) that maps the token's full representation down to a d_k-dimensional "search vector." Think of it as a search query typed into a search engine — the quality of retrieval depends entirely on how well the query captures the information need. Through training, the model learns to construct queries that reliably retrieve the semantically or syntactically relevant positions. For the word "it" in "The cat sat on the mat because it was tired," the query learned for "it" will end up pointing toward the direction of noun-phrase keys like "cat" and "mat."

\(Q = X \cdot W^Q \;\in\; \mathbb{R}^{n \times d_k}\)
KEY — "What do I present to be found?"

🗝️ K (Key)

The Key for each token encodes how that token presents itself to be found by other tokens' queries. It is produced by a separate learned projection \(W^K\). Crucially, \(W^K \neq W^Q\) — the space in which tokens "advertise" themselves is different from the space in which they "search." This asymmetry is important: the features of a token that make it relevant to be found may be very different from the features it uses when searching. For example, a noun might present its grammatical role prominently in its key (making it easy to find for agreement-checking queries), while its query might focus on semantic content (when looking for modifiers or co-references).

\(K = X \cdot W^K \;\in\; \mathbb{R}^{m \times d_k}\)
VALUE — "What information do I actually carry?"

📦 V (Value)

The Value for each token is the actual information payload that gets aggregated into the output. It is produced by yet another separate projection \(W^V\). The separation of Values from Keys is the most subtle but important design choice: it means the mechanism that determines relevance (Q·K) is entirely decoupled from the mechanism that determines what is communicated (V). A token can have a high-salience key that makes it easy to attend to, while its value carries a carefully compressed version of its meaning optimized for the downstream task. In many heads, the value subspace encodes different features than the key subspace — the model learns these subspaces independently.

\(V = X \cdot W^V \;\in\; \mathbb{R}^{m \times d_v}\)

Self-Attention vs. Cross-Attention

The Q, K, V abstraction naturally gives rise to two distinct configurations depending on where the queries and keys/values come from:

Why Three Separate Learned Projections? One might wonder: why not just use the raw input embeddings directly, or use one shared projection matrix? The three-matrix design is deliberate and well-motivated. First, it gives the model much greater expressivity: the space in which tokens "search" (Q), the space in which they are "found" (K), and the space in which they "communicate" (V) are all independently optimized by gradient descent for the specific task at hand. Second, without separate projections, the attention mechanism would have an undesirable symmetry: token A attending to token B would force the same score as B attending to A (since the attention score would just be the dot product of raw embeddings). With separate Q and K matrices, the relationship is asymmetric and directional, which is much more expressive. Third, the projections allow dimensionality reduction from \(d_{model}\) to \(d_k\), which helps control computation cost in multi-head settings.

Scaled Dot-Product Attention

Scaled Dot-Product Attention is the specific, concrete implementation of the attention mechanism used in the Transformer. It is called "scaled" because of the critical division by \(\sqrt{d_k}\), and "dot-product" because the similarity measure is the inner product of query and key vectors. Every design decision in this formula has a clear mathematical motivation.

The formula combines three distinct operations into a single differentiable computation graph: a similarity measurement (dot product), a normalization (softmax), and an aggregation (matrix multiply with values). Each step can be implemented as a highly optimized GPU kernel, which is why the Transformer trains as fast as it does. Understanding why each piece is designed the way it is — rather than just memorizing the formula — is the key to being able to extend and adapt the mechanism for new architectures.

Full Formula
\[ \text{Attention}(Q, K, V) = \underbrace{\text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)}_{\text{attention weights } \alpha \,\in\, [0,1]^{n \times m}} V \]

The raw, unscaled attention score between query token \(i\) and key token \(j\) is the dot product of their corresponding vectors:

\[ e_{ij} = \frac{q_i \cdot k_j}{\sqrt{d_k}} \]

The dot product \(q_i \cdot k_j = \sum_{l=1}^{d_k} q_{il} \cdot k_{jl}\) accumulates products across all \(d_k\) dimensions of the query-key space. Two vectors pointing in exactly the same direction yield the maximum score; two orthogonal vectors yield zero; two opposite vectors yield a negative score. In practice, after training, the projection matrices \(W^Q\) and \(W^K\) learn to rotate the input space such that semantically or syntactically related pairs of tokens naturally align in the query-key subspace, automatically producing high scores for relevant pairs and low scores for irrelevant ones.

e_ij
The raw, unnormalized attention score from query position i to key position j. Can be any real number — positive means likely relevant, negative means likely irrelevant. Not yet a probability.
q_i
The query vector for token i — the i-th row of matrix Q. Shape: \(1 \times d_k\). Represents the information need of token i as learned by \(W^Q\).
k_j
The key vector for token j — the j-th row of matrix K, transposed for the dot product. Shape: \(d_k \times 1\). Represents how token j presents itself for retrieval, as learned by \(W^K\).

This is the most commonly under-explained detail in the formula, but it is critical for training stability. To understand why scaling is necessary, consider what happens when we take the dot product of two high-dimensional random vectors.

If \(q\) and \(k\) are vectors of dimension \(d_k\) with components independently drawn from \(N(0,1)\), then each component product \(q_l \cdot k_l\) has mean 0 and variance 1. Since the dot product is the sum of \(d_k\) such products, by the variance-of-sums rule:

\[ \text{Var}(q \cdot k) = \sum_{l=1}^{d_k} \text{Var}(q_l k_l) = d_k \]

The standard deviation of the dot product grows as \(\sqrt{d_k}\). At \(d_k = 64\), we expect dot products of typical magnitude ≈ 8. At \(d_k = 512\), magnitude ≈ 22. Now consider the effect on softmax: if the maximum score is 22 and the second-highest is 18, the softmax output will be dominated by the maximum to an extreme degree — \(e^{22} / (e^{22} + e^{18}) \approx 0.982\). This near-one-hot distribution has essentially zero gradient at all non-maximum positions, grinding learning to a halt. Dividing by \(\sqrt{d_k}\) restores unit variance regardless of model size:

\[ \text{Var}\!\left(\frac{q \cdot k}{\sqrt{d_k}}\right) = \frac{d_k}{d_k} = 1 \]
Empirical Consequence Without Scaling Vaswani et al. (2017) explicitly note that without scaling, they empirically observed extremely small gradients in the attention layers. The scaling is not just a theoretical nicety — it is empirically necessary for models with even moderate \(d_k\). At \(d_k = 512\) (used in large models), the difference in training stability between scaled and unscaled dot-product attention is dramatic.

Softmax converts the row of raw scores for each query position into a valid probability distribution. It is applied independently to each row of the score matrix — meaning each query token distributes its attention independently of other query tokens:

\[ \alpha_{ij} = \frac{\exp(e_{ij})}{\sum_{l=1}^{m} \exp(e_{il})} \]

The exponential function does two things: it ensures all values are positive (since \(e^x > 0\) for all real \(x\)), and it amplifies differences — a score that is twice as large as another gets exponentially more weight. Dividing by the sum normalizes the row to sum to 1. The resulting weights \(\alpha_{ij}\) represent the fraction of query \(i\)'s "attention budget" allocated to position \(j\). The output for query \(i\) is then:

\[ o_i = \sum_{j=1}^{m} \alpha_{ij} \cdot v_j \]
α_ij
The normalized attention weight from query position i to value/key position j. Always in [0,1]. The full row \((\alpha_{i1}, \ldots, \alpha_{im})\) sums to exactly 1. High α = strong attention, near 0 = near-complete suppression.
m
The number of key/value positions being attended to — the "database" size. In self-attention, m = n (sequence length). In cross-attention, m is the encoder sequence length and n is the decoder sequence length, and they can differ.
o_i
The output vector for query position i: a convex combination of all value vectors, weighted by relevance. Its shape is \(d_v\). It represents a context-aware summary of all positions in the sequence, as seen from the perspective of token i.

Causal masking (also called autoregressive masking or look-ahead masking) is applied in the decoder's self-attention sub-layer. The fundamental constraint of language generation is that when predicting the token at position \(t\), the model should only have access to positions 1 through \(t-1\) — it cannot "cheat" by looking at the answer it is supposed to predict. This is implemented by zeroing out the attention weights for all future positions before the softmax. The standard approach is to add a masking matrix \(M\) to the scores:

\[ \tilde{e}_{ij} = \begin{cases} e_{ij} & \text{if } j \leq i \\ -\infty & \text{if } j > i \end{cases} \]

Since \(\exp(-\infty) = 0\), after softmax the attention weights at all future positions are exactly zero. In practice, we use a large finite constant like −10⁹ instead of −∞ to avoid numerical issues in float32. The mask is typically computed once as an upper-triangular matrix and reused across all layers and heads — it is not a learned parameter.

Note that encoder self-attention uses no mask (or only a padding mask) — encoder tokens can freely attend to all other encoder tokens in both directions, which is why BERT-style encoders are said to be "bidirectional." Decoder self-attention is "unidirectional" or "causal" due to the mask.

Padding Masks vs. Causal Masks These are two distinct types of masking that are often combined. A causal mask zeros out future positions (upper triangle). A padding mask zeros out positions corresponding to padding tokens — tokens added to make all sequences in a batch the same length. Both are applied as additive masks to the score matrix before softmax. Without padding masks, padding tokens would "pollute" the attention distribution of real tokens by receiving non-trivial attention weights.

Multi-Head Attention

A single attention head computes one set of attention weights and produces one contextual representation per position. But natural language is structured along many independent dimensions simultaneously — syntax, semantics, coreference, positional proximity, and more. Multi-Head Attention (MHA) is the architectural solution: run \(h\) independent attention mechanisms in parallel, each learning to route information along a different relational dimension, then merge all their outputs into a single unified representation.

The intuition is powerful. Consider the sentence: "The animal didn't cross the street because it was too tired." When processing the word "it", a model needs to resolve: (1) does "it" refer to "animal" or "street"? — a coreference task; (2) what is the grammatical role of "it" relative to "was"? — a syntax task; (3) how far away is "it" from "animal"? — a positional task; (4) what semantic class does "it" belong to? — a semantic task. A single attention head cannot simultaneously specialize in all four of these. But with 8 heads, each head can independently develop a different routing strategy, and the final output layer learns to assemble these perspectives into a coherent representation.

Multi-Head Attention
\[ \text{MultiHead}(Q,K,V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)\, W^O \] \[ \text{where} \quad \text{head}_i = \text{Attention}(Q W^Q_i,\; K W^K_i,\; V W^V_i) \]
h
Number of Attention Heads — In the original Transformer, \(h = 8\). Each head is a fully independent scaled dot-product attention operation with its own set of projection matrices. The heads run in parallel — there is no communication between heads during the attention computation itself. More heads means more distinct relational patterns can be captured simultaneously, but each head operates at a lower dimension (\(d_k = d_{model}/h\)), so total computation stays constant. Modern models use 12, 16, or even 128 heads depending on model scale.
W^Q_i, W^K_i, W^V_i
Per-Head Projection Matrices — Each head \(i\) has its own independent set of three weight matrices: \(W^Q_i \in \mathbb{R}^{d_{model} \times d_k}\), \(W^K_i \in \mathbb{R}^{d_{model} \times d_k}\), and \(W^V_i \in \mathbb{R}^{d_{model} \times d_v}\). These are what make each head truly independent — head 1 learns to project inputs into a subspace useful for coreference resolution, head 2 into a subspace useful for syntactic agreement, and so on, all driven entirely by the gradient signal during training. There is no explicit assignment of "what each head should learn" — the specialization emerges organically from the optimization process.
d_k = d_v = d_model / h
Per-Head Reduced Dimension — To ensure that the total computation of multi-head attention is comparable to single-head attention at full \(d_{model}\) dimension, the original paper divides the model dimension equally across heads. With \(d_{model} = 512\) and \(h = 8\), each head has \(d_k = d_v = 64\). The total parameter count across all projection matrices is \(h \times 3 \times d_{model} \times d_k = 3 \times d_{model}^2\), the same as a single head at full dimension. This is the key efficiency trade-off: we get the benefit of diverse routing patterns from multiple heads without increasing the total parameter count or FLOPs.
Concat(head₁, …, headₕ)
Head Concatenation — Each head produces an output tensor of shape \(n \times d_v\). The outputs from all \(h\) heads are concatenated along the last axis, yielding a tensor of shape \(n \times (h \cdot d_v) = n \times d_{model}\). Concatenation preserves all the information from all heads without any information loss. The concatenated tensor contains the independent perspectives of all heads stacked side by side — it is then up to the output projection \(W^O\) to synthesize these perspectives into a coherent final representation.
W^O
Output Projection Matrix — Shape \(d_{model} \times d_{model}\). This is the final learned linear layer in multi-head attention. It takes the concatenated head outputs and projects them back into the model's representation space. Crucially, this layer learns to blend the outputs of all heads — it can up-weight heads that are providing useful signal, down-weight redundant or noisy heads, and combine information across head boundaries. Without \(W^O\), the heads' outputs would be entirely independent and uncoordinated. The output projection is what transforms a collection of independent analyses into an integrated, coherent token representation.

Why Multi-Head Attention is More than the Sum of Its Parts The power of multi-head attention is not simply additive. It is multiplicative in a subtle sense: by learning different subspaces to project into, each head is essentially computing a different "view" of the same sequence. When the output projection \(W^O\) merges these views, the resulting representation encodes relationships along multiple dimensions simultaneously. This is fundamentally more expressive than any single attention pattern, because real language contains many overlapping, simultaneous relational structures — and a single-head model cannot represent them all at once without interference. Multi-head attention effectively decomposes the full representation space into subspaces, and each subspace specializes independently.
Complexity Analysis — MHA vs. RNN
\[ \text{Self-Attention: } O(n^2 \cdot d) \qquad \text{RNN: } O(n \cdot d^2) \]

Self-attention is quadratic in sequence length \(n\) but linear in model depth \(d\). RNNs are linear in \(n\) but quadratic in \(d\), and critically cannot be parallelized over \(n\). For modern NLP tasks where \(n\) is in the hundreds to thousands and \(d\) is 512–4096, the Transformer's parallelizability over \(n\) is a decisive advantage on GPU hardware. The O(n²) cost of full self-attention only becomes a practical bottleneck for very long sequences (n > 4096), which motivated the development of efficient attention variants like Longformer, BigBird, and Flash Attention.

Transformer Architecture

The full Transformer architecture combines all the components we have studied — multi-head attention, positional encoding, feed-forward networks, and residual + layer normalization — into a coherent, stackable structure. The design is modular: encoder blocks and decoder blocks each repeat \(N\) times, and the decoder connects to the encoder via cross-attention.

Understanding the architecture means understanding not just what each component does in isolation, but how information flows through the entire system during both encoding and decoding. The encoder reads the full input simultaneously (parallel, bidirectional), building deeply contextual representations of each source token. The decoder generates the output one token at a time (sequential at inference, but parallel at training via teacher forcing), conditioned on the encoder's representations through cross-attention at every decoder layer.

This is the mechanism by which the decoder "reads" the source: at each decoding step and each decoder layer, cross-attention allows each output token to selectively retrieve information from any part of the encoded source sentence. It replaces the single fixed context vector of the old RNN Seq2Seq models with a dynamic, position-specific, multi-layer retrieval mechanism.

Positional Encoding

Attention is a set operation at its core — if you permute the tokens in the input, the output representations permute in exactly the same way. The model is completely blind to the order in which tokens appear. For image patches or knowledge graphs this may be acceptable, but for language — where "dog bites man" and "man bites dog" have entirely different meanings — order is everything. Positional Encoding is the mechanism by which order information is injected into the model.

The solution is to add a position-dependent signal directly to the input embeddings before they enter the first Transformer layer. This signal must meet several requirements: it must be unique per position (so positions are distinguishable), it must be consistent across sequences of different lengths, it must preserve enough information for the model to compute relative positions (not just absolute ones), and it should generalize to sequence lengths not seen during training. The sinusoidal encoding proposed in the original paper meets all of these requirements elegantly.

Why Attention Alone Cannot Learn Position Consider two sentences identical in content but with swapped word order. Self-attention, computing \(QK^T\) from embeddings alone, would produce the same set of attention scores for both — because it sees the same set of token embeddings, just in a different order. The attention operation is order-agnostic: it depends on what is in the sequence, not where. Without positional information baked into the embeddings, the model literally cannot distinguish "The cat chased the dog" from "The dog chased the cat."
Sinusoidal Positional Encoding
\[ PE_{(pos,\, 2i)} = \sin\!\left(\frac{pos}{10000^{2i/d_{model}}}\right) \] \[ PE_{(pos,\, 2i+1)} = \cos\!\left(\frac{pos}{10000^{2i/d_{model}}}\right) \]
pos
Token position index — An integer from 0 to sequence_length − 1 indicating the absolute position of the token in the input sequence. Position 0 is the first token, position 1 the second, etc. The PE formula produces a unique vector for each value of pos, and this vector is added (not concatenated) to the word embedding at that position. The final input to the first Transformer layer is therefore \(E_{token} + PE_{pos}\).
i
Dimension pair index — An integer from 0 to \(d_{model}/2 - 1\). Each pair of adjacent dimensions (2i, 2i+1) in the positional encoding vector uses a different frequency sinusoid. Dimension pair 0 uses the highest frequency (shortest wavelength), dimension pair \(d_{model}/2 - 1\) uses the lowest frequency (longest wavelength). This multi-scale structure allows the encoding to represent position at both fine-grained (local) and coarse-grained (global) levels simultaneously.
10000^(2i/d_model)
Frequency denominator (wavelength factor) — This creates a geometric progression of frequencies across the embedding dimensions, from very high frequency (at \(i=0\), wavelength ≈ 2π ≈ 6.28 tokens) to very low frequency (at \(i=d_{model}/2\), wavelength ≈ 10000·2π ≈ 62,832 tokens). The factor 10000 was chosen so that the frequencies span a wide enough range to encode positions from 0 to sequences of typical NLP length (~5000 tokens) without aliasing. Think of it like binary counting — low-order bits (high frequency) change quickly with position, high-order bits (low frequency) change slowly.
sin / cos pairs
Orthogonal frequency components — Using both sine and cosine at each frequency, rather than just one, gives the encoding an important property: the relative offset between two positions can be expressed as a fixed linear transformation of the positional encodings, regardless of absolute position. Formally, \(PE(pos+k)\) can be written as a linear function of \(PE(pos)\) for any offset \(k\). This means the dot product in the attention mechanism can implicitly compute relative distances, not just absolute positions — a crucial capability for understanding long-range dependencies.
Sinusoidal vs. Learned Positional Embeddings — Trade-offs The sinusoidal approach is deterministic and requires no extra parameters. It naturally generalizes to sequence lengths not seen during training because the formula can be evaluated at any integer position. The encoding also has the relative-offset property described above. The main disadvantage is that it constrains the representation of position to a fixed functional form — the model cannot freely adjust how position is encoded during training. Learned positional embeddings (used in BERT, GPT-2/3/4, and most modern LLMs) simply add a trainable embedding table indexed by position — one learned vector per position, optimized end-to-end. This is more flexible and often performs slightly better in practice, but is limited to the maximum sequence length seen during training and cannot trivially extend to longer sequences. More recent approaches like RoPE (Rotary Positional Embeddings, used in LLaMA) encode position in the attention scores directly rather than the embeddings, offering better length generalization.

Position-wise Feed-Forward Network

After each attention sub-layer, every position's representation passes through a two-layer fully-connected network. This is sometimes called the "thinking" step: while attention is responsible for gathering contextually relevant information from across the sequence, the FFN is responsible for transforming and processing that gathered information within each position independently.

The FFN is applied position-wise — the exact same set of weight matrices is applied independently to each of the \(n\) token representations. There is no interaction between positions in the FFN; positions only interact in the attention sub-layers. This division of labor is elegant: attention handles routing (deciding which information to aggregate from where), while the FFN handles transformation (deciding how to process the aggregated information). The two operations are complementary, and stacking them repeatedly across \(N\) layers allows the model to build progressively more abstract representations.

FFN Formula
\[ \text{FFN}(x) = \max(0,\; xW_1 + b_1)\, W_2 + b_2 \]
x
Input to FFN — The output of the preceding attention sub-layer (after residual + LayerNorm), shape \(n \times d_{model}\). Each of the \(n\) rows is processed independently by the same FFN — the operation is exactly equivalent to applying a single-hidden-layer MLP to each token vector individually. This position-wise independence means the FFN has no direct access to positional relationships — it transforms each token's representation based solely on its own content, with no knowledge of neighboring positions.
W₁, b₁
First (expansion) linear layer — Projects from \(d_{model}\) up to \(d_{ff}\), the inner hidden dimension of the FFN. Shape: \(d_{model} \times d_{ff}\). The expansion ratio is typically 4×: in the original paper, 512 → 2048. This upward projection creates a much higher-dimensional space in which the non-linear activation can operate, giving the model a large "working memory" for each token's transformation. The larger the ratio, the more expressive the FFN, but also the more parameters and computation.
max(0, ·) — ReLU
Non-linear activation function — Applied element-wise to the expanded intermediate representation. ReLU (\(max(0, x)\)) zeros out all negative values and passes positive values unchanged. Without this non-linearity, the entire FFN would collapse to a single linear transformation (since the composition of two linear functions is linear). The non-linearity is what gives the FFN its ability to represent complex, non-linear functions of the input. ReLU's "dead neuron" problem (units that stay at zero permanently) has motivated the shift toward smoother alternatives like GELU and SwiGLU in modern LLMs.
W₂, b₂
Second (projection) linear layer — Projects from \(d_{ff}\) back down to \(d_{model}\). Shape: \(d_{ff} \times d_{model}\). This bottleneck projection compresses the expanded intermediate representation back to the model's standard dimensionality, making the FFN output compatible with the residual connection that follows. The net effect of W₁ → ReLU → W₂ is a learned, position-specific, non-linear transformation of each token's representation — a highly expressive operation that accounts for roughly two-thirds of all Transformer parameters in most architectures.
FFN Layers as Key-Value Memory Banks A striking finding by Geva et al. (2021) reframes what FFN layers are actually doing. The first weight matrix \(W_1\) can be thought of as storing a set of "keys" — each row is a pattern that, when matched by the input via the dot product, triggers a high activation. The second weight matrix \(W_2\) can be thought of as storing the corresponding "values" — when a key is activated, its corresponding column in \(W_2\) contributes to the output. Under this interpretation, the FFN is performing a soft content-addressable retrieval: "if the input token resembles pattern \(k\), add value \(v\) to the output." Experiments showed that specific neurons in FFN layers reliably "fire" for specific concepts, facts, or syntactic patterns — effectively storing factual associations learned during pretraining. Attention gathers contextual signals; FFN layers consult learned knowledge.
Modern Activation Functions — Beyond ReLU The original Transformer used ReLU, but subsequent research found smoother activations improve performance. GELU (Gaussian Error Linear Unit), used in BERT and GPT: \(GELU(x) = x \cdot \Phi(x)\) where \(\Phi\) is the standard normal CDF. GELU applies a gentle, non-zero gradient even for negative inputs, unlike ReLU's hard zero. SwiGLU, used in LLaMA, PaLM, GPT-4: \(SwiGLU(x, W, V) = \text{Swish}(xW) \odot (xV)\) — a gated variant that multiplies two independent projections together. SwiGLU consistently outperforms GELU by a few perplexity points but requires 50% more parameters in the FFN to achieve the same hidden dimension, prompting a reduction in \(d_{ff}\) to compensate.

Add & Layer Normalization

Every sub-layer in the Transformer — both the multi-head attention and the feed-forward network — is wrapped in a two-part stabilization mechanism: a residual (skip) connection and layer normalization. These are not optional refinements. Without them, training a 6-layer Transformer (let alone a 96-layer GPT-3) would be practically impossible due to vanishing/exploding gradients and activation drift.

The two components are applied together in the pattern shown below. Understanding why each component is needed — and what goes wrong without them — is essential for understanding how deep Transformers are trained stably.

Post-LN (Original Paper)
\[ \text{Output} = \text{LayerNorm}\!\left(x + \text{Sublayer}(x)\right) \]

Residual Connection — The Skip Path

Skip Connection
\[ y = x + F(x) \]

The residual connection, introduced by He et al. (2015) for deep convolutional networks and adopted wholesale into the Transformer, adds the layer's input \(x\) directly to its output \(F(x)\) before passing to the next stage. This simple operation has a profound effect on training dynamics.

The key insight is in the gradient. During backpropagation, the gradient of the loss with respect to \(x\) becomes \(\frac{\partial L}{\partial x} = \frac{\partial L}{\partial y} \cdot (1 + \frac{\partial F}{\partial x})\). The +1 term means the gradient always has a direct, unobstructed path back through the skip connection, regardless of what \(F\) does to it. In very deep networks without skip connections, gradients must flow through every transformation, and if any transformation is poorly conditioned, the gradient shrinks (vanishes) or grows (explodes). The residual connection guarantees a "gradient highway" through the entire network depth.

A second benefit is that each sub-layer only needs to learn the residual — the deviation from identity — rather than a full transformation from scratch. If a layer contributes nothing useful, it can learn weights close to zero, making the residual connection approximately a pass-through. This makes optimization much easier: layers can start near the identity function and gradually specialize during training.

Layer Normalization — Taming Activation Drift

LayerNorm Formula
\[ \text{LayerNorm}(x) = \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} \cdot \gamma + \beta \] \[ \mu = \frac{1}{d}\sum_{i=1}^d x_i \qquad \sigma^2 = \frac{1}{d}\sum_{i=1}^d (x_i - \mu)^2 \]
μ (mu)
Per-token mean — Computed across the feature dimension \(d_{model}\) for each token independently. If token i's representation is \(x_i \in \mathbb{R}^{512}\), then μ is the average of those 512 values. This is fundamentally different from Batch Normalization, which normalizes across the batch dimension. Because LayerNorm operates per-token (per-sample), it works equally well for batch size 1 and does not depend on batch statistics — making it ideal for inference and for variable-length sequences in NLP.
σ² (sigma squared)
Per-token variance — Also computed across the \(d_{model}\) feature dimension for each token independently. Together, \(\mu\) and \(\sigma\) characterize the distribution of a single token's activation vector. Subtracting the mean and dividing by the standard deviation "standardizes" the representation so it has zero mean and unit variance — regardless of how large or small the activations became during the forward pass. This prevents any one token's representation from growing unboundedly through layers, which would destabilize gradients.
ε (epsilon)
Numerical stability additive — A small constant (typically 1e-5 or 1e-6) added to the denominator inside the square root. This prevents division by zero in the rare case where all features of a token happen to be identical (σ² = 0). Without ε, the normalization would produce NaN, which would corrupt the entire forward and backward pass. The value of ε is a hyperparameter — larger values add more bias but provide more stability.
γ (gamma)
Learned scale parameter — A vector of shape \(d_{model}\), initialized to all-ones. After normalization, each dimension is independently re-scaled by its corresponding γ value. This is necessary because the normalization step might remove information that was encoded in the scale (magnitude) of the representation. The γ parameter allows the model to restore that scale if needed, effectively "undoing" part of the normalization along certain dimensions. Learned by gradient descent alongside all other model parameters.
β (beta)
Learned shift (bias) parameter — A vector of shape \(d_{model}\), initialized to all-zeros. After normalization and re-scaling by γ, each dimension is shifted by its β value. Without β, the normalized output is constrained to be zero-mean, which might not be optimal for every layer. β allows each dimension to have an independently learned bias. Together, γ and β give the normalized layer the full representational freedom of an arbitrary affine transformation — so LayerNorm can always reproduce the original unnormalized distribution if that's what minimizes the loss.
Post-LN vs. Pre-LN — A Critical Architectural Difference The original Transformer paper applied LayerNorm after the residual addition (Post-LN): \(y = \text{LayerNorm}(x + F(x))\). This worked for the original 6-layer model but was found to be unstable for very deep networks. Most modern LLMs (GPT-2, GPT-3, LLaMA, Mistral) use Pre-LN instead, applying LayerNorm to the input before the sublayer: \(y = x + F(\text{LayerNorm}(x))\). With Pre-LN, the residual stream is always in a stable, normalized scale — the raw representations accumulate through the skip path without being normalized, while each sublayer receives a well-conditioned input. Pre-LN models train without learning rate warm-up and are stable across a much wider range of hyperparameters. \[ \text{Pre-LN: } y = x + F(\text{LayerNorm}(x)) \]

Transformer Variants

The original Transformer was designed for machine translation and used a full encoder-decoder architecture. Within a few years, the NLP community discovered that specializing the architecture — using only the encoder, only the decoder, or modifying the attention pattern — yields models better suited for different tasks and scales. This exploration produced the modern landscape of large language models.

The key axis of variation is which tokens can attend to which other tokens — a choice that fundamentally determines what the model can learn and how it can be deployed. Encoder-only models see the full bidirectional context and excel at understanding tasks. Decoder-only models see only past context and excel at generation. Encoder-decoder models use both and are best for conditional generation (translation, summarization). Efficient attention variants address the O(n²) bottleneck for long documents.

Attention Complexity Variants

Full Attention — O(n²)

Every token attends to every other token. The score matrix has n² entries and must be computed and stored. Exact and highly expressive. Practical for sequences up to ~4K tokens on modern hardware. The baseline against which all variants are measured. Used in virtually all Transformers for sequences under 4K tokens.

Sparse Attention — O(n√n)

Each token attends to only a fixed subset of other tokens — typically a local window of k neighbors, plus a set of "global" tokens that attend to everything. Examples: Longformer (sliding window + global), BigBird (random + window + global). Reduces memory and computation while preserving most of the benefits of full attention for typical NLP tasks.

Linear Attention — O(n)

Approximates the softmax attention using kernel methods. Instead of computing \(softmax(QK^T)V\), computes \(\phi(Q)(\phi(K)^T V)\) where φ is a feature map approximating softmax. By associativity, this can be computed as \(\phi(Q)((\phi(K)^T V))\) in O(n·d²) — linear in sequence length. Trades exactness for scalability. Used in Performer, RWKV, and linear Transformer variants.

Flash Attention — O(n²) time, O(n) memory

The same mathematical result as full attention, but computed with a memory-efficient tiled algorithm. Instead of computing the full n×n matrix and storing it in GPU HBM (slow, high-bandwidth memory), Flash Attention processes tiles of the score matrix entirely in fast SRAM (on-chip memory). This IO-aware approach dramatically reduces memory access — the dominant bottleneck for GPU throughput on attention — making the computation 2–4× faster in practice.

Scaled Dot-Product Attention

Python / PyTorch Scaled Dot-Product Attention 
import torch
import torch.nn.functional as F
import math

def scaled_dot_product_attention(
    Q: torch.Tensor,   # (batch, heads, n, d_k)
    K: torch.Tensor,   # (batch, heads, m, d_k)
    V: torch.Tensor,   # (batch, heads, m, d_v)
    mask: torch.Tensor = None
) -> torch.Tensor:
    d_k = Q.size(-1)

    # Step 1: scores = QK^T / sqrt(d_k)  →  (batch, heads, n, m)
    scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)

    # Step 2: causal / padding mask — fill future positions with -inf
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))

    # Step 3: softmax → attention weights α, shape (batch, heads, n, m)
    alpha = F.softmax(scores, dim=-1)

    # Step 4: weighted sum of values → context vectors
    output = torch.matmul(alpha, V)   # (batch, heads, n, d_v)
    return output, alpha

Multi-Head Attention Module

Python / PyTorch Multi-Head Attention Module 
import torch.nn as nn

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model: int, h: int, dropout: float = 0.1):
        super().__init__()
        assert d_model % h == 0, "d_model must be divisible by h"
        self.h   = h
        self.d_k = d_model // h       # per-head dimension

        # Four learnable projection matrices
        self.W_q = nn.Linear(d_model, d_model, bias=False)
        self.W_k = nn.Linear(d_model, d_model, bias=False)
        self.W_v = nn.Linear(d_model, d_model, bias=False)
        self.W_o = nn.Linear(d_model, d_model)
        self.dropout = nn.Dropout(dropout)

    def _split_heads(self, x):
        # (B, L, d_model) → (B, h, L, d_k)
        B, L, _ = x.shape
        return x.view(B, L, self.h, self.d_k).transpose(1, 2)

    def forward(self, Q, K, V, mask=None):
        # 1. Project + split into h heads
        Q = self._split_heads(self.W_q(Q))
        K = self._split_heads(self.W_k(K))
        V = self._split_heads(self.W_v(V))

        # 2. Scaled dot-product attention — all heads in parallel
        out, attn = scaled_dot_product_attention(Q, K, V, mask)

        # 3. Concat heads: (B, h, L, d_k) → (B, L, d_model)
        B, _, L, _ = out.shape
        out = out.transpose(1, 2).contiguous().view(B, L, -1)

        # 4. Final output projection W^O
        return self.W_o(out), attn

Transformer Encoder Layer

Python / PyTorch Transformer Encoder Layer 
class EncoderLayer(nn.Module):
    def __init__(self, d_model: int, h: int, d_ff: int, dropout=0.1):
        super().__init__()
        self.attn    = MultiHeadAttention(d_model, h, dropout)
        self.ffn     = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),              # GELU outperforms ReLU empirically
            nn.Dropout(dropout),
            nn.Linear(d_ff, d_model)
        )
        self.norm1   = nn.LayerNorm(d_model)
        self.norm2   = nn.LayerNorm(d_model)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x, mask=None):
        # Pre-LN style (more stable than original Post-LN)
        # Sub-layer 1: Multi-Head Self-Attention
        attn_out, _ = self.attn(self.norm1(x), self.norm1(x), self.norm1(x), mask)
        x = x + self.dropout(attn_out)     # residual connection

        # Sub-layer 2: Position-wise Feed-Forward Network
        x = x + self.dropout(self.ffn(self.norm2(x)))  # residual
        return x

Architecture Comparison

The Transformer did not emerge in a vacuum — it replaced RNNs and LSTMs as the dominant sequence model, and it was also compared extensively against CNNs applied to sequences. Understanding precisely where and why the Transformer outperforms these architectures is essential context for knowing when to use which tool.

The comparison is multi-dimensional. RNNs have lower parameter counts per layer and work well for short sequences. CNNs are highly parallelizable and efficient for local pattern matching. But the Transformer uniquely solves the long-range dependency problem with constant-path-length attention — a fundamental architectural advantage that no amount of engineering can replicate in sequential or locally-connected models.

The O(1) Path Length — Why It Matters More Than It Looks In an RNN, information from position 1 must travel through n−1 hidden state updates before reaching position n, and each update potentially dilutes or corrupts the original signal. This means that for a sentence of length 50, word 1's representation has been "touched" 49 times before it influences the model's representation at position 50. Even with gradient clipping and careful initialization, this compounding effect limits what can be reliably learned. In a Transformer, any two positions are directly connected by a learned attention weight — updated in a single matrix multiply. The gradient path for the association "word 1 → word 50" has length exactly 1 (within one layer), or L (across L layers, which is far better than n). This is why Transformers learn long-range dependencies that LSTMs essentially cannot, even with unlimited training data.

Encoder-Only · Decoder-Only · Encoder-Decoder

Not every task needs the full encoder-decoder Transformer. The architectural choice — encoder-only, decoder-only, or encoder-decoder — is a direct consequence of what the task requires. Understanding why each design exists is as important as understanding how it works.

The central question: Does the task require understanding, generation, or conditioned generation?

If a task only needs to read and understand a fixed input — classification, NER, extractive QA — an encoder-only model is all you need. If it needs to generate freely from a prompt with no separate source sequence — open-ended writing, chat, code completion — a decoder-only model is the natural fit. If it needs to produce a new sequence conditioned on a complete source — translation, summarization, structured generation — the full encoder-decoder pair is correct.

Encoder-Only Architecture

An encoder-only Transformer reads the entire input sequence simultaneously. Every token attends to every other token with full, bidirectional self-attention — there is no mask hiding future positions. Token at position 3 can freely attend to token at position 10, and vice versa. The result is a rich, context-aware representation of each token that incorporates information from both directions.

This bidirectionality is the key property. For understanding tasks, knowing what comes after a word is just as informative as what comes before it. In "The bank by the river was muddy," understanding that "bank" means a riverbank — not a financial institution — requires reading the entire sentence, not just the words before "bank." Bidirectional attention enables exactly this kind of global disambiguation.

🔍

What encoder-only is good at

Tasks where the full input is known upfront and the output is a label, a span, or a fixed-size vector. The model reads everything, then produces a single answer. Examples: text classification, sentiment analysis, named entity recognition, question answering (extracting a span from a context), natural language inference.

🚫

What encoder-only cannot do

Open-ended generation — producing a sequence of arbitrary length that didn't exist in the input. An encoder produces one output vector per input token; it has no mechanism to generate tokens that go beyond the input. You cannot ask a pure encoder to write a paragraph or translate a sentence.

Attention type
Bidirectional self-attention — every token attends to every other token with no masking. Token \(i\) can see tokens at positions \(j < i\) (past) and \(j > i\) (future) simultaneously. This is the defining property that makes encoders powerful for understanding.
Output
Contextual embeddings — one vector per input token, each enriched with full-sequence context. These embeddings are then passed to a task-specific head: a linear layer for classification, a span predictor for QA, a token-level classifier for NER.
Representative model
BERT (Bidirectional Encoder Representations from Transformers, Devlin et al. 2018) — the canonical encoder-only model. Pre-trained with Masked Language Modeling (MLM): 15% of tokens are randomly masked, and the model must predict them from bidirectional context. This forces the model to use both left and right context, producing genuinely bidirectional representations.
Other examples
RoBERTa (optimised BERT training), DistilBERT (66% smaller, 97% performance), ALBERT (parameter sharing), DeBERTa (disentangled attention), sentence-transformers (sentence embeddings for semantic search).
Why no decoder?
A decoder adds parameters, computation, and cross-attention layers that serve no purpose when there is no target sequence to generate. Encoder-only models are leaner and faster at inference for understanding tasks. The absence of causal masking is also what enables bidirectionality — adding a decoder would require masking, which would restrict attention flow.

Decoder-Only Architecture

A decoder-only Transformer has no encoder at all. It consists solely of stacked decoder blocks, each with a single type of attention: causal (masked) self-attention. Every token can attend to all preceding tokens in the sequence, but not to future ones — the upper-triangular positions of the attention score matrix are set to −∞ before softmax, so they contribute zero to the output. This left-to-right constraint is the defining property of decoder-only models.

The input and output live in the same sequence. Rather than having a separate source and target, a decoder-only model is given a prompt — a prefix of tokens — and asked to continue it. Generation happens by appending one token at a time: the model computes a probability distribution over the vocabulary at the final position, samples or argmaxes a token, appends it, and feeds the extended sequence back in. This autoregressive loop is repeated until an end-of-sequence token is produced or a length limit is hit.

The reason decoder-only models have become dominant in large-scale LLMs is elegant: next-token prediction is a universal pre-training objective. Given enough data, predicting the next word requires the model to implicitly learn grammar, facts, reasoning, and style — all at once. There is no need to design separate tasks or labels. And because the training objective and the generation procedure are identical, there is no train-test mismatch. At scale, this simplicity proves extraordinarily powerful.

✍️

What decoder-only is good at

Open-ended generation: text completion, creative writing, chat, code synthesis, instruction following. Any task where the output is a direct continuation or transformation of the input prompt — with no need to encode a separate, fixed source document.

🔑

In-context learning

Decoder-only models uniquely support in-context learning (ICL): you can demonstrate a task with a few input-output examples in the prompt itself, and the model generalises to a new example without any gradient updates. This emerges from causal attention over the full prompt — the model conditions on the demonstrations the same way it conditions on any prior context.

🚫

What decoder-only cannot do natively

Deep bidirectional understanding. Because every token only sees its past, the representation of token i cannot incorporate information from tokens j > i at the same layer. For classification or retrieval tasks that benefit from full-context representations, encoder-only models still hold advantages — though large decoder-only models can approximate this via prompting.

Attention type
Causal (masked) self-attention only — token at position \(t\) attends only to positions \(1 \ldots t\). The upper-triangular attention score matrix is masked to \(-\infty\) before softmax. No encoder, no cross-attention sublayer.
Output
Next-token probability distribution — at each position \(t\), the model projects the hidden state through a linear layer to vocabulary size and applies softmax. During generation, this distribution is sampled or greedy-decoded to produce the next token.
Pre-training objective
Causal Language Modeling (CLM) — predict token \(t\) given all tokens \(1 \ldots t{-}1\). This is the standard left-to-right language modelling objective, also called autoregressive language modelling. Training loss is cross-entropy averaged over all positions simultaneously, making every token in every training example a supervised signal.
Representative models
GPT series (OpenAI) — GPT-1 proved the approach; GPT-2 showed surprising zero-shot capability; GPT-3 (175B) demonstrated in-context learning at scale; GPT-4 added multimodality. LLaMA / LLaMA-2 / LLaMA-3 (Meta) — open-weight decoder-only models. Mistral, Gemma, Falcon, Qwen — further open-weight variants. All modern chat assistants (ChatGPT, Claude, Gemini) are built on decoder-only foundations.
Why no encoder?
An encoder is only necessary when there is a distinct source sequence that needs to be separately encoded before generation. When the task is formulated as prompt continuation — which covers almost everything via instruction tuning — the prompt and response live in the same sequence. The decoder's causal self-attention handles both the "understanding of the prompt" and the "generation of the response" within a single unified mechanism, eliminating the need for separate encoder-decoder machinery.
KV Cache
During autoregressive inference, the key and value tensors for each previously generated token are identical across steps — only the newest token changes. Decoder-only models exploit this by caching the K and V matrices from prior steps and reusing them, so each new token only requires a single new attention computation rather than recomputing the entire sequence. This KV cache is what makes fast decoder inference practical.

Encoder-Decoder Architecture

An encoder-decoder Transformer separates the job into two specialised components. The encoder reads the source sequence with full bidirectional self-attention and produces a set of contextual key-value representations — one per source token. The decoder then generates the target sequence one token at a time, using three sub-layers per block: causal (masked) self-attention over previously generated tokens, cross-attention into the encoder's key-value representations, and a feed-forward network.

The cross-attention is the architectural heart of conditional generation. At every decoding step, the decoder can look at the entire encoded source — not just a fixed-size summary vector, but a full set of contextualized representations. This is what solves the bottleneck problem of old seq2seq models: the decoder has direct, dynamic access to any part of the source at any generation step.

Why translation and summarization need encoder-decoder: Both tasks take a complete, known source (the original sentence or article) and produce a new, variable-length target (the translation or summary). The encoder can process the entire source bidirectionally to build the best possible representation. The decoder then generates the target token by token, attending to whichever source tokens are most relevant at each step via cross-attention — for example, attending to "chien" when generating "dog" in translation.
🔄

What encoder-decoder is good at

Sequence-to-sequence tasks where both a complete source and a variable-length target exist: machine translation (source language → target language), abstractive summarization (article → shorter summary), question generation (passage → question), dialogue (context → response), code generation from docstrings.

⚖️

The cross-attention advantage

Cross-attention gives the decoder a different set of keys and values (from the encoder) than its own queries. This means the decoder can ask "what source information is relevant to generating this next token?" — a fundamentally different computation from self-attention. Without cross-attention, the decoder cannot condition on the source; it would just generate unconstrained text.

Encoder attention
Bidirectional self-attention — same as encoder-only. Every source token attends to every other source token. Produces a rich, context-aware representation of the full source sequence.
Decoder self-attention
Causal (masked) self-attention — token at position \(t\) can only attend to positions \(\leq t\). This is required for autoregressive generation: when producing token \(t\), the model must not peek at future target tokens it hasn't generated yet.
Cross-attention
Decoder queries, encoder keys & values — the decoder's current state provides the queries; the encoder's output provides the keys and values. This allows each decoder step to dynamically retrieve the most relevant source information. Queries change at every decoding step; keys and values are fixed (the encoder runs only once).
Representative model
T5 (Text-to-Text Transfer Transformer, Raffel et al. 2020) — frames every NLP task as text-to-text. Translation becomes "translate English to German: …", summarization becomes "summarize: …". The encoder-decoder structure handles all of these uniformly. Also: the original Transformer (Vaswani et al. 2017), MarianMT (translation), BART (generation with denoising pre-training).
Why not encoder-only?
An encoder cannot generate novel sequences. It maps input tokens to output representations of the same length — it has no mechanism to produce a shorter summary or a sentence in a different language. Generation requires a decoder with an autoregressive loop and a vocabulary projection at each step.

The architectural choice in plain terms: If you want to classify or understand a fixed piece of text — reach for an encoder-only model (BERT family). If you want to chat, complete a prompt, or do anything open-ended where the output grows token by token from user input — reach for a decoder-only model (GPT/LLaMA family). If you have a well-defined source document and want to produce a structured target from it — translation, summarization, structured output from a fixed input — reach for an encoder-decoder model (T5/BART/MarianMT). The demos in Section 12 use T5 and MarianMT precisely because both translation and summarization have a complete, fixed source: the encoder reads it fully with bidirectional attention, and the decoder generates the output by dynamically querying that encoding via cross-attention — one token at a time, at each step asking "which parts of the source matter most right now?"
