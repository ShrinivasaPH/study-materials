Chapter 01

What Is Tableau & Why Use It?

Tableau is a visual analytics platform that lets you connect to data, build interactive charts, and share insights — without writing SQL or Python every time.

🔌
Connect Anywhere
Excel, Google Sheets, PostgreSQL, BigQuery, Salesforce, REST APIs — Tableau speaks them all through native connectors.
🖱️
Drag-and-Drop Analysis
Drag a field to a shelf and Tableau intelligently chooses the best mark type. No code needed for 80% of use cases.
⚡
Live vs Extract
Live connection for always-fresh data; Extract (.hyper) for blazing-fast offline analysis.
📤
Share & Embed
Publish to Tableau Server / Public / Cloud. Embed dashboards in websites or Confluence pages with one URL.

📖 Key Nomenclature

Every tool has its own vocabulary. Learn these terms and Tableau becomes much less confusing.

Term Plain English Example Icon
Dimension A categorical label — who, what, where, when City, Product Name, Customer Segment Blue Pill
Measure A numeric value you can aggregate Sales, Profit, Quantity, Discount Green Pill
Aggregation How you roll up a measure (sum, avg, count…) SUM(Sales), AVG(Profit), COUNTD(CustomerID)
Mark A single visual element on the canvas (bar, dot, line…) Each bar in a bar chart = one Mark
Row / Column Shelf Determines what goes on X and Y axes Drag [Region] → Rows to get horizontal grouping
Marks Card Controls Color, Size, Label, Detail, Tooltip of marks Drag [Profit] → Color → marks change by profit
View / Viz A single chart or table built in a worksheet One tab = one Viz
Dashboard A canvas holding multiple Vizzes + text + images Sales Overview dashboard with 4 charts
Story A slideshow of dashboards/views for presentations Quarterly review story with 6 story points
Filter Shelf Restricts which data rows appear in the view Filter to Year = 2023 only
Context Filter A filter that runs before all other filters Filter to "Top 10 Customers" using context
Table Calculation Computed from the data already in the view (not the source) RUNNING_SUM, RANK, PERCENT OF TOTAL Δ Delta
LOD Expression Computed at a fixed level regardless of view granularity { FIXED [Region]: SUM(Sales) }
Parameter A user-controllable value you can plug into formulas A slider that changes the profit threshold P
Calculated Field A new field you create with a formula Profit Margin = [Profit] / [Sales] = fx
💡
Blue vs Green pills: Tableau color-codes fields. Blue = discrete Dimensions (create headers). Green = continuous Measures (create axes). When you drag a date, it defaults to discrete (blue) by year — right-click to change to continuous for a timeline axis.

Chapter 02

The Tableau Interface

Knowing where things live cuts your learning time in half. Here's a guided tour.

1
Data Pane (Left Panel)
Lists all fields from your data source. Top half = Dimensions (blue, categorical). Bottom half = Measures (green, numeric). Drag fields from here to shelves.
2
Columns & Rows Shelves (Top)
These define your chart axes. Columns → horizontal (X-axis). Rows → vertical (Y-axis). You can stack multiple fields for nested grouping.
3
Marks Card
Controls how each mark looks: Color (encode a field), Size (encode magnitude), Label (show value), Detail (add granularity without visual encoding), Tooltip (on-hover info).
4
Filter Shelf
Drag any field here to restrict your view. Order matters: Extract filters → Data source filters → Context filters → Dimension filters → Measure filters → Table calculations.
5
Show Me Panel (Right)
Tableau's smart chart recommender. Based on what fields you have selected, it highlights valid chart types. Great for beginners — but experts often build charts manually for full control.
6
Pages Shelf
Animates your chart across a field (e.g., Year). Creates a flipbook effect — great for "growth over time" presentations. Like a built-in animation timeline.

Chapter 03

Chart Types & When to Use Them

Choosing the wrong chart confuses your audience. Here's a practical decision guide with Tableau recipes.

Bar Chart
Best for comparing values across categories.
Tableau Recipe:
Drag Category → Rows
Drag Sales → Columns
Sort descending for ranked bars.
Grouped Bar
Compare sub-categories side by side.
Tableau Recipe:
Drag Category → Columns
Drag Sales → Rows
Drag Sub-Cat → Color (Marks Card)
Stacked Bar
Show part-to-whole within each category.
Tableau Recipe:
Drag Region → Rows
Drag Sales → Columns
Drag Segment → Color
Line Chart
Trends over continuous time or ordered values.
Recipe:
Drag Order Date → Columns → set to Month
Drag Sales → Rows
Marks type → Line
Area Chart
Like a line chart but fills area — emphasises volume.
Recipe:
Same as Line Chart
Marks type → Area
Dual-Line Chart
Compare two measures over the same time axis.
Recipe:
Drag Sales → Rows, then drag Profit onto the same Y-axis → Dual Axis → Synchronise.
Pie / Donut Chart
Part-to-whole for ≤5 categories. Use sparingly.
Recipe: Drag Segment → Color, Sales → Angle. Marks → Pie.
Treemap
Hierarchical part-to-whole when you have many categories.
Recipe: Marks → Square. Drag Sales → Size, Category → Color, Sub-Category → Detail.
Scatter Plot
Correlation between two measures. Spot outliers instantly.
Recipe: Sales → Columns, Profit → Rows. Drag Customer Name → Detail for granularity.
Heatmap
Shows density or intensity across two dimensions.
Recipe: Weekday → Columns, Hour → Rows. Marks → Square. Orders → Color.
Filled Map
Choropleth: color regions by measure value.
Recipe: Double-click State → Tableau auto-builds a map. Drag Sales → Color.
Symbol Map
Plot circles sized by measure at lat/long coordinates.
Recipe: Double-click City. Marks → Circle. Drag Profit → Size.

Chapter 04

Calculated Fields & Formulas

Tableau's formula language lets you create new fields from existing ones. Think of it as Excel's formula bar — but for your entire dataset.

Arithmetic & Aggregation

📐 Profit Margin % SUM([Profit]) / SUM([Sales])
⚠️ Always aggregate before dividing — not [Profit]/[Sales] which divides row-by-row then sums.
📐 Average Order Value SUM([Sales]) / COUNTD([Order ID])
📐 Year-over-Year Growth % (SUM([Sales]) - LOOKUP(SUM([Sales]), -1)) / ABS(LOOKUP(SUM([Sales]), -1))
LOOKUP() is a table calculation that references another row relative to current.
🔑
Format as %: Right-click the calculated field → Default Properties → Number Format → Percentage.

String Functions

🔤 Concatenate Name [First Name] + " " + [Last Name]
🔤 Extract Domain from Email MID([Email], FIND([Email],"@") + 1, LEN([Email]))
FIND() returns position of "@". MID() extracts from that position to end.
🔤 Clean Leading Spaces TRIM([Product Name])
FunctionWhat it doesExample
LEFT(str, n)First n charactersLEFT([SKU], 3) → "PRD"
RIGHT(str, n)Last n charactersRIGHT([Zip], 4)
CONTAINS(str, sub)Boolean — does str contain sub?CONTAINS([Notes],"urgent")
REPLACE(str, old, new)Substitutes a substringREPLACE([Name],"Inc.","Ltd.")
UPPER / LOWERChange caseUPPER([City])
STR(num)Cast number to stringSTR([Year]) + " Report"

Date Functions

📅 Days Since Order DATEDIFF('day', [Order Date], TODAY())
📅 Quarter Label "Q" + STR(DATEPART('quarter', [Order Date])) + " " + STR(YEAR([Order Date]))
Result: "Q3 2023"
📅 First Day of Month (truncate) DATETRUNC('month', [Order Date])
FunctionWhat it does
DATEPART('part', date)Extracts integer — year, quarter, month, week, day, hour
DATENAME('part', date)Extracts string — e.g., "March", "Tuesday"
DATEADD('part', n, date)Adds n units to a date — DATEADD('month', 3, [Date])
DATEDIFF('part', start, end)Difference between two dates in the given unit
MAKEDATE(y, m, d)Constructs a date from year, month, day integers

Conditional Logic

🔀 Simple IF-ELSE IF [Profit] > 0 THEN "Profitable" ELSE "Loss" END
🔀 CASE — Customer Tier CASE [Segment]
WHEN "Consumer" THEN "Standard"
WHEN "Corporate" THEN "Priority"
WHEN "Home Office" THEN "Elite"
ELSE "Unknown"
END
🔀 IIF (inline if) — Profit Flag IIF([Profit] >= 0, "✅ Profit", "🔴 Loss")
IIF(condition, true_result, false_result) — compact one-liner.
🔀 Nested IF — Discount Tier IF [Discount] = 0 THEN "Full Price"
ELSEIF [Discount] <= 0.10 THEN "Low Discount"
ELSEIF [Discount] <= 0.25 THEN "Medium Discount"
ELSE "High Discount"
END

Table Calculations — Computed from the view

📌
What makes table calcs special: They run after Tableau has already aggregated and filtered data. They can only "see" what's in the current view — which makes them powerful for rank, running totals, and % of total.
📊 Running Total of Sales RUNNING_SUM(SUM([Sales]))
📊 % of Total SUM([Sales]) / TOTAL(SUM([Sales]))
Format as Percentage. Direction = Table (across) or Table (down).
📊 Rank (Sales) RANK(SUM([Sales]), 'desc')
Dense rank: RANK_DENSE(). Unique rank: RANK_UNIQUE(). Percentile: RANK_PERCENTILE().
📊 Month-over-Month Change SUM([Sales]) - LOOKUP(SUM([Sales]), -1)
LOOKUP(expr, offset) — offset of -1 means "previous row". Set Compute Using → Month.

🛒
Retail / E-Commerce
Which products are profitable — and which are silently draining revenue?
+
🔴 Problem
A retail manager sees total revenue of $2.3M but net profit is only $180K. Something is wrong. They need to find the product sub-categories that are causing losses despite high sales volume.
✅ Tableau Solution
Scatter Plot — Sales vs Profit by Sub-Category
Drag Sales → Columns, Profit → Rows, Sub-Category → Label + Detail. Add a Reference Line at Profit = 0. Items in the bottom-right quadrant (high sales, negative profit) are the culprits — often Furniture sub-categories with heavy discounting.
Calculated Field: Discount Impact SUM([Sales]) * AVG([Discount]) * -1
Shows how much revenue is lost per category due to discounts.

🧪 Try in Google Colab

Python — Superstore Profit Analysis 
# Run this in Google Colab — paste URL in next cell to get data
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

# Load Superstore dataset (Tableau's built-in sample)
url = "https://raw.githubusercontent.com/dsrscientist/dataset1/master/superstore.csv"
df = pd.read_csv(url, encoding='latin-1')

# Aggregate Sales & Profit by Sub-Category
grp = df.groupby('Sub-Category')[['Sales','Profit']].sum().reset_index()
grp['Margin'] = grp['Profit'] / grp['Sales'] * 100

# ── Scatter Plot ──
fig, ax = plt.subplots(figsize=(10, 6))
colors = ['#C94F3A' if p < 0 else '#2A7A4B' for p in grp['Profit']]
ax.scatter(grp['Sales'], grp['Profit'], c=colors, s=120, zorder=5)
ax.axhline(0, color='gray', linewidth=1, linestyle='--')

for _, row in grp.iterrows():
    ax.annotate(row['Sub-Category'], (row['Sales'], row['Profit']),
                textcoords="offset points", xytext=(5,4), fontsize=8)

ax.set_title("Sales vs Profit by Sub-Category", fontsize=14)
ax.set_xlabel("Total Sales ($)"); ax.set_ylabel("Total Profit ($)")
plt.tight_layout(); plt.show()

# Print loss-makers
print("\n🔴 Loss-making Sub-Categories:")
print(grp[grp['Profit'] < 0][['Sub-Category','Sales','Profit','Margin']]
      .sort_values('Profit').to_string(index=False))

Supply Chain / Logistics
Are our delivery times getting better or worse over time?
+
🔴 Problem
The ops team is receiving customer complaints about late deliveries. They need to track average shipping days per month, by ship mode, to find where the bottleneck is.
✅ Tableau Solution
Line Chart with Ship Mode as a colour dimension
Create calculated field Days to Ship = DATEDIFF('day', [Order Date], [Ship Date]). Drag Order Date (Monthly) → Columns, AVG(Days to Ship) → Rows, Ship Mode → Color.
Calculated Field: Late Flag IF [Days to Ship] > 5 THEN "Late" ELSE "On Time" END
Use this on Color in Marks card to highlight late shipments instantly.
Python — Shipping Trend Analysis 
import pandas as pd
import matplotlib.pyplot as plt

url = "https://raw.githubusercontent.com/dsrscientist/dataset1/master/superstore.csv"
df = pd.read_csv(url, encoding='latin-1')
df['Order Date'] = pd.to_datetime(df['Order Date'])
df['Ship Date']  = pd.to_datetime(df['Ship Date'])

# Calculated field equivalent
df['Days to Ship'] = (df['Ship Date'] - df['Order Date']).dt.days

# Monthly average by Ship Mode
df['Month'] = df['Order Date'].dt.to_period('M')
pivot = df.groupby(['Month', 'Ship Mode'])['Days to Ship'].mean().unstack()

fig, ax = plt.subplots(figsize=(12, 5))
for col in pivot.columns:
    ax.plot(pivot.index.astype(str), pivot[col], marker='o', markersize=3, label=col)

ax.axhline(5, color='#C94F3A', linestyle='--', label='SLA = 5 days')
ax.set_title("Avg Days to Ship by Month & Ship Mode")
ax.set_xticks(ax.get_xticks()[::6])  # show every 6th month
ax.legend(); plt.tight_layout(); plt.show()

# Late order %
late_pct = (df['Days to Ship'] > 5).mean() * 100
print(ff"Overall late delivery rate: {late_pct:.1f}%")

HR / People Analytics
Which department has the highest employee attrition — and why?
+
🔴 Problem
HR notices 18% annual attrition but doesn't know if it's concentrated in specific departments, age groups, or salary bands. Exit interviews are inconsistent.
✅ Tableau Solution
Dashboard with 3 views:
1. Bar chart: Attrition % by Department (calculated field).
2. Heatmap: Age group vs. Job Role coloured by attrition rate.
3. Box plot: Monthly Income by Attrition (Yes/No).
Calculated Field: Attrition Rate % SUM(IF [Attrition] = "Yes" THEN 1 ELSE 0 END) / COUNT([Employee ID])
Python — HR Attrition Analysis 
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

# IBM HR Attrition dataset (public)
url = "https://raw.githubusercontent.com/dsrscientist/dataset1/master/HR_attrition.csv"
df = pd.read_csv(url)
df['Left'] = (df['Attrition'] == 'Yes').astype(int)

# ── Attrition % by Department ──

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

axes[0].barh(dept.index, dept.values, color=['#C94F3A','#E5A97A','#2A7A4B'])

axes[0].set_title("Attrition % by Department")

axes[1].set_title("Monthly Income by Attrition")

Finance / Sales
Build a KPI Dashboard — Revenue, Target, and Variance at a glance
+
🔴 Problem
The CFO wants a single-screen view of: actual revenue vs target by month, which regions are above/below target, and year-to-date trend.
✅ Tableau Solution
3-panel dashboard:
1. BANs (Big Ass Numbers) — text mark showing YTD Sales, % vs Target, MoM change.
2. Bullet Chart — Actual vs Target bar (use dual axis: bar for actual, reference line for target).
3. Map — Region coloured by % to target (calculated field below).
% to Target SUM([Sales]) / SUM([Target]) - 1
Format as % with +/- signs. Color red if < 0, green if >= 0 using a calculated color field.
YTD Sales (using FIXED LOD) { FIXED YEAR([Order Date]) : SUM([Sales]) }
Returns total sales for the whole year regardless of month-level granularity in the view.
Python — KPI Dashboard with Matplotlib 
import pandas as pd, numpy as np
import matplotlib.pyplot as plt

url = "https://raw.githubusercontent.com/dsrscientist/dataset1/master/superstore.csv"
df = pd.read_csv(url, encoding='latin-1')
df['Order Date'] = pd.to_datetime(df['Order Date'])

# Monthly Sales
monthly = (df[df['Order Date'].dt.year == 2023]
           .groupby(df['Order Date'].dt.month)['Sales'].sum())

# Simulate a target (110% of 2022 average per month)
target = monthly.mean() * 1.10

fig, axes = plt.subplots(1, 2, figsize=(14, 5))
months = ['Jan','Feb','Mar','Apr','May','Jun',
          'Jul','Aug','Sep','Oct','Nov','Dec']

# Bullet chart equivalent
colors = ['#2A7A4B' if v >= target else '#C94F3A' for v in monthly]
axes[0].bar(months[:len(monthly)], monthly, color=colors, alpha=0.85)
axes[0].axhline(target, color='#1A1208', linewidth=2, linestyle='--', label=f'Target: ${target:,.0f}')
axes[0].legend(); axes[0].set_title("Monthly Sales vs Target (2023)")

# Running total
cumulative = monthly.cumsum()
axes[1].plot(months[:len(monthly)], cumulative, marker='o', color='#0E7C7B', linewidth=2)
axes[1].fill_between(months[:len(monthly)], cumulative, alpha=0.15, color='#0E7C7B')
axes[1].set_title("YTD Cumulative Sales (2023)")
plt.tight_layout(); plt.show()

Chapter 06

Advanced Features

🎛️ Parameters
A Parameter is a user-adjustable variable. Create via right-click in Data Pane → Create Parameter.

Use case: "Top N Products by Sales" — create an integer parameter [Top N], then use in a calculated field: RANK(SUM([Sales])) <= [Top N]. Filter to TRUE.
🔗 Relationships vs Joins
Since Tableau 2020.2, Relationships are the default way to link tables. Unlike Joins, they don't duplicate rows — they're context-aware and preserve correct aggregation across granularities. Use Joins only when you explicitly need to combine columns at row level.
🔍 Sets & Groups
Groups: Manually bundle members (e.g., combine "NY" + "NJ" → "Northeast"). Right-click a dimension member.

Sets: Dynamic or static subsets defined by a condition (e.g., "Customers with > $5000 lifetime spend"). Sets give you IN/OUT binary fields to use in calculations.
📏 Reference Lines & Bands
Right-click any axis → Add Reference Line. Options: constant value, average, median, custom expression, or a Parameter value (dynamic threshold). Reference Bands shade a range — useful for target zones or confidence intervals.

Chapter 07

LOD Expressions — Level of Detail

LOD Expressions are one of Tableau's most powerful — and most misunderstood — features. They let you compute at a different granularity than the current view.

🧠
Mental model: Normal calculated fields compute at the view's level of detail. LOD expressions say "ignore the view — compute at THIS level instead." Think of it like a subquery in SQL.
FIXED
Computes at the specified dimension, regardless of filters (except Context filters)
Syntax { FIXED [dim1], [dim2] : AGG([measure]) }
Example: Customer's first order date
{ FIXED [Customer ID] : MIN([Order Date]) }
INCLUDE
Adds dimensions to the view-level computation (more granular)
Syntax { INCLUDE [dim] : AGG([measure]) }
Example: Avg orders per customer per region
{ INCLUDE [Customer ID] : COUNT([Order ID]) }
EXCLUDE
Removes dimensions from the computation (less granular)
Syntax { EXCLUDE [dim] : AGG([measure]) }
Example: % of category total on a sub-category view
SUM([Sales]) / { EXCLUDE [Sub-Category]: SUM([Sales]) }

Practical LOD: Cohort Analysis

Step 1 — First Purchase Year (FIXED) { FIXED [Customer ID] : MIN(YEAR([Order Date])) }
Step 2 — Years Since Acquisition YEAR([Order Date]) - [First Purchase Year]

Now you can build a cohort retention table: Rows = First Purchase Year, Columns = Years Since Acquisition, Values = COUNT(DISTINCT Customers).

Python — Cohort Analysis equivalent 
import pandas as pd, seaborn as sns, matplotlib.pyplot as plt

url = "https://raw.githubusercontent.com/dsrscientist/dataset1/master/superstore.csv"
df = pd.read_csv(url, encoding='latin-1')
df['Order Date'] = pd.to_datetime(df['Order Date'])

# FIXED LOD equivalent: first order year per customer
first_yr = df.groupby('Customer ID')['Order Date'].min().dt.year
first_yr.name = 'Cohort Year'
df = df.join(first_yr, on='Customer ID')
df['Order Year'] = df['Order Date'].dt.year
df['Years Since'] = df['Order Year'] - df['Cohort Year']

# Cohort table: unique customers per cohort per year-since
cohort = (df.groupby(['Cohort Year','Years Since'])['Customer ID']
            .nunique().unstack())

# Normalise to retention %
cohort_pct = cohort.div(cohort.iloc[:,0], axis=0) * 100

fig, ax = plt.subplots(figsize=(10,5))
sns.heatmap(cohort_pct, annot=True, fmt='.0f', cmap='YlGn',
            linewidths=0.5, ax=ax)
ax.set_title("Customer Cohort Retention % (Tableau LOD equivalent)")
plt.tight_layout(); plt.show()
