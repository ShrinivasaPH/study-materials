What is Machine Learning?

Algorithms that improve through experience — without being explicitly programmed for every case.

Arthur Samuel coined the term in 1959. The idea: instead of writing rules by hand, we give a computer examples, and it figures out the rules itself.

The Core Idea

Find a function f such that f(X) ≈ y — mapping input features X to outputs y, learned from data rather than hand-crafted.

The ML Workflow

1
Collect & explore data
Gather labelled examples. Understand distributions, missing values, correlations (EDA).
2
Feature engineering
Transform raw data into meaningful numerical representations. Normalise, encode, create new features.
3
Choose & train a model
Select an algorithm, define a loss function, optimise parameters on the training set.
4
Evaluate & tune
Measure generalisation on a held-out validation set. Tune hyperparameters. Check for over/underfitting.
5
Deploy & monitor
Serve predictions in production. Watch for data drift and retrain as the world changes.

Underfitting

Model is too simple — high bias. It misses patterns in training data and generalises poorly.

Fix: add more features, use a more complex model, reduce regularisation.

Overfitting

Model memorises training data — high variance. Great on train, terrible on test.

Fix: more data, regularisation (L1/L2/Dropout), simpler model, cross-validation.

Supervised Learning

Regression

Predict a continuous number — house prices, temperatures, stock returns.

Linear Regression

The simplest model: assume the relationship between inputs and output is a straight line (or hyperplane in multiple dimensions). Fit a line through data by minimising the sum of squared errors.

Model hypothesis
ŷ = θ₀ + θ₁x₁ + θ₂x₂ + … + θₙxₙ = θᵀx
ŷ predicted value θ₀ bias / intercept θᵢ weight for feature i xᵢ feature i
Mean Squared Error — Loss function
J(θ) = (1/2m) Σᵢ (ŷ⁽ⁱ⁾ − y⁽ⁱ⁾)²
m number of examples ŷ⁽ⁱ⁾ prediction for example i y⁽ⁱ⁾ true label for example i
Gradient Descent Update Rule
θⱼ ← θⱼ − α · (∂J/∂θⱼ) = θⱼ − (α/m) Σᵢ (ŷ⁽ⁱ⁾ − y⁽ⁱ⁾) · xⱼ⁽ⁱ⁾
α learning rate ∂J/∂θⱼ gradient (slope of loss surface)

Regularisation

Adding a penalty term to the loss prevents overfitting by keeping weights small.

Ridge (L2)

J + λ Σθⱼ²

Shrinks all weights smoothly towards zero. Never makes them exactly zero. Good when all features matter a little.

Lasso (L1)

J + λ Σ|θⱼ|

Drives unimportant weights to exactly zero — built-in feature selection. Great for high-dimensional sparse problems.

Python Code

import numpy as np
from sklearn.linear_model import LinearRegression, Ridge, Lasso
from sklearn.preprocessing import PolynomialFeatures
from sklearn.pipeline import Pipeline

# --- Simple linear regression from scratch ---
class LinearReg:
    def __init__(self, lr=0.01, epochs=1000):
        self.lr, self.epochs = lr, epochs

    def fit(self, X, y):
        m, n = X.shape
        self.theta = np.zeros(n + 1)          # weights + bias
        Xb = np.c_[np.ones(m), X]              # add bias column
        for _ in range(self.epochs):
            y_hat = Xb @ self.theta
            grad = (Xb.T @ (y_hat - y)) / m   # ∂J/∂θ
            self.theta -= self.lr * grad        # gradient descent step
        return self

    def predict(self, X):
        return np.c_[np.ones(len(X)), X] @ self.theta

# --- Sklearn pipeline with polynomial features + Ridge ---
model = Pipeline([
    ('poly', PolynomialFeatures(degree=2)),
    ('ridge', Ridge(alpha=1.0))
])
model.fit(X_train, y_train)
print(f"R² = {model.score(X_test, y_test):.4f}")

Key Metrics

R²

R-Squared

Fraction of variance explained. 1 = perfect, 0 = no better than mean, negative = worse than mean.

MAE

Mean Absolute Error

Average absolute difference. Same units as target. Robust to outliers.

RMSE

Root Mean Sq. Error

Same units as target. Penalises large errors more than MAE.

Supervised Learning

Classification

Predict a discrete category — spam/not-spam, disease/healthy, which digit 0–9.

Logistic Regression

Despite the name, it's a classifier. We squash the linear output through a sigmoid to get a probability between 0 and 1.

Sigmoid activation
σ(z) = 1 / (1 + e⁻ᶻ)    where    z = θᵀx
σ(z) ∈ (0, 1) — probability of class 1 e Euler's number ≈ 2.718
Binary Cross-Entropy Loss
J(θ) = −(1/m) Σᵢ [ y⁽ⁱ⁾ log(ŷ⁽ⁱ⁾) + (1−y⁽ⁱ⁾) log(1−ŷ⁽ⁱ⁾) ]
log(ŷ) penalises confident wrong predictions harshly

Other Algorithms

Decision Tree

Recursively splits data by asking yes/no questions on features. Highly interpretable. Prone to overfit alone.

Split criterion: max Information Gain or min Gini Impurity

SVM

Find the hyperplane with maximum margin between classes. Kernel trick maps data to higher dimensions for non-linear boundaries.

min ½‖w‖² s.t. yᵢ(wᵀxᵢ+b) ≥ 1

k-Nearest Neighbours

Predict by majority vote of the k closest training examples. No explicit training phase — just store and look up.

distance(a,b) = √Σ(aᵢ−bᵢ)²

Naive Bayes

Applies Bayes' theorem assuming features are conditionally independent given the class. Blazing fast. Great for text.

P(y|X) ∝ P(y) Πᵢ P(xᵢ|y)

Unsupervised Learning

Clustering

Find natural groups in data — without any labels. The algorithm discovers structure on its own.

k-Means Clustering

Partition n points into k clusters by minimising within-cluster variance. Simple, fast, widely used.

k-Means Objective (Inertia)
J = Σₖ Σᵢ∈Cₖ ‖xᵢ − μₖ‖²
Cₖ set of points in cluster k μₖ centroid (mean) of cluster k ‖·‖² squared Euclidean distance
1
Initialise
Randomly pick k points as initial centroids (k-means++ picks them far apart for better initialisation).
2
Assign
Assign each data point to the nearest centroid. This creates k clusters.
3
Update
Move each centroid to the mean position of all points assigned to it.
4
Repeat
Alternate Assign ↔ Update until centroids stop moving (convergence). Always converges in finite steps.

Density-based. Finds arbitrarily shaped clusters and marks outliers as noise. No need to specify k upfront.

Hierarchical

Builds a tree (dendrogram) of nested clusters. Cut the tree at any height to get any number of clusters.

No k neededDendrogram

Gaussian Mixture

Soft assignments — each point has a probability of belonging to each cluster. Fitted via EM algorithm.

from sklearn.cluster import KMeans, DBSCAN
from sklearn.metrics import silhouette_score
import numpy as np

# k-Means with elbow method
inertias = []
K_range = range(1, 11)
for k in K_range:
    km = KMeans(n_clusters=k, init='k-means++', n_init=10, random_state=42)
    km.fit(X)
    inertias.append(km.inertia_)

# Best k via silhouette score (higher = better separated clusters)
best_k, best_s = 2, -1
for k in range(2, 11):
    labels = KMeans(n_clusters=k, random_state=42).fit_predict(X)
    s = silhouette_score(X, labels)
    if s > best_s: best_k, best_s = k, s

print(f"Best k = {best_k}, silhouette = {best_s:.3f}")

# DBSCAN — good for irregular shapes
db = DBSCAN(eps=0.5, min_samples=5).fit(X)
labels = db.labels_                    # -1 = noise
n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
Meta-Learning

Ensemble Methods

Combine many weak learners into one powerful predictor. The wisdom of crowds — for models.

A single decision tree is unstable (high variance). A single linear model is rigid (high bias). Ensemble methods fix this by combining many models. The key insight: errors that are uncorrelated tend to cancel out.

Bias–Variance Decomposition

Expected Error = Bias² + Variance + Irreducible Noise

Bagging reduces variance. Boosting reduces bias. Stacking can reduce both.

1. Bagging — Bootstrap Aggregating

Train many models on different bootstrap samples (random subsets with replacement) of the data. Average their predictions. Models are independent — they reduce variance.

Bagging prediction
ŷ_bag = (1/B) Σᵦ hᵦ(x)   [regression]    or    majority vote   [classification]
B number of base models hᵦ b-th base learner trained on bootstrap sample b

Random Forest

Bagging + extra randomness: at each split, only a random subset of √n features is considered. This de-correlates trees further, giving a huge variance reduction. One of the most reliable general-purpose algorithms.

features per split ≈ √n (classification) or n/3 (regression)

2. Boosting — Sequential Error Correction

Train models sequentially, each one focusing on the mistakes of the previous. Combines many weak learners (stumps) into a strong one. Reduces both bias and variance.

AdaBoost

Re-weights training examples after each round. Misclassified points get higher weight — next learner pays more attention to hard cases.

ŷ = sign(Σᵦ αᵦ hᵦ(x))

Gradient Boosting

Each new tree fits the residuals (pseudo-residuals = negative gradient of loss) of the current ensemble. Generalises boosting to any differentiable loss.

Fₜ(x) = Fₜ₋₁(x) + α · hₜ(x)

XGBoost / LightGBM / CatBoost

Industrial-strength gradient boosting libraries. They add regularisation (L1/L2 on tree weights), second-order gradient approximations, GPU support, and handling of missing values. XGBoost won dozens of Kaggle competitions; LightGBM is faster with very large datasets.

Tabular SOTAKaggle winnerProduction ready

3. Stacking

Train a meta-learner (level-1 model) on the out-of-fold predictions of base models (level-0). The meta-learner learns how to best combine the base models.

Stacking — two-level structure
Level 0: h₁(x), h₂(x), …, hₙ(x) — diverse base models
Level 1: g([h₁(x), h₂(x), …, hₙ(x)]) — meta-learner

Sequential Data

Time Series

Data ordered in time. The past predicts the future — but the relationship is dynamic.

Components of a Time Series

📈

Trend

Long-term direction — upward, downward, or flat. E.g. global temperature rising over decades.

🔄

Seasonality

Regular repeating patterns tied to calendar. E.g. retail sales surge every December.

〰️

Cyclicality

Irregular, longer-period fluctuations. E.g. economic boom-bust cycles.

⚡

Residual / Noise

The unexplained variation after removing trend and seasonality. Ideally white noise.

ARIMA — The Classical Model

ARIMA(p, d, q) combines three ideas: autoregression, integration (differencing), and moving average.

ARIMA(p, d, q) model
Φ(B)(1−B)ᵈ yₜ = Θ(B) εₜ
AR(p): yₜ = c + φ₁yₜ₋₁ + φ₂yₜ₋₂ + … + φₚyₜ₋ₚ + εₜ
MA(q): yₜ = μ + εₜ + θ₁εₜ₋₁ + θ₂εₜ₋₂ + … + θqεₜ₋q
I(d): difference the series d times to make it stationary
p AR order — lags of yₜ d differencing order q MA order — lags of εₜ B backshift operator: Byₜ = yₜ₋₁ εₜ white noise residual

Modern Approaches

Prophet (Meta)

Additive model: trend + seasonality + holidays. Robust to missing data and outliers. Excellent for business forecasting.

y(t) = g(t) + s(t) + h(t) + ε(t)

LSTM / GRU

Recurrent neural networks with gating mechanisms. Learn long-range dependencies. Powerful but data-hungry and slow to train.

Deep LearningLong horizon

Temporal Fusion Transformer

Attention-based architecture for multi-horizon forecasting with interpretable temporal patterns and feature importance.

TransformerMulti-horizon

N-BEATS / N-HiTS

Pure MLP architectures that decompose the forecast into trend/seasonality blocks. State-of-the-art on many benchmarks.

No recurrenceInterpretable
Forecast evaluation metrics
MAE = (1/n) Σ|yₜ − ŷₜ|
RMSE = √[(1/n) Σ(yₜ − ŷₜ)²]
MAPE = (1/n) Σ|yₜ − ŷₜ|/|yₜ| × 100%
SMAPE = (2/n) Σ|yₜ−ŷₜ|/(|yₜ|+|ŷₜ|)
