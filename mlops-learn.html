02 — Process

ML Model Lifecycle

A model's life begins at conception and never truly ends — it continuously loops through evaluation and retraining.

1
Problem Definition
Define the ML task (classification, regression, ranking). Establish success metrics aligned with business KPIs.
2
Data Collection
Gather labelled and unlabelled data from multiple sources. Ensure coverage of all real-world scenarios.
3
EDA & Preparation
Exploratory analysis, missing value handling, outlier detection, and schema validation.
4
Feature Engineering
Transform raw data into numeric representations. Encode, normalise, and aggregate features.
5
Training & Tuning
Select algorithm, optimise hyperparameters (HPO), track experiments with tools like MLflow or W&B.
6
Evaluation
Offline metrics, slice-based analysis, bias/fairness auditing, shadow deployment testing.
7
Deployment
Package as a container, deploy to serving infrastructure. Choose canary, blue-green, or shadow strategy.
8
Monitoring
Track prediction distribution, input drift, latency, error rate, and business metrics continuously.
9
Retrain / Retire
Trigger retraining on drift or scheduled cadence. Retire models when replaced or deprecated.

Key Model Metrics

Acc
Accuracy
Fraction of correct predictions. Misleading on imbalanced data.
= (TP + TN) / N
P
Precision
Of all predicted positives, how many are truly positive?
= TP / (TP + FP)
R
Recall
Of all actual positives, how many did we find?
= TP / (TP + FN)
F₁
F1 Score
Harmonic mean of precision and recall. Balances both.
= 2·P·R / (P + R)
MAE
Mean Abs Error
Average absolute difference between predicted and actual values.
= Σ|yᵢ − ŷᵢ| / n
RMSE
Root MSE
Penalises large errors more than MAE. Sensitive to outliers.
= √(Σ(y−ŷ)²/n)
04 — Deep Dive

Key Concepts Explained

Click any concept to expand the full explanation.

Data & Concept Drift monitoring
▾

Data drift (a.k.a. covariate shift) occurs when the statistical distribution of input features X changes between training and production. Example: a loan model trained on pre-COVID income data faces very different distributions post-COVID.

Concept drift is more subtle — the relationship between X and y itself changes. A spam filter may need retraining as spammers adapt tactics, even if email distributions look similar.

Detection methods: Population Stability Index (PSI), Kolmogorov-Smirnov test, Jensen-Shannon divergence. A PSI > 0.2 typically triggers retraining.

Training-Serving Skew data
▾

Skew happens when the features computed during training differ from those computed at serving time. A classic cause: using mean(user_purchases_last_30d) in training via a historical batch join, but computing it differently in production code.

The solution is a Feature Store — a shared layer that ensures the exact same transformation logic runs offline and online. Feast, Tecton, and Hopsworks are popular options.

Experiment Tracking training
▾

Every training run generates parameters, metrics, code versions, and artefacts. Without tracking, reproducing your best model becomes impossible. Experiment tracking tools log: hyperparameters (learning rate, batch size), metrics (loss, AUC), code commits, data versions, and output artefacts.

MLflow uses runs grouped into experiments. Each run captures: params, metrics (logged per step), tags, and artefacts (model files). Weights & Biases (W&B) adds collaborative dashboards and sweep-based HPO on top of this.

Deployment Strategies deploy
▾

Blue-Green: Run two identical environments (blue = old, green = new). Switch traffic instantly. Easy rollback by redirecting back to blue.

Canary: Route a small percentage (e.g. 5%) of traffic to the new model, monitor metrics, then gradually increase. Safer for high-traffic systems.

Shadow (Dark launch): New model receives all requests but its predictions are not served to users. Used to compare output distributions before going live — zero risk.

A/B Testing: Split users into control (model A) and treatment (model B) groups. Measure business outcomes (CTR, conversion) to determine the winner statistically.

Hyperparameter Optimisation (HPO) training
▾

Hyperparameters (η, batch size, tree depth, dropout rate) are not learned from data — they're set before training. HPO automates their search.

Grid Search: Exhaustively try every combination. Expensive. Random Search: Sample randomly — often finds good results 60× faster than grid for the same budget. Bayesian Optimisation: Model the loss surface with a surrogate (Gaussian Process), suggest next point intelligently. Optuna, Ray Tune, and W&B Sweeps all implement this.

Model Serving Patterns deploy
▾

Online (real-time) serving: Model runs behind a REST or gRPC endpoint. Predictions returned synchronously in <100ms. Used for recommendations, fraud detection, chatbots.

Batch serving: Run predictions offline on a large dataset, store results in a database. Users query pre-computed scores. Simpler and cheaper. Used for churn prediction, marketing segmentation.

Streaming serving: Model runs on a stream (Kafka, Kinesis). Events trigger inference as they arrive. Used for IoT anomaly detection, real-time fraud.

Model packaging formats: ONNX (cross-framework), TorchScript (PyTorch), SavedModel (TensorFlow), Pickle/Joblib (sklearn). TorchScript and ONNX remove the Python runtime dependency for production.

Data Versioning & Lineage data
▾

Reproducing a model result requires knowing exactly which data was used. Data versioning tools (DVC, Delta Lake, Iceberg) snapshot datasets and track how they evolve.

Lineage maps: raw source → transformation → training dataset → model → prediction. This audit trail is critical for debugging ("why did the model behaviour change?") and regulation (GDPR right-to-explanation).

ML Pipeline Orchestration engineering
▾

Orchestrators schedule, retry, and monitor multi-step pipelines expressed as DAGs (Directed Acyclic Graphs). Each node is a task (data load, feature compute, train, evaluate, push) with defined inputs and outputs.

Popular orchestrators: Apache Airflow (mature, Python-based DAGs, widely used), Prefect (better error handling and dynamic flows), ZenML / Kubeflow Pipelines (ML-native, runs on Kubernetes), Metaflow (Netflix, great for data scientists).

import mlflow
import mlflow.sklearn
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import accuracy_score, f1_score

# Set tracking URI (local or remote server)
mlflow.set_tracking_uri("http://mlflow-server:5000")
mlflow.set_experiment("churn-prediction-v2")

with mlflow.start_run(run_name="rf-baseline") as run:

    # Log hyperparameters
    params = {"n_estimators": 200, "max_depth": 8, "random_state": 42}
    mlflow.log_params(params)

    # Train model
    model = RandomForestClassifier(**params)
    model.fit(X_train, y_train)

    # Evaluate
    preds = model.predict(X_val)
    metrics = {
        "accuracy": accuracy_score(y_val, preds),
        "f1": f1_score(y_val, preds, average="weighted")
    }
    mlflow.log_metrics(metrics)

    # Log model with signature (schema enforcement)
    signature = mlflow.models.infer_signature(X_train, preds)
    mlflow.sklearn.log_model(
        model, "model",
        signature=signature,
        registered_model_name="ChurnClassifier"
    )

    print(f"Run ID: {run.info.run_id}")
    print(f"AUC: {metrics['f1']:.4f}")

from zenml import step, pipeline
from zenml.integrations.mlflow.flavors import mlflow_experiment_tracker_settings
import pandas as pd

# Each @step is a node in the pipeline DAG

@step
def load_data() -> pd.DataFrame:
    """Ingest and validate raw training data."""
    df = pd.read_parquet("s3://my-bucket/data/train.parquet")
    assert df.isnull().sum().max() == 0, "Nulls detected!"
    return df

@step

    """Transform raw data into model-ready features."""
    df["tenure_months"] = df["tenure_days"] // 30
    df["spend_per_month"] = df["total_spend"] / (df["tenure_months"] + 1)
    return df

@step(experiment_tracker="mlflow_tracker")
def train_model(df: pd.DataFrame):
    """Train, evaluate, and register the model."""
    # ... training logic, auto-logged to MLflow
    pass

# Wire steps into a pipeline
@pipeline
def churn_training_pipeline():
    df = load_data()
    features = feature_engineer(df)
    train_model(features)

if __name__ == "__main__":
    churn_training_pipeline()
python — serve.py (FastAPI + MLflow) 
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import mlflow.pyfunc
import pandas as pd
import time

app = FastAPI(title="Churn Prediction API", version="2.1.0")

# Load model once at startup (not per request)
MODEL_URI = "models:/ChurnClassifier/Production"
model = mlflow.pyfunc.load_model(MODEL_URI)

class PredictRequest(BaseModel):
    tenure_months: int
    spend_per_month: float
    num_products: int
    is_active: bool

class PredictResponse(BaseModel):
    churn_probability: float
    prediction: str
    latency_ms: float

@app.post("/predict", response_model=PredictResponse)
def predict(req: PredictRequest):
    t0 = time.time()
    df = pd.DataFrame([req.dict()])

    try:
        proba = model.predict(df)[0]
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

    return PredictResponse(
        churn_probability=round(float(proba), 4),
        prediction="churn" if proba > 0.5 else "retain",
        latency_ms=round((time.time() - t0) * 1000, 2)
    )

@app.get("/health")
def health(): return {"status": "ok", "model": MODEL_URI}
python — drift_detection.py (Evidently) 
from evidently import Report
from evidently.metric_preset import DataDriftPreset, DataQualityPreset
from evidently.metrics import ColumnDriftMetric
import pandas as pd

# Reference: training data. Current: last 7 days of production data
reference = pd.read_parquet("data/train_reference.parquet")
current = pd.read_parquet("data/production_last7d.parquet")

# Build a drift report
report = Report(metrics=[
    DataDriftPreset(),
    DataQualityPreset(),
    ColumnDriftMetric(column_name="spend_per_month"),
])

report.run(reference_data=reference, current_data=current)

# Export HTML report + extract PSI for alerting
report.save_html("drift_report.html")
result = report.as_dict()

psi = result["metrics"][0]["result"]["dataset_drift"]

if psi:
    print("⚠ Dataset drift detected — triggering retraining pipeline")
    # Trigger Airflow/Prefect DAG or GitHub Actions workflow
    trigger_retraining(reason="data_drift")
