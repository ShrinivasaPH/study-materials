Chapter 01

What Is Prompt Engineering?

Prompt engineering is the practice of designing and refining inputs to a large language model (LLM) to produce a desired output. It is the interface between human intent and model capability — a discipline that blends linguistics, logic, domain expertise, and empirical testing.

Think of an LLM as a very powerful function. The quality of its output is proportional to the quality of its input.

Core Principle — Input/Output Mapping
Output = f(Prompt, Model, Temperature, Context)
f = the LLM's internal function (parameters, weights)  ·  Prompt = your textual instruction  ·  Temperature ∈ [0, 2] = creativity/randomness  ·  Context = conversation history, injected documents

Why It Matters

  • Consistency: Identical tasks, identical format — every time.
  • Accuracy: Reduce hallucination by anchoring the model with precise constraints.
  • Efficiency: Fewer back-and-forth iterations. First-shot quality outputs.
  • Safety: Guard-rails that prevent misuse or off-topic drift.

Mental Model

An LLM is a next-token predictor trained on a vast corpus of text. When you write a prompt, you are sculpting a probability distribution over possible continuations. A well-engineered prompt narrows that distribution toward the desired output.

Token Prediction
P(tokenn+1 | token1..n, prompt)
The model assigns probability to every possible next token. Your prompt shifts this distribution.
💡 Key Insight
Prompt engineering is not about "tricking" the model. It is about communicating clearly — the same skill that makes a good manager, teacher, or technical writer.
Chapter 02

Anatomy of a Prompt

Most effective prompts are composed of up to six distinct components. You don't always need all of them — but knowing each one lets you consciously choose when to include it.

Full Prompt Formula
Prompt = [Role] + [Task] + [Context] + [Format] + [Constraints] + [Examples]
[ ] = optional component  ·  order can vary  ·  Task is the only truly required element
Role optional
Who should the model act as? Setting a persona primes the model's vocabulary, tone, and domain knowledge. Example: You are a senior data analyst at a Fortune 500 company.
Task required
What should the model do? Start with a clear action verb. Be specific. Example: Summarize the following customer review into three bullet points.
Context optional
What background does the model need? Inject data, documents, or situational info the model can't infer. Example: The user is a first-year medical student with no coding background.
Format optional
How should the output look? Specify structure, length, tone, or medium. Example: Respond in a JSON object with keys: title, summary, sentiment.
Constraints optional
What are the guardrails? Define what to avoid, word limits, language, etc. Example: Do not use technical jargon. Keep your answer under 100 words.
Examples optional
Show, don't just tell. Concrete examples are the fastest way to align model output with your expectation. This is called few-shot prompting. Example: Input: "I hate this product!" → Output: {"sentiment":"negative"}

Chapter 03

Core Prompting Techniques

Each technique is a pattern that solves a specific class of problem. Master each one, then know when to combine them.

01
Zero-Shot Prompting
No examples given
Ask the model to perform a task it has never seen in this session. Relies entirely on pre-trained knowledge. Best for simple, well-defined tasks.
02
Few-Shot Prompting
k examples (k ≥ 1)
Provide 1–8 input/output examples before the real task. Dramatically improves format adherence and accuracy for specialized outputs.
03
Chain of Thought
Step-by-step reasoning
Instruct the model to "think aloud" before giving the final answer. Reduces errors on multi-step math, logic, and planning tasks.
04
Role Prompting
Persona assignment
Assign a professional identity to prime domain vocabulary, tone, and response depth. "You are a…" is your scalpel.
05
Self-Consistency
Sample, vote, aggregate
Generate the same prompt N times with temperature>0, then take the majority answer. Increases reliability on reasoning-heavy tasks.
06
ReAct Prompting
Reason + Act
Interleave reasoning steps with external actions (search, code execution, API calls). Foundation of modern AI agents and tool-use architectures.
07
Instruction Chaining
Multi-step pipelines
Break complex tasks into sequential sub-prompts, where the output of one becomes the input of the next. Reduces single-prompt overload.
08
Constraint-Based
Negative + positive rules
Define both what to DO and what NOT to do. Negative constraints often do more work: "Do not include personal opinions."
09
Tree of Thought
Branching exploration
Ask the model to explore multiple solution paths simultaneously, evaluate each, and select the best. Inspired by deliberate problem-solving.

Zero-Shot: Direct Instructions Only

No examples needed. The model relies on instruction-tuning and pre-training.

✗ Weak Zero-Shot
Translate this.
⚠ Translate to what language? What text? Ambiguous.
✓ Strong Zero-Shot
Translate the following English sentence to French. Return only the translated text, nothing else. Text: "The quarterly report shows a 12% increase in revenue."
✓ Le rapport trimestriel montre une augmentation de 12% des revenus.
Python
import anthropic

client = anthropic.Anthropic()

prompt = """Translate the following English sentence to French.
Return only the translated text, nothing else.

Text: The quarterly report shows a 12% increase in revenue."""

message = client.messages.create(
    model="claude-opus-4-5",
    max_tokens=256,
    messages=[{"role": "user", "content": prompt}]
)
print(message.content[0].text)

Few-Shot: Learning by Example

Provide input→output pairs that demonstrate the pattern. The model generalizes from these to your actual task.

Few-Shot Structure
Prompt = Instruction + [Example₁ (input→output)] + ... + [Exampleₖ] + Actual_Input
Optimal k is usually 3–5. More than 8 rarely helps and wastes context tokens.
Few-Shot Prompt Template
# Task: Extract product sentiment from reviews
# Format: JSON {"product": ..., "sentiment": positive|negative|neutral, "score": 1-5}

### Examples ###
Review: "This blender is amazing! Makes perfect smoothies every morning."
Output: {"product": "blender", "sentiment": "positive", "score": 5}

Review: "Arrived broken, packaging was terrible."
Output: {"product": "unknown", "sentiment": "negative", "score": 1}

Review: "It works. Nothing special."
Output: {"product": "unknown", "sentiment": "neutral", "score": 3}

### Your Task ###
Review: "The laptop runs fast but the battery dies after 2 hours."
Output:
Research Finding
Brown et al. (2020) — the GPT-3 paper — showed that 4 examples in the prompt can match fine-tuned performance on classification tasks. Quality of examples > quantity.

Chain of Thought (CoT): Reasoning Out Loud

Instruct the model to produce intermediate reasoning steps before the final answer. This dramatically improves performance on math, logic, and multi-step inference.

CoT Pattern
Answer = f(Problem + "Let's think step by step")
Wei et al. (2022) showed this single phrase improves accuracy on arithmetic benchmarks by 40–70%.
✗ Without CoT
A shop sells 3 types of apples: Red (₹5/kg), Green (₹7/kg), Yellow (₹6/kg). I buy 2kg Red, 1.5kg Green, and 3kg Yellow. What is my total bill?
⚠ Model might just output ₹38 (wrong) by miscalculating
✓ With CoT
A shop sells apples: Red (₹5/kg), Green (₹7/kg), Yellow (₹6/kg). I buy 2kg Red, 1.5kg Green, 3kg Yellow. Think step by step, then give the total.
✓ Red: 2×5=₹10 → Green: 1.5×7=₹10.5 → Yellow: 3×6=₹18 → Total: ₹38.50
CoT Prompt — Structured Reasoning
You are a careful reasoning assistant. When given a problem:
1. Break it into steps labeled [Step 1], [Step 2], etc.
2. Show your working at each step.
3. State your final answer clearly after [Final Answer].

Problem: A data pipeline processes 50,000 records/hour.
After an upgrade, throughput increased by 35%.
If we run it for 8 hours, how many records are processed total?

Role Prompting: Activating Domain Expertise

Assigning a role shifts vocabulary, reasoning style, and domain-specific heuristics. It works because instruction-tuned models encode different "modes" associated with different professional contexts.

RoleEffect on OutputBest For
Senior Software EngineerFormal, DRY code, comments, edge casesCode review, architecture design
Data ScientistStatistical rigor, caveats, methodologyAnalysis, model evaluation
Legal ExpertFormal language, hedging, precedent referencesContract review (with caution)
Friendly TeacherAnalogies, simple language, encouraging toneExplanations for beginners
Devil's AdvocateCounter-arguments, risks, stress testing ideasStrategy, decision-making

ReAct: Reason + Act

ReAct prompts the model to interleave thought (reasoning) with action (tool calls, searches). This is the foundation of modern AI agents.

ReAct Loop
While not done: Thought → Action → Observation → next Thought
Yao et al. (2022): ReAct outperforms CoT alone on knowledge-intensive QA by 11%.
ReAct Prompt Template
You have access to: [Search(query), Calculator(expression), ReadFile(path)]

Use this format:
  Thought: [your reasoning]
  Action: [tool_name]([args])
  Observation: [result of action]
  ... (repeat as needed)
  Final Answer: [your answer]

Question: What is the current population of Bengaluru, and how many
times larger is it than Pune?

Thought: I need current population data for both cities.
Action: Search("Bengaluru population 2025")
Observation: Bengaluru population is approximately 14.2 million (2025).
Thought: Now I need Pune's population.
Action: Search("Pune population 2025")
Observation: Pune population is approximately 7.7 million (2025).
Thought: Now calculate the ratio.
Action: Calculator(14.2 / 7.7)
Observation: 1.844
Final Answer: Bengaluru (~14.2M) is approximately 1.84× larger than Pune (~7.7M).

Real-Life Use Cases

Each use case follows the Problem → Prompt Solution → Expected Output pattern with complete, copy-pasteable prompts.

📊
Data Analysis — Explain a CSV to a Non-Technical Manager
▼
⚡ Problem

You have a sales CSV with 12 columns. Your manager wants a written summary — no code, no jargon — sent before the 9am meeting.

✦ Prompt Solution
Prompt
You are a business analyst writing for a non-technical audience.

Below is a summary of our Q2 sales data:

[PASTE CSV SUMMARY OR KEY STATS HERE]

Task:
1. Write a 3-paragraph executive summary (max 150 words total).
2. Highlight the single most important trend.
3. Suggest one concrete action based on the data.

Rules:
- No technical terms, no code, no statistics notation.
- Use plain English. Active voice only.
- Format: Paragraph 1 (Overview), Paragraph 2 (Key Insight), Paragraph 3 (Recommendation).
→ Expected Output Shape

Three clear paragraphs. The model will translate numbers into human narratives like "Sales in the South region grew faster than any other territory…"

🐛
Code Debugging — Root Cause Analysis with Fix
▼
⚡ Problem

Your Python script crashes with a cryptic error. You need a root cause explanation + a corrected version.

✦ Prompt Solution
Prompt
You are a senior Python developer doing code review.

Here is a buggy script:
```python
[PASTE YOUR CODE HERE]
```

Error message received:
```
[PASTE ERROR TRACEBACK HERE]
```

Please:
1. Identify the root cause of the bug in one sentence.
2. Explain WHY this error occurs (concisely, 2-3 sentences).
3. Provide the corrected code with inline comments marking your changes.
4. List any other potential issues you spot (even if not causing the current error).

Format your response with headers: ## Root Cause, ## Explanation, ## Fixed Code, ## Other Issues
✉️
Email Writing — Difficult Professional Email
▼
⚡ Problem

You need to tell a client their project will be delayed by 2 weeks. The email must be professional, honest, but not panicky.

✦ Prompt Solution
Prompt
Write a professional email to a client informing them of a 2-week project delay.

Context:
- Client: Ravi Sharma, Director at TechCorp (senior, formal relationship)
- Project: Custom CRM integration, originally due: March 15
- Reason for delay: Third-party API vendor took longer than expected
- New deadline: March 29
- We are offering: A free 1-month post-delivery support extension as compensation

Tone: Professional, apologetic but confident, solution-focused.
Length: 150-200 words.
Do NOT use: Passive voice, vague language, excessive apologies (max 1 apology).
DO include: Specific new date, specific compensation, next steps.

Data Cleaning — Generate Pandas Code from Plain English
▼
⚡ Problem

You have a messy CSV with missing values, mixed date formats, and duplicate rows. You need a Python cleaning script.

✦ Prompt Solution
Prompt
You are a data engineer. Write a Python script using pandas to clean a CSV file.

Dataset description:
- File: sales_data.csv
- Columns: order_id, customer_name, order_date, amount, region, status
- Known issues:
  1. order_date has mixed formats: "DD/MM/YYYY" and "YYYY-MM-DD"
  2. amount column has some entries like "₹1,200.50" (string with currency symbol)
  3. ~5% of rows are exact duplicates
  4. region has values: "North", "north", "NORTH" (case inconsistency)
  5. status has blank strings ("") which should be NaN

Requirements:
- Add clear comments explaining each cleaning step
- Print a before/after row count
- Save cleaned file as sales_data_clean.csv
- Use only pandas (no external libraries)

📚
Teaching — Generate Tiered Explanations (Feynman Technique)
▼
⚡ Problem

You're creating educational content for mixed audiences — some beginners, some advanced. One explanation doesn't fit all.

✦ Prompt Solution
Prompt
Explain "machine learning overfitting" at three different levels.

Level 1 — ELI5 (Explain Like I'm 5):
Use a simple real-world analogy. Max 3 sentences. No technical terms.

Level 2 — Undergraduate Student:
Define it properly. Explain causes and consequences. Mention train/test split.
Use a simple formula or visual description if helpful. ~100 words.

Level 3 — Practitioner / ML Engineer:
Cover bias-variance tradeoff, regularization techniques (L1/L2, dropout),
cross-validation, and early stopping. Include code snippet showing
train_loss vs val_loss plotting. ~200 words.

Format each level with a clear header.
Chapter 05

Advanced Patterns & Formulas

The RFTF Framework

A reliable mental checklist for any prompt:

RFTF Formula
Prompt = Role + Frame the Task + Task Specifics + Format
Role: "You are a…"  ·  Frame: Why/context  ·  Task: Verb + object + constraints  ·  Format: Output structure

The Escalating Constraint Pattern

Start broad, then progressively add constraints until output quality plateaus. This is how you calibrate prompts empirically.

Escalating Constraints
# Iteration 1 (baseline)
"Summarize this article."

# Iteration 2 (add length)
"Summarize this article in under 80 words."

# Iteration 3 (add audience)
"Summarize this article in under 80 words for a high school student."

# Iteration 4 (add format)
"Summarize this article in under 80 words for a high school student.
Use: 1 sentence overview + 3 bullet points."

# Iteration 5 (add negative constraint)
"Summarize this article in under 80 words for a high school student.
Use: 1 sentence overview + 3 bullet points.
Do not use jargon. Do not editorialize."

Output Format Schemas

Forcing JSON/YAML output makes downstream processing reliable and eliminates parsing guesswork.

JSON Schema Pattern
Extract the following information from the job posting below.
Respond ONLY with valid JSON. No explanation, no markdown fences.

Schema:
{
  "job_title": "string",
  "company": "string",
  "location": "string or null",
  "remote": "boolean",
  "required_skills": ["string"],
  "salary_range": {"min": number_or_null, "max": number_or_null, "currency": "string"},
  "experience_years": number_or_null
}

Job Posting:
[PASTE JOB DESCRIPTION HERE]

The Anchor Pattern (Preventing Hallucination)

When accuracy matters, anchor the model to a specific document and forbid inference outside it.

Grounding / RAG Prompt
You are a document Q&A assistant. Your ONLY job is to answer questions
using information from the document below.

Rules:
- If the answer is in the document: quote the relevant line, then explain.
- If the answer is NOT in the document: respond exactly with "Not found in document."
- Never use outside knowledge. Never guess.

DOCUMENT:
===
[PASTE YOUR DOCUMENT HERE]
===

QUESTION: [USER'S QUESTION]

Self-Evaluation / Critic Pattern

Ask the model to critique its own output. This is a meta-cognitive pattern that surfaces errors the model made in step 1.

Self-Critique Prompt
Step 1: [Perform the task]
Write a Python function to find the largest prime factor of a number.

Step 2: Critique your solution.
- Is it correct for edge cases (0, 1, negative numbers, primes)?
- Is the time complexity optimal?
- What would you change if this ran on a number with 20 digits?

Step 3: Rewrite the function incorporating your critique.

Chapter 07

Common Pitfalls & How to Fix Them

⚠️
Pitfall 1: Ambiguous Task Verbs
▼
✗ Vague
"Process this data." / "Handle this text." / "Do something with the list."
⚠ "Process" could mean filter, transform, summarize, sort — or all of the above.
✓ Precise Verbs
Use: Extract / Classify / Summarize / Translate / Generate / Compare / List / Calculate / Format / Rewrite
✓ Each verb has an unambiguous meaning. The model knows exactly what to do.
⚠️
Pitfall 2: Overloaded Single Prompt
▼

Asking for 10 different things in one prompt degrades quality. The model treats it as a multi-objective function — and satisfices rather than optimizes.

⚠ Rule of Thumb
If your prompt has more than 3–4 distinct deliverables, split it into a prompt chain: output of step 1 becomes input of step 2.
⚠️
Pitfall 3: Trusting Numbers Without Grounding
▼

LLMs confidently fabricate statistics, citations, and figures. Never ask "what is the revenue of Company X?" without providing the data.

⚠ Fix
For any factual output: use the Anchor Pattern (inject the source) or explicitly add If you don't know, say "I don't know" — do not guess.
⚠️
Pitfall 4: Ignoring Temperature for the Task Type
▼
Task TypeRecommended TemperatureWhy
Factual Q&A, CodeT = 0.0 – 0.2Deterministic, accurate
Summarization, ClassificationT = 0.3 – 0.5Balanced
Creative Writing, BrainstormingT = 0.7 – 1.0Diverse, surprising
Poetry, ExperimentalT = 1.0 – 1.5High variance, wild outputs
