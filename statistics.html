Measures of Central Tendency

The simplest question in statistics: "Where is the data centred?" Central tendency gives us a single representative value for a dataset — a summary that captures the typical observation.

Mean (Arithmetic Average)

Sum all values and divide by the count. Sensitive to outliers. The "balance point" of your data.

Median

The middle value when data is sorted. Robust to extreme outliers. Best for skewed data like income.

Mode

The most frequently occurring value. Useful for categorical data (e.g. most common shoe size sold).

Mean

x̄ = (1/n) · Σi=1n xi    |    μ = (1/N) · Σi=1N xi
SymbolNameWhat it represents
x̄x-bar (sample mean)Average of a sample drawn from a population
μmu (population mean)True average of the entire population (usually unknown)
nsample sizeNumber of observations in your sample
Npopulation sizeTotal number of elements in the whole population
Σsigma (summation)Sum all values from i=1 to n
xix sub-iThe i-th individual data point
📍 Real-Life Example — Employee Salaries

Scenario: A startup has 5 employees with monthly salaries (₹): 30,000 · 35,000 · 40,000 · 42,000 · 5,00,000 (CEO)

Step 1. Sum = 30,000 + 35,000 + 40,000 + 42,000 + 5,00,000 = 6,47,000
Step 2. Mean = 6,47,000 ÷ 5 = ₹1,29,400
Insight. The mean is ₹1,29,400 — but 4 out of 5 employees earn far less! The CEO's salary pulled the mean up drastically. This is why median is better for skewed distributions like income.

Median

Median = x(n+1)/2  [odd n]  |   Median = (xn/2 + x(n/2)+1) / 2  [even n]
SymbolNameMeaning
x(k)order statisticThe k-th value after sorting the data in ascending order
nsample sizeTotal count of observations
📍 Same Salaries — Median

Sorted: 30,000 · 35,000 · 40,000 · 42,000 · 5,00,000

n = 5 (odd) → Median = x(5+1)/2 = x3 = ₹40,000

The median of ₹40,000 is a much more representative salary here!

💡 Rule of thumb: Use mean when data is symmetric (no extreme outliers). Use median when data is skewed or has outliers (income, house prices, wait times). Use mode for categorical data.

Measures of Spread (Dispersion)

Central tendency tells us where data is centred. Spread tells us how scattered it is. Two datasets can have the same mean but wildly different spreads — dispersion is what distinguishes them.

Range

Max − Min. Simple but fragile; one outlier breaks it.

Variance (σ², s²)

Average squared deviation from the mean. Penalises large deviations heavily.

Std Deviation (σ, s)

Square root of variance. Same units as data — most interpretable spread measure.

IQR

Q3 − Q1. Middle 50% spread. Resistant to outliers — used in box plots.

Variance & Standard Deviation

s² = Σ(xi − x̄)² / (n−1)  [Sample]    |    σ² = Σ(xi − μ)² / N  [Population]
s = √s²    |    σ = √σ²
SymbolNameMeaning
s²sample varianceEstimated spread of a sample; uses n−1 (Bessel's correction) to be unbiased
σ²population varianceTrue spread of the entire population; divides by N
n−1degrees of freedomWe lose 1 degree of freedom because x̄ is estimated from the same data
(xi−x̄)²squared deviationSquaring makes all deviations positive and penalises large errors more
ssample std devSpread in original units — "on average, data is ±s from the mean"
📍 Real-Life Example — Two Coffee Shops

Shop A wait times (min): 5, 5, 5, 5, 5   → Mean = 5 min

Shop B wait times (min): 1, 3, 5, 7, 9   → Mean = 5 min

Shop A. All times = 5 → deviations = 0 → s² = 0 → s = 0. Perfectly consistent!
Shop B. Deviations: (1−5)²+(3−5)²+(5−5)²+(7−5)²+(9−5)² = 16+4+0+4+16 = 40 → s² = 40/(5−1) = 10 → s = 3.16 min
Insight. Same average wait, but Shop B has 3× more unpredictability. If you hate uncertainty, choose Shop A.

IQR & Five-Number Summary

IQR = Q3 − Q1    |    Outlier if: x < Q1 − 1.5·IQR  or  x > Q3 + 1.5·IQR
SymbolNameMeaning
Q11st quartile (P25)25th percentile — 25% of data falls below this value
Q2Median (P50)50th percentile — the middle of the dataset
Q33rd quartile (P75)75th percentile — 75% of data falls below this value
IQRInter-Quartile RangeThe range of the central 50% of data — robust to outliers
1.5·IQRTukey fenceStandard rule for classifying extreme outliers in box plots

import numpy as np

data = [4, 8, 15, 16, 23, 42, 100]

rng      = np.ptp(data)                    # Range = max - min
variance = np.var(data, ddof=1)           # Sample variance (ddof=1 = Bessel)
std_dev  = np.std(data, ddof=1)           # Sample standard deviation
q1, q3   = np.percentile(data, [25, 75])
iqr      = q3 - q1

print(f"Range    : {rng}")
print(f"Variance : {variance:.2f}")
print(f"Std Dev  : {std_dev:.2f}")
print(f"Q1={q1}, Q3={q3}, IQR={iqr}")

# Outlier detection (Tukey's fences)
lower, upper = q1 - 1.5*iqr, q3 + 1.5*iqr
outliers = [x for x in data if x < lower or x > upper]
print(f"Outliers : {outliers}")

Probability Basics

Probability is the language of uncertainty. It quantifies how likely an event is to occur on a scale from 0 (impossible) to 1 (certain). Everything in statistics — inference, prediction, machine learning — is built on probability.

Core Definitions

P(A) = |A| / |S|   [Classical]    |    0 ≤ P(A) ≤ 1    |    P(S) = 1
SymbolNameMeaning
P(A)Probability of ALikelihood of event A occurring (between 0 and 1)
|A|cardinality of ANumber of outcomes favourable to event A
|S|cardinality of STotal number of equally likely outcomes in sample space S
Ssample space (Ω)Set of ALL possible outcomes of an experiment
A'or Aᶜcomplement of AAll outcomes NOT in A; P(Aᶜ) = 1 − P(A)

Addition & Multiplication Rules

P(A ∪ B) = P(A) + P(B) − P(A ∩ B)
P(A ∩ B) = P(A) · P(B)  [if A, B independent]
SymbolNameMeaning
A ∪ BUnion (OR)Event A or event B (or both) occur
A ∩ BIntersection (AND)Both event A and event B occur simultaneously
P(A ∩ B)Joint probabilityFor independent events: simply multiply their probabilities
📍 Real-Life Example — Quality Control

Scenario: In a factory, 5% of products are defective. You pick 2 products at random (with replacement). What is the probability that at least one is defective?

Step 1. P(defective) = 0.05 → P(not defective) = 0.95
Step 2. P(both OK) = 0.95 × 0.95 = 0.9025  (independent events)
Step 3. P(at least 1 defective) = 1 − P(both OK) = 1 − 0.9025 = 0.0975 ≈ 9.75%
Insight. Using the complement rule (1 − P(none)) is often easier than adding individual cases.
Python 
import random

# Simulate the factory quality control problem
p_defective = 0.05
trials = 100_000
at_least_one = 0

for _ in range(trials):
    p1 = random.random() < p_defective
    p2 = random.random() < p_defective
    if p1 or p2:
        at_least_one += 1

simulated = at_least_one / trials
analytical = 1 - (1 - p_defective)**2

print(f"Simulated  : {simulated:.4f}")
print(f"Analytical : {analytical:.4f}")
# Both ≈ 0.0975

Conditional Probability & Bayes' Theorem

Conditional probability asks: "Given that something has already happened, how does that change the probability of another event?" Bayes' theorem is its most powerful application — updating beliefs with new evidence.

Conditional Probability

P(A | B) = P(A ∩ B) / P(B)
SymbolNameMeaning
P(A|B)"P of A given B"Probability of A, knowing that B has already occurred
P(A ∩ B)joint probabilityProbability that both A and B occur
P(B)marginal probabilityProbability of B (must be > 0 to condition on it)

Bayes' Theorem

P(H | E) = P(E | H) · P(H) / P(E)
SymbolNameMeaning
P(H|E)PosteriorUpdated belief about hypothesis H after seeing evidence E
P(E|H)LikelihoodHow likely is evidence E if hypothesis H were true?
P(H)PriorInitial belief about H before seeing any evidence
P(E)Marginal likelihoodTotal probability of evidence E across all hypotheses
📍 Real-Life Example — Medical Test

Disease prevalence: 1% of population has disease D

Test accuracy: 99% sensitivity (true positive), 95% specificity (true negative)

Question: If you test positive, what is the probability you actually have the disease?

Given. P(D) = 0.01 | P(+|D) = 0.99 | P(+|no D) = 0.05
P(+). P(+) = P(+|D)·P(D) + P(+|no D)·P(no D) = 0.99×0.01 + 0.05×0.99 = 0.0099 + 0.0495 = 0.0594
Bayes. P(D|+) = (0.99 × 0.01) / 0.0594 = 0.0099 / 0.0594 ≈ 16.7%
Insight. Even with a 99% accurate test, a positive result only means a 16.7% chance of actually being sick — because the disease is rare! This is the base rate fallacy.
Python 
# Bayes' Theorem — Medical Test Example
p_disease   = 0.01   # Prior: 1% prevalence
sensitivity = 0.99   # P(+ | disease)
fpr         = 0.05   # False positive rate P(+ | no disease)

p_no_disease = 1 - p_disease

# Total probability of testing positive (law of total probability)
p_positive = (sensitivity * p_disease) + (fpr * p_no_disease)

# Bayes: P(disease | positive)
p_disease_given_pos = (sensitivity * p_disease) / p_positive

print(f"P(+) = {p_positive:.4f}")
print(f"P(disease | positive test) = {p_disease_given_pos:.4f} = {p_disease_given_pos*100:.1f}%")
# ≈ 16.7% — surprisingly low!

Combinatorics — Counting Methods

Combinatorics is the art of counting without listing. Whether it's lottery odds, password strength, or A/B test arrangements — being able to count possibilities is the backbone of probability calculations.

Permutations

Order matters. How many ways to arrange r items from n? (e.g. medals: gold, silver, bronze)

Combinations

Order doesn't matter. How many ways to choose r items from n? (e.g. lottery numbers)

Multiplication Rule

If one task has m ways and another has n ways, together they have m × n ways.

Factorial, Permutations, Combinations

n! = n × (n−1) × ... × 2 × 1
P(n,r) = n! / (n−r)!    |    C(n,r) = n! / (r! · (n−r)!)
SymbolNameMeaning
n!n factorialTotal arrangements of n distinct items; 0! = 1 by definition
P(n,r)nPr, permutationOrdered selections of r items from n (order matters)
C(n,r)nCr, "n choose r"Unordered subsets of r items from n (order doesn't matter)
ntotal itemsPool size — total number of items to choose from
rchosen itemsHow many items are selected at once
(n−r)!discarded arrangementsRemoves the ordering of items not chosen (permutation)
r!r factorialRemoves duplicate orderings of chosen items (combination)
📍 Real-Life Example — Password Strength & Lottery

Problem 1: A 4-digit PIN from digits 0–9. How many unique PINs?

Answer. Repetition allowed → 10 × 10 × 10 × 10 = 10,000 (Multiplication Rule)

Problem 2: Lottery: Pick 6 numbers from 1–49. How many possible tickets?

Answer. C(49,6) = 49! / (6! × 43!) = 13,983,816 possible tickets!

Problem 3: A team of 3 wins gold, silver, bronze from 10 athletes. How many outcomes?

Answer. P(10,3) = 10! / 7! = 10 × 9 × 8 = 720 ordered outcomes

The Normal Distribution

The bell curve. The most important distribution in all of statistics. Heights, blood pressure, exam scores, measurement errors — nature loves the normal distribution. Understanding it unlocks most of classical statistics.

f(x) = (1 / (σ√(2π))) · e−½((x−μ)/σ)²
SymbolNameMeaning
f(x)PDF valueProbability Density Function — height of the curve at x (not a probability itself)
μmeanCentre of the bell curve — the peak is always at x = μ
σstandard deviationControls the spread (width) of the bell; larger σ = flatter/wider curve
eEuler's number≈ 2.71828 — the base of natural logarithm
πpi≈ 3.14159 — appears in the normalising constant √(2π)
N(μ, σ²)normal notationShorthand: "X follows a Normal distribution with mean μ and variance σ²"

The Standard Normal & Z-Score

z = (x − μ) / σ    |    Z ~ N(0, 1)
SymbolNameMeaning
zz-score (standard score)How many standard deviations x is above/below the mean
Z ~ N(0,1)standard normalSpecial case: mean=0, variance=1; all normal problems reduce to this
Φ(z)CDF of standard normalP(Z ≤ z) — area under the curve to the left of z
📍 Empirical Rule (68-95-99.7)

Exam scores follow N(μ=70, σ=10).

68%. P(60 ≤ X ≤ 80) = P(μ−σ ≤ X ≤ μ+σ) ≈ 68.27%
95%. P(50 ≤ X ≤ 90) = P(μ−2σ ≤ X ≤ μ+2σ) ≈ 95.45%
99.7%. P(40 ≤ X ≤ 100) = P(μ−3σ ≤ X ≤ μ+3σ) ≈ 99.73%
Z-score. What percentile is a score of 85? z = (85−70)/10 = 1.5 → Φ(1.5) ≈ 93.3rd percentile

Binomial & Poisson Distributions

Not all data is continuous. Sometimes we count — number of successes, number of arrivals. Binomial models binary outcomes (success/fail); Poisson models rare events in a fixed interval.

Binomial Distribution

P(X = k) = C(n,k) · pk · (1−p)n−k
E[X] = np    |    Var(X) = np(1−p)
SymbolNameMeaning
ntrialsFixed number of independent binary trials
ksuccessesNumber of successful outcomes we're calculating probability for
psuccess probabilityProbability of success on any single trial (constant)
(1−p) = qfailure probabilityProbability of failure on each trial
C(n,k)binomial coefficientNumber of ways to arrange k successes in n trials
npexpected value E[X]Average number of successes over many experiments
📍 Real-Life Example — Marketing Email Campaign

Scenario: An email has a 20% open rate. You send 10 emails. What's P(exactly 3 opens)?

Given. n=10, k=3, p=0.20
Calc. P(X=3) = C(10,3) × 0.20³ × 0.80⁷ = 120 × 0.008 × 0.2097 = 0.2013 ≈ 20.1%
Mean. E[X] = 10 × 0.20 = 2 opens expected

Poisson Distribution

P(X = k) = (λk · e−λ) / k!
E[X] = λ    |    Var(X) = λ
SymbolNameMeaning
λlambda (rate)Average number of events per interval (mean = variance = λ)
kobserved eventsThe specific count whose probability we're calculating (k = 0,1,2,...)
e−λPoisson kernelNormalising factor ensuring all probabilities sum to 1
📍 Real-Life Example — Customer Arrivals

Scenario: A coffee shop gets an average of λ=3 customers per minute. What's P(exactly 5 arrive in 1 minute)?

Calc. P(X=5) = (3⁵ × e⁻³) / 5! = (243 × 0.0498) / 120 = 0.1008 ≈ 10.1%
P(≥1). P(X≥1) = 1 − P(X=0) = 1 − e⁻³ = 1 − 0.0498 = 95%
Python 
from scipy.stats import binom, poisson
import matplotlib.pyplot as plt

axes[1].set_title(f'Poisson(λ={lam})')
print(f"P(X=5) = {poisson.pmf(5, lam):.4f}")   # ≈ 0.1008

plt.tight_layout()
plt.show()

Central Limit Theorem

The most remarkable result in statistics: regardless of the shape of the original population distribution, the distribution of sample means approaches a normal distribution as the sample size grows. This is why normal distributions appear everywhere in practice.

x̄ ~ N(μ, σ²/n)    as n → ∞
SE = σ / √n    |    z = (x̄ − μ) / (σ/√n)
SymbolNameMeaning
x̄sample meanMean calculated from one sample of size n
μpopulation meanTrue mean of the population we're sampling from
σ²/nvariance of x̄Variance shrinks by factor n — larger samples → more precise estimate
SEStandard ErrorStandard deviation of the sampling distribution of x̄; SE = σ/√n
n → ∞as n grows largeRule of thumb: n ≥ 30 is usually sufficient for CLT to kick in
📍 Real-Life Example — Delivery Times

Population: Delivery times follow a skewed distribution with μ=45 min, σ=20 min

Question: Take samples of n=36. What's P(sample mean ≤ 42 min)?

SE. SE = σ/√n = 20/√36 = 20/6 = 3.33 min
Z-score. z = (42 − 45) / 3.33 = −3/3.33 = −0.9
P. P(x̄ ≤ 42) = Φ(−0.9) ≈ 18.4%
Key Insight. Even though individual times are skewed, the distribution of sample means is approximately normal because n=36 ≥ 30!

Hypothesis Testing

The scientific method for data: start with a null hypothesis (status quo), collect data, and decide whether the evidence is strong enough to reject it. Used everywhere from clinical trials to A/B tests.

H0: μ = μ0    (Null)    |    Ha: μ ≠ μ0 or μ > μ0 or μ < μ0    (Alternative)
t = (x̄ − μ0) / (s / √n)      z = (x̄ − μ0) / (σ / √n)
SymbolNameMeaning
H0Null hypothesisThe "no effect / no difference" claim we're trying to disprove
HaAlternative hypothesisWhat we suspect is actually true (our scientific claim)
αsignificance levelThreshold for rejection; typically 0.05 (5% chance of false positive)
p-valueprobability valueP(observing this data or more extreme | H₀ is true) — NOT P(H₀ is true)
ttest statisticHow many standard errors x̄ is from the hypothesised μ₀
Type I errorFalse positiveRejecting H₀ when it's actually true; rate = α
Type II errorFalse negativeFailing to reject H₀ when H₀ is actually false; rate = β
📍 Real-Life Example — A/B Test (Website CTR)

Scenario: Old button (Control): average CTR = 5%. New button (Treatment): 50 users tested, sample mean CTR = 7%, s = 3%. Is the improvement real or just noise?

H₀. μ = 5% (no change)
Hₐ. μ > 5% (one-tailed — we only care if it improved)
t-stat. t = (7 − 5) / (3 / √50) = 2 / 0.424 = 4.72
p-value. t(4.72, df=49) → p ≈ 0.000012 (extremely small!)
Decision. p < α=0.05 → Reject H₀. The new button's CTR is significantly higher!
Python 
import numpy as np
from scipy.stats import ttest_1samp, ttest_ind

# One-sample t-test: is new CTR > 5%?
np.random.seed(42)
n, mu0, sample_mean, sample_std = 50, 5.0, 7.0, 3.0

# Simulate sample data matching our statistics
sample = np.random.normal(sample_mean, sample_std, n)

t_stat, p_value = ttest_1samp(sample, mu0)
p_one_tail = p_value / 2   # Convert to one-tailed

print(f"t-statistic : {t_stat:.4f}")
print(f"p-value (one-tailed) : {p_one_tail:.6f}")
print(f"Decision : {'Reject H0 ✓' if p_one_tail < 0.05 else 'Fail to reject H0'}")

# --- Two-sample t-test (compare two groups) ---
group_a = np.random.normal(5.0, 3.0, 50)   # Control
group_b = np.random.normal(7.0, 3.0, 50)   # Treatment
t2, p2 = ttest_ind(group_a, group_b)
print(f"\nTwo-sample t-test p = {p2:.6f}")

Correlation & Linear Regression

Correlation asks "are two variables related?" Regression asks "if X changes, how much does Y change?" One of the most widely used tools in data science, economics, and science.

Pearson Correlation Coefficient

r = Σ[(xi−x̄)(yi−ȳ)] / √[Σ(xi−x̄)² · Σ(yi−ȳ)²]
SymbolNameMeaning
rPearson rCorrelation coefficient: −1 ≤ r ≤ 1. r=1 perfect positive, r=−1 perfect negative, r=0 no linear relation
ȳy-barMean of the Y variable
r²R-squaredProportion of variance in Y explained by X (coefficient of determination)

Simple Linear Regression

Ŷ = β0 + β1X    |    β1 = Σ(xi−x̄)(yi−ȳ) / Σ(xi−x̄)²    |    β0 = ȳ − β1x̄
SymbolNameMeaning
ŶY-hat (predicted)Model's predicted value of Y for a given X
β0interceptPredicted Y when X = 0 (y-intercept of the regression line)

import numpy as np
from sklearn.linear_model import LinearRegression
from sklearn.metrics import r2_score
import matplotlib.pyplot as plt

# Dataset: study hours vs exam score
hours  = np.array([2,3,4,5,6,7,8]).reshape(-1,1)
scores = np.array([55,60,65,70,75,80,85])

model = LinearRegression().fit(hours, scores)

beta0 = model.intercept_
beta1 = model.coef_[0]
r2    = r2_score(scores, model.predict(hours))

print(f"β₀ (intercept) = {beta0:.2f}")
print(f"β₁ (slope)     = {beta1:.2f}")
print(f"R²             = {r2:.4f}")
print(f"Prediction(9h) = {model.predict([[9]])[0]:.1f}")

# Plot
x_line = np.linspace(0, 10, 50).reshape(-1,1)
plt.scatter(hours, scores, color='#4f8ef7', zorder=5, label='Data')
plt.plot(x_line, model.predict(x_line), '#e8b84b', lw=2, label=f'Ŷ={beta0:.1f}+{beta1:.1f}X')
plt.legend()
plt.show()
